### 对抗性自检协议（V5.3.0 新增）

> 研究依据：Du et al.（ICML 2024, arXiv:2305.14325）证明多智能体辩论即使初始答案全错也能收敛到正确答案。SELF-REFINE（arXiv:2303.17651）证明CoT式自检提升约20%。

在开始质检前，必须先执行对抗性自检初始化：

```
【对抗性自检初始化】（V5.3.1 弹性区间版）
你现在是一个挑剔的质检编辑，你的任务是找出这份初稿中的真实问题。

审查原则：
1. 质量优先于数量：每个问题必须有具体位置+原文证据+修复建议
2. 误报惩罚：标记问题经复核不成立会降低审查可信度
3. 数量指引（非硬性下限）：
   - 初稿评分<60分：通常存在≥5个真实问题
   - 初稿评分60-79分：通常存在3-5个真实问题
   - 初稿评分≥80分：可能只有1-3个真实问题，找不到更多时如实报告

请先假设"这份初稿可能不合格"，然后列出不合格的具体证据，再综合判断是否真的不合格。

注意：你不是在确认"写得不错"，你是在找茬。每一处你认为没问题的地方，
请再问自己一遍："如果这是一个付费读者，他会觉得这里出戏吗？"
但不要为了凑数量而编造问题——误报会降低你的审查可信度。
```

### 混合评分模型（V5.3.1）

```
最终评分 = 脚本客观分（60分） + LLM主观分（40分）
         = 最高100分

┌─────────────────────────────────────────────────────────────┐
│ 第一部分：脚本客观分（60分）— 运行 check-quality-score.js   │
├─────────────────────────────────────────────────────────────┤
│ 情节推进力（15分）：信息密度检测，每500字分段计算            │
│ 节奏控制（15分）：burstiness CV≥0.5满分，<0.3得0分          │
│ 质感密度（10分）：感官词频统计 + 比喻密度≤3次/千字          │
│ 角色一致性（10分）：名称/数值/知识边界矛盾检测              │
│ AI味基线（10分）：blocking级问题计数（0个=10分）             │
└─────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────┐
│ 第二部分：LLM主观分（40分，不压缩）                          │
├─────────────────────────────────────────────────────────────┤
│ 情绪到达率（20分）：读者身体反应测试是否通过？               │
│   评分锚点：20=读者生理性紧张/15=能感知情绪/10=情绪可理解    │
│   但平淡/0=无情绪传递                                       │
│ 角色辨识度（20分）：遮名测试，对话是否能区分说话人？         │
│   评分锚点：20=遮名可辨/15=有口癖/10=区分度弱/0=所有人说话   │
│   一样                                                      │
│                                                              │
│ V5.3.1: 客观分主导+主观分不压缩，不再使用0.8校准因子         │
└─────────────────────────────────────────────────────────────┘

执行方式：
1. 先运行 node scripts/check-quality-score.js --file=正文路径 --metadata=设定JSON
2. 获取脚本客观分（0-60分）和客观维度详情
3. LLM仅评估"情绪到达率"和"角色辨识度"两个主观维度（各20分）
4. 最终评分 = 脚本客观分 + LLM主观分（最高100分）

│ 评分区间     │ 后续流程               │ 下游执行步骤               │
├─────────────┼───────────────────────┼──────────────────────────┤
│ ≥80分       │ 快速通道 /flow_fast    │ 04b 深度打磨 → 脚本收尾    │
│ 60-79分     │ 完整流程 /flow_full    │ 04 精修 → 05 去AI味 → 脚本 │
│ <60分       │ 返工重写               │ 退回03a重写 → 重新03b质检  │
└─────────────┴───────────────────────┴──────────────────────────┘
```

## burstiness 检测（V5.1 新增 — 句长变异度检测）

> AI生成文本句长分布过于均匀（缺乏"突发性"），是AI味的隐性特征。人类写作有自然长短交替。

```bash
node scripts/check-ai-patterns.js --json <本章正文文件>
```

脚本 `sentence-variance` 检测器输出：句长标准差、变异系数（CV=标准差/平均句长）、burstiness指数

| 指标 | 通过 | 标记 | 拦截 |
|------|------|------|------|
| 变异系数CV | ≥0.5 | 0.3-0.5 | <0.3 |
| 连续等长句（±2字） | ≤3句 | 4-6句 | ≥7句 |

修正：CV过低→在关键情绪点插入极短句（3-5字）或长句（40字+）| 连续等长句过多→长短句交替重写。锐度保护清单中标记的"短句节奏"类表达享豁免权。

## quality-checker Agent 协作说明（V5.2.0 — 每章必执行）

> 03b自检存在"确认偏误"。引入独立Agent做对抗式审查，从"挑刺"视角发现漏检问题。
> **V5.2.0 变更**：从"可选步骤，关键章节时启用"改为"**每章必执行**"。Agent不可用时降级为主线程自检，但必须在质检报告中标注"Agent未执行（降级模式）"。

```
1. 03b完成自检+质量Gate→产出【03b质检报告】
2. 将【03a初稿】+【03b质检报告】+【衔接包】交给 quality-checker Agent
3. Agent角色：挑剔编辑，目标"找出03b漏检的问题"
   ├─ 审查：03b标记"通过"的项是否真通过？是否有未覆盖维度？
   ├─ 锐度保护清单中标记的表达是否被误判为问题？
   └─ 质量Gate评分是否合理？
   输出：【对抗审查报告】（异议项+理由+建议）
4. 03b收到报告→逐条评估：接受→修正质检报告→反馈03a | 驳回→标注理由→交付下游供人工判断
```

Agent审查是建议性的，不自动覆盖03b结论。分歧项标注"待人工裁决"。

**降级机制**：当 quality-checker Agent 不可用时（如平台不支持Agent调用）：
- 03b主线程自行执行"对抗式自查"——以"挑剔编辑"视角重新通读全文
- 在质检报告中标注"⚠️ Agent降级模式（主线程自检），对抗性审查强度降低"
- 建议用户在关键章节（黄金三章/卷末高潮）手动调用Agent进行补充审查

## 冲突四件套完整性检查（V5.1 新增 — 防止"伪冲突"）

> AI扩写容易写出"看起来有冲突但实际没有张力"的段落。本检查确保每个核心冲突具备完整四要素。

```
【冲突四件套 — 每个核心冲突必须6项全"是"才合格】

1. 目标（Goal）：角色是否有明确、具体的想要达成的事？否=角色在"漂流"，需补充目标
2. 障碍（Obstacle）：是否有具体阻力（人/环境/规则/自身）阻止达成？否=无张力
3. 路径（Path）：角色是否采取了具体行动策略克服障碍？否=角色"等着事情发生"
4. 代价（Cost）：角色为克服障碍付出了什么（时间/资源/关系/身体/心理）？否=免费通关，无紧张感
5. 不确定性（Uncertainty）：读者是否无法提前预判结果？否=无悬念
6. 后果链（Consequence）：本冲突结果是否影响后续章节？否=孤立冲突，对主线无推动

执行规则：每章至少1个核心冲突6项全"是" | 过渡章可降级为至少4项是（目标/障碍/路径/代价必须有）
≥2项"否"→判定"伪冲突"→标记返工，补充缺失要素 | 检查时机：P1抽查时执行（抽查最核心1-2个冲突场景）
```

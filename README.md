## 设计理念

### 核心思想：工业化分工

网文创作的痛点不是"不会写"，而是"写了前面忘了后面、质量波动大、AI 味重"。本 OS 采用**三道工序流水线**：

```
基建层（Step 0-2）  →  生产层（Step 3-4）  →  精装层（Step 5-6）
项目+世界观+宏观结构      细纲+初稿               精修+去AI味
     ↓                       ↓                       ↓
   全书1次             滚动建纲+每章循环            每章循环
```

每一步有明确的**输入/输出/边界/不可越界**。下游发现问题，回溯到源头修复，不向下游打补丁。

### 三大流水线法则

| 法则 | 含义 |
|------|------|
| **衔接包唯一生产者** | 只有 Step 3（细纲）产出衔接包。Step 4 只读不写。 |
| **下游不重复上游工作** | Step 5 不重做 Step 4 的基础去 AI 味，Step 6 不重做 Step 5 的潜台词打磨。 |
| **回溯而非打补丁** | 精修阶段发现剧情逻辑问题 → 回溯到扩写甚至细纲阶段修复，而非在精修时改写情节。 |

### 番茄小说特供优化

- **爽感保护协议**：第 1 章前 500 字必须包含一个爽点
- **感官优先级倒置**：触觉/嗅觉 > 听觉 > 视觉（打破 AI 默认的"视觉优先"）
- **流氓逻辑约束**：主角话锋犀利、护短、果断，不圣母、不犹豫
- **低理解门槛**：避免复杂设定堆砌，读者 30 秒内能进入剧情

## 快速开始

### 第一次使用？

1. **告诉 AI 你的核心灵感** — 一句话即可
2. **AI 会问你 4 个问题**：类型/平台/字数/偏好
3. **确认后进入 Step 0**：创建项目目录结构
4. **沿着流水线一步步推进**，每步完成后 AI 会等你确认

```
你: "我想写一本都市重生小说，主角回到 2000 年用互联网知识改变命运"
AI: 确认信息 → Step 0 → Step 1 → Step 2 → ... → 出书
```

### 已有部分产物？

如果你已经写了一些内容，可以**从任意步骤开始**：

| 你已有什么 | 从 Step 几开始 |
|-----------|:---:|
| 只有灵感 | Step 0 |
| 已有设定文档 | Step 2（大纲） |
| 已有大纲 | Step 3（细纲） |
| 已有细纲 | Step 4（扩写） |
| 已有正文初稿 | Step 5（精修） |
| 已在平台发了部分章节 | Step 扩展 — 07(导入师) |

> AI 会要求你提供已完成的产物，确保下游质量不受影响。

### 模式控制

在任意 Skill 的 Prompt 开头添加模式指令：

| 指令 | 效果 |
|------|------|
| `/mode_assisted`（默认） | 全规则静默运行，减少主动建议 |
| `/mode_full` | 全规则激活，主动给出建议和提醒 |
| `/mode_hardcore` | 仅核心约束，跳过主动干预模块 |

## 七步流水线详解

### Step 0 — 项目初始化师

| 属性 | 值 |
|------|-----|
| **文件** | `00.5_项目初始化.md` (V1.0) |
| **频率** | 全书 1 次 |
| **输入** | 书名、作者、类型、平台、目标字数 |
| **产出** | 标准目录结构 + `上下文管理模板.md` |

**干什么**：创建 `设定/` `大纲/` `细纲/` `正文/` `追踪/` 五个标准目录，初始化断点续跑追踪文件。所有后续步骤的产出物都落到这个目录结构里。

**目录结构**：
```
{书名}/
├── 设定/          ← Step 1 设定文档 + JSON 元数据
├── 大纲/          ← Step 2 分卷大纲 + 情绪曲线图
├── 细纲/          ← Step 3 全书细纲 + 逐章衔接包链
│   └── 衔接包链/  ←  每章一个独立衔接包文件
├── 正文/
│   ├── 初稿/      ← Step 4 产出（逐章）
│   ├── 精修/      ← Step 5 产出（逐章）
│   └── 终稿/      ← Step 6 产出（逐章）
├── 追踪/          ← 上下文管理模板 + 伏笔追踪 + 断点快照
└── 对标/          ← 可选（Step 06 拆文产出）
```

### Step 1 — 小说设定架构师

| 属性 | 值 |
|------|-----|
| **文件** | `00_小说设定架构师.md` (V3.2) |
| **频率** | 全书 1 次 |
| **输入** | 核心灵感 + 基本要求 |
| **产出** | 结构化设定文档 + JSON 元数据 |

**产出的 JSON 元数据是整条流水线的标准数据接口**，包含：

```json
{
  "novel_meta": {     // 书名/类型/平台/字数/核心标签
  },
  "characters": [],    // 角色名(唯一标识)/性格/成长弧/关系
  "relationships": [], // 角色关系矩阵
  "anti_patterns": {   // 反例对照表 — 下游所有 skill 加载为约束
    "forbidden": [],   //   禁止使用的情感词/模板句式
    "required": []     //   必须遵守的约束（如感官优先级）
  }
}
```

**下游消费者**：Step 2（大纲）、Step 3（细纲）、Step 4（扩写）、Step 5（精修）、Step 6（去 AI 味）——全部依赖此 JSON。

**核心能力**：
- 番茄平台合规审查（敏感元素预检）
- 金手指设计 + 成长阶梯（升级体系）
- 角色关系动力学（CP/对手/师徒等）
- 素材融合与设定保鲜机制
- 防御性设计（扩展性与一致性自检）

### Step 2 — 小说大纲构建师

| 属性 | 值 |
|------|-----|
| **文件** | `01_小说大纲构建师.md` (V2.2) |
| **频率** | 全书 1 次 |
| **输入** | Step 1 的设定文档 + JSON 元数据 |
| **产出** | 分卷大纲 + 情绪曲线 + 核心卖点定位 |

**两阶段设计**：

**第一部分 — 读者方向诊断**：在情节设计之前，先回答"读者到底在为什么买单"——爽感/恐惧/情感拉扯/智商碾压/成长逆袭。

**第二部分 — 结构设计工具箱**：
- **角色位抽象**：将角色抽象为功能位（主角位/反派位/导师位/CP 位/谐星位/工具人位等），确保每个位都有不可替代的剧情功能
- **钩子 20 式**：章末 13 式 + 章首 7 式，连续 3 章不重复
- **反转 7 种类型**：身份/立场/真相/强弱/情感/认知/命运，含节奏控制
- **拆文推荐框架**：如何利用拆文数据指导大纲设计

**产出包含**：
- 每卷核心冲突与爽点分布图
- 黄金三章规划（前 3 章每一章的钩子/情绪温度/爽点类型）
- 全书伏笔计划
- 情感温度曲线预标注

### Step 3 — 细纲编写技能

| 属性 | 值 |
|------|-----|
| **文件** | `02_细纲编写技能.md` (V4.3) |
| **频率** | 滚动建纲（默认）/ 一次性全书建纲（/mode_full） |
| **输入** | Step 1 JSON 元数据 + Step 2 分卷大纲 |
| **产出** | 逐章细纲 + 衔接包链 + 伏笔追踪表 |

**这是全书连贯性的唯一总控**。Step 3 是衔接包（跨章上下文）的唯一生产者。

**章节蓝图五段式**（V4.3 新增）：
```
起因 → 发展 → 转折 → 高潮 → 结尾
```

**字数预算编排**：密(≥250字) / 疏(≈40字) / 铺垫(120-150字)

**大纲五检**：每卷/每章必答 5 问——情绪/冲突/节奏/伏笔/定位分布

**滚动建纲模式**（默认）：
1. 首次产出前 10 章细纲 + 衔接包链
2. 进入 Step 4 逐章扩写
3. 每写完 5 章触发"补纲"——回到 Step 3 补充后续 5-10 章细纲
4. 已进入写作的细纲**锁定**，未经确认不得修改

> 全书 ≤30 章时可一次全建。使用 `/mode_full` 可强制全书建纲。

**衔接包内容**（每章一个）：
- 前章回顾 + 角色状态表
- 延续细节（需要在本章出现的上章设定）
- **禁止矛盾清单**（如"李四左臂骨折"→本章不得双手操作）
- 新增状态（本章新增的状态/道具/关系变化）
- 伏笔追踪（ACTIVE → DORMANT → WAKING → CLOSED）

### Step 4 — 细纲扩写执行系统

| 属性 | 值 |
|------|-----|
| **文件** | `03_细纲扩写执行系统.md` (V4.3) |
| **频率** | **逐章循环** |
| **输入** | 每章细纲 + 衔接包 + Step 1 反例对照表 |
| **产出** | 初稿（3500-4500字） + 自检报告 + 定位报告 |

**核心使命**：骨架血肉——剧情推进流畅、逻辑不崩、字数达标、基础去 AI 味。**不做深度润色**（那是 Step 5 的工作）。

**规则分层体系**：
- **铁律**（违反 = 重写）：字数/章节结构/对话比例/衔接包禁止矛盾
- **弹性建议**（可破但要知道为什么破）：句式多样性/场景节奏
- **风格可选项**（按题材适配）：打斗密度/甜度/虐度

**生成-检测分离架构**（V4.0+）：
1. 先自由生成正文
2. 再对全章执行 9 项自检
3. 检测到问题 → 修正 → 重新检测

**9 项自检**（V4.3）：
0. 预检面板回执核对（远景/中景/特写比例）
1. 衔接包"禁止矛盾"清单核对
2. 铁律检查（字数/场景/结构类型）
3. 句式复用检测（已用句式清单喂入闭环）
4. 感官优先级检查
5. 对话比例检查
6. "不是 A 是 B"句式限流（每 3000 字 ≤2 次）
7. 章末收尾轮换（5 种轮换，不连续两章同款）
8. 跨章句式去重协议

**交付给 Step 5**：初稿 + 定位报告（标注句式单一/词汇重复/基础 AI 味超标区域）。

### Step 5 — 小说正文精修师

| 属性 | 值 |
|------|-----|
| **文件** | `04_小说正文精修师.md` (V4.3) |
| **频率** | **逐章循环** |
| **输入** | Step 4 初稿 + 定位报告 |
| **产出** | 精修正文（潜台词/金句/风格注入） |

**核心使命**：灵魂质感——不改变情节、不改变设定、不改变结构，只让读者读完"身体有反应"。

**第零部分：顶级严肃文学范式**（V4.0 最高优先级）：

从"叙述者视角"转换为"客观摄影机视角"，参考双雪涛的冷冽、麦家的隐秘、海明威的硬汉白描。

**五大最高铁律**：
1. **潜台词拦截器**：禁止直接写情绪词（"他很生气"→写他捏碎了杯子）
2. **封杀弱智明喻**：禁止"A 像 B"比喻句式，强制替换为物理痕迹法或生理反馈法
3. **防顺畅错位引擎**：禁止"因为 X 所以 Y"因果链，让读者自己补因果
4. **叙事块原则**：叙事块 = 空间细节 + 动作 + 感官 ≥3 句
5. **道具去戏剧化白描**：金手指/关键道具走白描路线，禁止光环化描写

**六大深度技法**：
- 潜台词打磨（冰山原则）
- 情感留白（不写满，让读者自己感受）
- 金句提炼（每章 2-3 句）
- 闲笔注入（与主线无关但增加真实感的细节）
- 风格注入（个人特色句式/语法习惯）
- 标点规范化（省略号 ≤5/破折号 ≤8/感叹号 ≤15 每章）

### Step 6 — 去 AI 味精修师

| 属性 | 值 |
|------|-----|
| **文件** | `05_去AI味精修师.md` (V1.3) |
| **频率** | **逐章循环** |
| **输入** | Step 5 精修正文 |
| **产出** | 终稿 + AI 味评分报告 + 脚本检查报告 |

**核心使命**：人味儿质检——让文字读起来像真人写的，而非 AI 生成的。

**11 Gate 系统**（顺序检测，不可跳跃）：

| Gate | 名称 | 检测内容 | 级别 |
|:----:|------|---------|:----:|
| 1 | 禁用词过滤 | 一级/二级禁用词清单 | 阻断 |
| 2 | 句式去套路 | AI 固定句式模板 | 阻断 |
| 3 | 心理描写外化 | 内部心理→外部行为 | 核心 |
| 4 | 节奏打碎 | 打破 AI 匀速推进 | 核心 |
| 5 | 对话去腔调 | 消除教科书腔 | 核心 |
| 6 | 结尾去升华 | 消除排比/哲理/情感归纳 | 阻断 |
| 7 | 意象疲劳检测 | 情绪容器反复使用 | 警告 |
| 8 | 叙述语调变化 | 打破机械稳健 | 警告 |
| 9 | 剧透式叙述/信息不对称 | 拒绝动作意图直给 | 核心 |
| 10 | 情节信息含金量熔断 | 注水率检测（"高级废话"） | 阻断 |
| 11 | 解释腔/上帝感/安排感 | 作者意志暴露 | 核心 |

**工程化保护体系**（V1.3 新增）：
- **白名单豁免**：`.deslop-whitelist` 文件中声明的句式/词汇不检测
- **分级执行**：轻(建议)/中(警告)/重(阻断)三级，按需执行
- **删除比例上限**：轻 15% / 中 25% / 重 35%，防止过度去 AI 味
- **确定性脚本兜底**：3 个脚本在 LLM 检测之后执行最终检查

**量化评分**（10 维度，各 1-10 分）：
禁用词/句式/心理/节奏/对话/结尾/意象/语调/剧透/信息含金量 → 加权总分

## 扩展技能

以下 4 个技能独立于主流水线，可在任意阶段单独调用：

### 拆文师 (`06_小说拆文师.md` V2.0)

**使用时机**：开书前（Step 0-2 之间）做市场调研

**四阶段扫榜流程**：
```
Phase 1: 确认平台和方向（番茄/起点/晋江/七猫）
Phase 2: 数据采集（排行榜数据，支持浏览器自动化或手动输入）
Phase 3: 跨样本模式识别（单本上榜=线索，多本重复=信号）
Phase 4: 可行性判断（不是按热度给结论，而是"你能写得出来吗"）
```

**四维拆文**：
- **市场定位**：目标读者/签约难度/流量密码
- **角色塑造**：角色功能位映射（把对标角色抽象为可替换的功能位）
- **剧情结构**：情节节点/节奏图谱/钩子类型
- **文风技法**：句式/感官/情绪密度/对话风格

**产出**：结构化拆文报告 → 写入项目 `对标/` 目录 → 反哺 Step 2 大纲设计。

---

### 导入师 (`07_小说导入师.md` V1.0)

**使用时机**：已有正文（已发章节/手稿），想接入流水线续写或精修

**功能**：从已有正文反向重建：
- 提取角色 → 生成设定文档
- 分析情节 → 重建大纲框架
- 追踪伏笔 → 生成伏笔追踪表
- 识别文风 → 提取文风特征

---

### 短篇写作 (`08_短篇写作技能.md` V1.0)

**使用时机**：写短篇（知乎盐言/故事会风格，8K-15K 字）

**三步流水线**（精简版）：
```
Step 1: 定位与设定 → Step 2: 正文写作 → Step 3: 去AI味审核
```

与长篇流水线的核心差异：一次性生成、03+04 合并、单次会话完成。

---

### 多视角审稿 (`09_多视角审稿师.md` V2.0)

**使用时机**：写完部分章节后，做质量检查

**三档模式**：

| 模式 | 审稿维度 | 适用场景 |
|------|---------|---------|
| `full` | 4 个 Agent 并行：编辑/AI 鉴定/读者/商业化 | 需要全面评估 |
| `lean` | 2 个 Agent 精简：编辑+AI 鉴定 | 快速检查 |
| `solo` | 主线程串行 4 视角 | 降级模式（Agent 不可用时） |

**四审流程**：
1. **编辑视角**（复用 Step 5 精修师）：质感评分 + 精修标记
2. **AI 鉴定视角**（复用 Step 6 去 AI 味）：10 Gate 检测 + AI 味评分
3. **读者视角**：追读意愿/情绪体验/理解门槛/记忆点
4. **商业化视角**：签约潜力/流量适配/长线价值

**多平台评分 rubric**：番茄/起点/知乎三平台独立评分标准。

**审稿前先跑确定性脚本预检**，阻断级 finding 必须清零后再进入四审。

## 工程化支撑体系

### 确定性脚本 (`scripts/`)

三个 Node.js 脚本提供 LLM 无法自检的确定性校验。**只报告不修改**——LLM 读取报告后自行修正。

| 脚本 | 行数 | 检测项 | 最高级别 |
|------|:---:|------|:---:|
| `check-ai-patterns.js` | 229 | 否定翻转("不是A而是B")/破折号(——)/碎句号/长段落 | **阻断** |
| `check-degeneration.js` | 201 | 逐字复读/截断/占位符泄漏/工程词泄漏/章节结构词泄漏 | **阻断** |
| `normalize-punctuation.js` | 183 | 省略号/破折号残留/markdown 分隔线/引号风格 | 建议 |

**使用方式**（Step 6 完成后）：
```bash
node scripts/check-ai-patterns.js --fail-on=blocking 正文/终稿/Ch1_终稿.md
node scripts/check-degeneration.js --fail-on=blocking 正文/终稿/Ch1_终稿.md
node scripts/normalize-punctuation.js --check 正文/终稿/Ch1_终稿.md
```

- `--check`：只检查不修改（normalize-punctuation 默认会修改，加此参数仅报告）
- `--json`：输出 JSON 格式（供 LLM 解析）
- `--fail-on=blocking`：遇到阻断级 finding 时 exit 1

### 会话钩子 (`hooks/`)

5 个 PowerShell 脚本实现会话生命周期管理：

| 钩子 | 类型 | 触发时机 | 功能 |
|------|:---:|---------|------|
| `session-start.ps1` | 信息 | 新会话启动 | 加载上下文模板、进度快照、断点续跑检查清单 |
| `session-end.ps1` | 信息 | 会话结束 | 更新上下文模板、追加项目日志 |
| `detect-story-gaps.ps1` | 巡检 | 随 session-start 执行 | 自动巡检设定缺口/大纲缺失/伏笔断线/细纲覆盖 |
| `pre-compact.ps1` | 保存 | 上下文压缩前 | 保存进度快照到 `追踪/compact-snapshots/` |
| `guard-outline-before-prose.ps1` | **阻断** | 写正文前 | 检查对应章节细纲是否存在，缺则 exit 1 阻止写作 |

**手动调用示例**：
```powershell
.\hooks\session-start.ps1 -ProjectPath "C:\Users\...\小说\书名"
.\hooks\detect-story-gaps.ps1 -ProjectPath "C:\Users\...\小说\书名"
.\hooks\guard-outline-before-prose.ps1 -ProjectPath "C:\Users\...\小说\书名" -Chapter 5
```

## 关键约束速查

### 每章硬性指标

| 指标 | 约束 | 来源 |
|------|------|------|
| 字数 | 3500-4500 中文汉字（4000 ± 500） | Step 4 |
| 场景数 | 3-5 个 | Step 4 |
| 对话比例 | ≤ 60% | Step 4 |
| 句子节奏 | 短句(≤10字) 40-60%，长句(15-40字) 做呼吸 | Step 4 |
| 省略号 | ≤ 5 处/章 | Step 5 |
| 破折号 | ≤ 8 处/章 | Step 5 |
| 感叹号 | ≤ 15 处/章 | Step 5 |

### 场景分类与字数预算

| 级别 | 字数 | 用途 |
|------|:---:|------|
| 核心场景 | 1200-1800 | 章内最重要事件 |
| 重要场景 | 800-1200 | 次重要事件 |
| 过渡场景 | 400-800 | 连接与节奏调整 |
| 点缀场景 | 200-400 | 一句话扩展或收束 |

### 钩子系统

- **章末钩子 13 式**：悬念钩/反转钩/情感钩/金句钩/动作钩/信息钩/对话钩/场景钩/时间钩/视角钩/伏笔钩/节奏钩/复合钩
- **章首钩子 7 式**：危机开场/日常异常/对话悬念/结果前置/金手指觉醒/环境铺垫/状态承接
- **约束**：连续 3 章不重复同一种钩子类型

### 8 种章节结构类型

```
A: 单线推进  B: 双线交叉  C: 多线平行  D: 倒叙回收
E: 插叙补充  F: 视角切换  G: 时间跳跃  H: 空间跳跃
```

## 跨平台部署

### 平台配置对照

| 平台 | 配置目录 | Skill 安装位置 | Hook 格式 |
|------|---------|--------------|-----------|
| **OpenCode**（默认） | `.opencode/` | `~/.config/opencode/skills/` | `.ps1` / `.sh` |
| **Claude Code** | `.claude/` | `.claude/skills/` | `.sh`（bash） |
| **Codex** | `.codex/` | `.codex/skills/` | `.sh`（bash） |
| **Trae** | `.trae/` | `.trae/skills/` | `.ps1` / `.sh` |

### 部署原则

1. 所有 `.md` skill 文件是平台无关的纯 Prompt，任何 AI IDE 都可直接加载
2. Hooks 按平台格式提供：Windows 用 `.ps1`，Linux/Mac 用 `.sh`
3. Agent 配置文件按平台约定放在对应目录，不可用则降级为主线程执行
4. Settings 文件不跨平台覆盖——部署时只写当前平台的配置

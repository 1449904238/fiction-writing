# 05 去AI味执行卡片

> **V5.3 执行卡片 | 本卡片≤2000字，完整规则见 05_去AI味精修师.md**

## 7个核心Gate（正常模式必执行）

### Gate 1：禁用词过滤

```
引用 check-ai-patterns.js 结果辅助检测。

A级（blocking，出现即删）：
├─ 情绪直给词："他感到…""他心中涌起…""他不由得…""他忍不住…"
├─ AI腔连接词："与此同时""紧接着""就这样""不知不觉中"
├─ AI腔结尾词："这大概就是…吧""三千年来第一次…""他明白了一件事…"
└─ 量化模糊词："几分""一丝""某种说不清的"

B级（限流，同章≤2次）：
└─ "愣住""沉默""缓缓""微微""淡淡""轻轻" → 超标则替换为具体动作

通过条件：A级=0，B级超标≤2个
```

### Gate 2：句式去套路

```
检测项：
├─ "不是A是B"≤2次/章：扫描"不是…是…/看似…实则…/表面…实际…/与其说
│  …不如说…"合计次数。引用 check-ai-patterns.js 结果。对话中不计入。
├─ 重复语义四类去重：形容词重复/近义词重复/含义重复/上下文重复 → 只留一个最简洁的
├─ 情绪套娃检测："情绪词+动作+情绪副词+对话"四重叠加 → 只保留最有冲击力的1-2个
├─ 修饰词清扫：无功能形容词/定语/副词/指示代词 → 删除
├─ 套路段落结构："先…然后…接着…最后…"流水账/每段以"他"开头/机械三段式
└─ 对话腔调："愤怒地吼道""冷冷地说"等情绪修饰标签 → 删除，改用中性"说"或动作替代

通过条件：无blocking标记，warning≤3处
```

### Gate 6：结尾去升华

```
6种套路结尾（出现即标记）：
├─ 类型1 排比升华："三千年来第一次…""不仅是…也是…更是…" → ❌删整段
├─ 类型2 哲理总结："或许这就是人生吧。" → ❌删整句
├─ 类型3 情感归纳："他明白了一件事…""从那天起，一切都变了。" → ❌删或改写
├─ 类型4 视角跳出："很多年后，当…" → ❌删（除非预设叙事框架）
├─ 类型5 刻意留白："至于后来怎样——那是另一个故事了。" → ⚠️建议删
└─ 类型6 对称句式："他来时天是黑的。走时天还是黑的。" → ⚠️每卷≤2次

5种正确结尾方式：
├─ 戛然而止：情节自然收束，不总结不升华
├─ 物件承载：用一个物品细节替代情感总结
├─ 身体感受：用生理感受替代心理总结
├─ 环境收束：用环境细节画上句号
└─ 动作收尾/对话收尾（≤10字）

通过条件：无❌标记，⚠️≤1处
```

### Gate 9：叙事视角剧透检测（V5.3.1 恢复为核心Gate）

```
检测"叙述者提前剧透角色意图/信息不对称失效"：

① 叙述者剧透：叙述语是否提前透露角色尚未做出决定的结果？
   ❌ "他还没开口，答案就已经注定是否定的。"
   ✅ 保留角色的犹豫过程，让读者和角色一起等待结果。

② 意图前置暴露：角色行动前，叙述语是否已点明目的？
   ❌ "为了引出真相，他故意激怒了对方。"
   ✅ "他把杯子重重磕在桌上。'你说不说？'"

③ 信息对称失效：角色本不知道的信息，叙述语是否泄露给读者？
   ❌ "他不知道，门后的人早已听到了一切。"（如果是非全知视角）
   ✅ 仅在预设全知叙事框架下允许，否则标记为blocking。

通过条件：无blocking级问题（②类允许≤1处/advisory，③类视叙事框架判定）
```

### Gate 10：信息密度均匀化检测（V5.3.1 恢复为核心Gate）

```
检测"高级废话/信息密度均匀化"：

① 高级废话检测：看似深刻但实际无信息增量的句子
   ❌ "每一秒都像一个世纪那么漫长。"（纯渲染无信息）
   ❌ "沉默，是此刻最好的回答。"（格言式废话）
   ✅ 删除或替换为具体信息（动作/物件/对话推进）

② 信息密度均匀化：全章信息分布过于均匀，无疏密变化
   检测：每500字的信息推进点是否均匀分布？
   ❌ 每500字恰好1个信息点，机械均匀
   ✅ 核心场景信息密集（3-5个/500字），过渡段允许稀疏（0-1个/500字）

通过条件：①类无blocking标记（advisory≤3处），②类信息分布有疏密变化
```

### Gate 12：转折词堆砌检测（V5.3.1 新增核心Gate）

```
检测段首转折词机械堆砌：
① 段首"然而/不过/与此同时/于是/但是/因此"≥3次/章
② 连续2+段以相同转折词开头
③ 假转折——前后语义无转折关系

通过条件：段首转折词≤2次/章，无假转折
```

### Gate 14：模糊时间跳转检测（V5.3.1 新增核心Gate）

```
检测模糊时间表达造成的"时间黑洞"：
① "不知过了多久/片刻之后/良久"≥2次/章且无时间锚点
② 时间跳转后无场景重建

通过条件：模糊时间词≤1次/章，且必须有时间锚点
（时间锚点=具体动作/光影变化/物件状态计量）
```

## 分级执行策略简表

| AI味等级 | 量化标准 | 执行Gate | 删除上限 |
|---------|---------|---------|---------|
| 轻度 | 禁用词≤5处/千字，无连续3+套路 | Gate 1+2+6 | ≤15% |
| 中度 | 禁用词6-15处/千字，或有连续3+套路 | Gate 1-6核心全过 | ≤25% |
| 重度 | 禁用词>15处/千字，或5+Gate有问题 | 核心+可选全量+重点重写 | ≤35% |

> 判定：取六项指标（禁用词密度/连续排比/心理词占比/对话标签密度/平均段落句数/重复描写密度）最高档位。允许≤1档主观下调（须书面理由）。

## 收敛终止协议

```
1. 段落级停止：同一段连续两轮去AI后无新改动 → 标记✅收敛
2. 全文级停止：上限3轮重扫；第3轮仍有≥10处改动 → 标[需复核]移交人工
3. 边际收益递减：第2轮到第3轮AI味分数提升<5分 → 自动停止，不进第3轮

[需复核]项操作：不插入正文，报告末尾单列，给出3选项：
  A.接受现状（LLM固有特性） B.人工改写（给方向） C.放弃修改
```

## 过度去AI味保护（4层面回调）

> 去AI味的目标是"像人写的"，不是"每一句都和人不一样"。

| 层面 | 过度症状 | 回调方法 |
|------|---------|---------|
| 词汇 | 为不用"说"字每句换不同动作；为不用"看"全用"瞥/瞅/瞄" | 偶尔用"说""看"是正常的；偶尔保留"一丝""几分"无妨 |
| 句式 | 刻意打破句式导致节奏混乱；每段长度都不同但无规律 | 句式变化是"调味"非"主菜"；段落变化应有趋势 |
| 对话 | 每个角色都"话里有话"；为去腔调说话全是碎片 | 偶尔直白正常；对话基本功能是传达信息；口癖1-2个足够 |
| 结尾 | 每个结尾都戛然而止读者觉得没写完 | 偶尔总结性结尾正常；关键是不要用套路句式 |

```
过度去AI味自检（收敛前必读一遍）：
├─ 觉得"这人说话怎么这么别扭" → 过度了，回调
├─ 觉得"这文笔好是好，但读着累" → 过度了，回调
└─ 觉得"自然、流畅、有味道" → 刚好，收敛

原则：自然感 > 规则遵守。规则和自然感冲突时，选自然感。
```

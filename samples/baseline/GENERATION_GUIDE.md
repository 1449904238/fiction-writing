# 基线样本生成指南（V5.4）

> 本指南说明如何使用完整流水线生成基线样本，用于后续规则迭代的回归测试。
> V5.4 更新：新增 ch04-ch09 扩展样本（悬疑/言情/转折/过场/边界下限/边界上限），新增边界样本回归标准。

## 前提条件

1. 已有完整的小说项目（00设定+01大纲+02细纲）
2. 选择以下类型的章节作为基线：
   - **职场章（ch01）**：验证对话压制感、情绪压抑爆发、信息不对称
   - **日常章（ch02）**：验证对话风格、角色辨识度、信息密度
   - **情绪章（ch03）**：验证情绪工程、用户锚点融入、质感深化
   - **悬疑推理章（ch04）**：验证线索分层释放、信息差管理、误导设计、推理留白
   - **言情情感章（ch05）**：验证情感推进节奏、潜台词对话密度、Show Don't Tell
   - **转折高潮章（ch06）**：验证反转爆发节奏、伏笔回收密度、情绪工程
   - **过场过渡章（ch07）**：验证过场推进节奏、暗线信息密度、章中钩子设计
   - **字数下限章（ch08）**：边界样本，验证字数下限/对话占比上限/标点限额边界检测
   - **字数上限章（ch09）**：边界样本，验证字数上限/场景数上限/信息密度上限检测

## 生成步骤

### Step 1: 准备输入
```
samples/baseline/chXX_类型/
├── chXX_细纲.md          — 从02复制对应章节的细纲+衔接包
└── chXX_评分.json        — 复制 scoring_template.json
```

### Step 2: 执行完整流水线（/flow_full）
```
1. 03a_扩写执行.md → 生成初稿 → 保存为 chXX_03a_初稿.md
2. 03b_质量自检.md → 生成质检报告 → 保存为 chXX_03b_质检.md
3. 04_小说正文精修师.md → 生成精修稿 → 保存为 chXX_04_精修.md
4. 05_去AI味精修师.md → 生成终稿 → 保存为 chXX_05_终稿.md
```

### Step 3: 运行确定性脚本
```bash
# 在终稿上运行所有脚本，保存报告
node scripts/check-ai-patterns.js --json 终稿.md > chXX_检测报告.json
node scripts/check-degeneration.js --json 终稿.md >> chXX_检测报告.json
node scripts/normalize-punctuation.js --check 终稿.md >> chXX_检测报告.json
node scripts/check-consistency.js --meta=设定/JSON元数据.json 终稿.md >> chXX_检测报告.json
node scripts/check-emotion-density.js --json --anchor=解气 终稿.md >> chXX_检测报告.json
node scripts/extract-used-patterns.js --json --n=3 前章文件... >> chXX_检测报告.json
```

### Step 4: 三阶段对比
```
创建 chXX_diff对比.md，逐段对比：
- 03a初稿 vs 04精修：质感提升了多少？哪些句子被改？
- 04精修 vs 05终稿：AI味减少了多少？哪些模式被修？
- 03a初稿 vs 05终稿：总变化量，是否有过度修改？
```

### Step 5: 填写评分
```
根据脚本报告和人工评估，填写 chXX_评分.json：
- 10个维度各0-10分
- 总分100分
- 等级：>85通过 / 76-85基本合格 / 60-75需修改 / <60需重写
```

### Step 6: 锁定基线
```
基线样本一旦建立，不随规则迭代修改。
后续规则变更后，用同样输入重新生成（regression/YYYYMMDD_chXX/），对比是否退化。
```

## 回归测试触发条件

每次以下变更后执行回归测试：
- skill文件修改（03a/03b/04/05/02 任何规则变更）
- 新增/删除/修改确定性脚本
- references素材库变更

> 注：以下阈值的权威定义位于 scripts/lib/prose-utils.js，此处数值仅作参考。如需调整阈值，修改 prose-utils.js 中的常量定义即可，无需逐文件修改。

| 检查项 | 方法 | 通过标准 |
|--------|------|----------|
| 总分 | 10维评分 | 回归样本总分≥基线总分-5分 |
| 字数 | check-ai-patterns.js | 3500-4500字 |
| 对话占比 | check-ai-patterns.js | ≤60% |
| Burstiness | check-ai-patterns.js | >BURSTINESS_CV_THRESHOLD（0.35） |
| 标点限额 | check-ai-patterns.js | 省略号≤5/破折号≤4/感叹号≤15 |
| AI味模式 | check-ai-patterns.js | 无新增AI味模式 |
| 跨章重复 | extract-used-patterns.js | 无新增重复 |
| 情绪密度 | check-emotion-density.js | 5-12/千字 |
| blocking数 | 所有脚本 | 回归≤基线 |
| INC要素 | 人工评估 | ≥3/13 |

## 边界样本回归标准（V5.4 新增）

> 边界样本（ch08/ch09）用于验证脚本在极限条件下的检测精度。边界样本使用 `/flow_raw` 生成，不经过深度精修，以保留刻意设计的边界条件。

### 边界样本生成流程

```
1. 03a_扩写执行.md → 生成初稿（严格按细纲字数预算）→ 保存为 chXX_03a_初稿.md
2. 03b_质量自检.md → 生成质检报告 → 保存为 chXX_03b_质检.md
3. 05_去AI味精精师.md（精简版）→ 生成终稿 → 保存为 chXX_05_终稿.md
   注：跳过04精修步骤，避免修改破坏边界条件
```

| 检查项 | 方法 | 预期结果 | 边界说明 |

| 省略号 | normalize-punctuation.js --check | =5 | 恰好达限，验证边界检测是否误判 |
| 破折号 | normalize-punctuation.js --check | =4 | 恰好达限，验证边界检测是否误判 |

### ch09 字数上限边界回归标准

| 检查项 | 方法 | 预期结果 | 边界说明 |
|--------|------|----------|---------|
| 字数 | check-ai-patterns.js | 4450-4550字 | 上限边界，验证脚本在4500附近的判断精度 |

### 边界样本回归测试特殊规则

- 边界样本的评分预期**低于标准样本**（ch08 预期 70-80 分，ch09 预期 82-92 分），这是设计意图而非质量问题
- 边界样本的 blocking 数量可能高于标准样本，需区分"边界触发的告警"和"真正的质量问题"
- 回归测试时，边界样本的通过标准为**总分≥基线-10分**（标准样本为-5分），因为边界条件的微小波动会导致较大分数变化
- 边界样本不参与跨章重复检测（extract-used-patterns.js），因为其刻意设计的密度特征会产生大量模式匹配

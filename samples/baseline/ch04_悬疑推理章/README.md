# ch04 悬疑推理章 — 基线样本

## 1. 验证目标

本章为悬疑推理题材基线样本，重点验证以下核心能力：

- 线索分层释放（P0 核心 / P1 重要 / P2 辅助 / P3 红鲱鱼 四级分级，按节奏逐步投放）
- 信息差管理（角色知 vs 读者知的层级关系，确保不提前剧透也不无故隐瞒）
- 误导设计（红鲱鱼投放 + 双重解答路径并存，读者被引导但逻辑自洽）
- 推理留白（不直给答案，关键推理节点留给读者自行补完）

对应题材/场景类型：悬疑推理 / 减速蓄势型（迷雾推理章 D 型）。

## 2. 输入依赖

- 设定档案：`demo_悬疑设定.md`（含线索分级表、红鲱鱼清单、角色信息差矩阵）
- 细纲文件：`ch04_细纲.md`（已锁定，Ch.4 急促的呼吸，含五段式内容概括与多线情节安排）
- 前置章节：Ch.1-3 留言解码链前段（第1-3条留言已释放，703 数字线索已埋设）
- 衔接包：由 02a 细纲生成阶段产出，含前序角色状态、伏笔追踪、信息差快照

## 3. 章节特征定义

本章为非边界样本，核心特征如下：

| 维度 | 目标值 | 说明 |
|------|--------|------|
| 字数目标 | 4000 字（范围 3500-4500） | 标准悬疑章字数 |
| 场景数 | 5 场景 | 听留言→声纹分析→二访公寓→追问证词→卷宗整理 |
| INC 要素数 | 5 项 | 悬疑章≥3 达标，本章 5 项以验证密度 |
| 情绪类型 | 迷雾+不安（主）/ 困惑（辅） | 减速蓄势，为 Ch.5 突破铺垫 |
| 章首钩子 | 感官冲击钩 | 第4条留言急促呼吸声 |
| 章末钩子 | 悬念留白钩 | 703 旧案关联未点破 |
| 信息密度 | 中高（声纹分析技术细节 + 红鲱鱼投放） | 推理节点密集 |
| 潜台词密度 | 中 | 证词矛盾靠读者自行比对 |
| 红鲱鱼 | C-P3-02 小陈失眠药 | 为 Ch.9 双红鲱鱼回收埋线 |

## 4. 生成流程

建议使用 `/flow_full` 完整流程（悬疑章对推理留白与信息差精度要求高，不宜走快速通道）。

| 步骤 | 操作 | 输出文件命名 |
|------|------|--------------|
| 03a 扩写 | 基于 ch04_细纲.md 生成骨架+血肉 | `ch04_03a_初稿.md` |

| 04 精修 | Show Don't Tell / 潜台词 / 推理留白强化 | `ch04_04_精修稿.md` |
| 05 去AI味 | 11 Gate 全量执行 | `ch04_05_终稿.md` |

注意：悬疑章为推理留白单一责任人（04 唯一执行点），心理直给禁止，解释腔禁止。

- 本章不验证：动作场景描写能力（悬疑章无打斗）、情感推进节奏（属 ch05 验证范围）、爽点爆发（属 ch06/ch09 验证范围）。

- 本章使用独立悬疑设定档案（demo_悬疑设定.md），与 ch06/ch07/ch08/ch09 的 demo_设定.md 体系不同，不参与跨设定伏笔一致性比对。

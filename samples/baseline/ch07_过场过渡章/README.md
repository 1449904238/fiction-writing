# ch07 过场过渡章 — 基线样本

## 1. 验证目标

本章为过场过渡题材基线样本，重点验证以下核心能力：

- 过场推进节奏（承上启下 + 关键信息埋点，减速过渡不拖沓）
- 暗线信息密度（隐性信息为后续章节铺路，FS-004 埋设、FS-001 推进）
- 对话极简留白（老王一句"年轻人，手别太勤快"式的精炼提示，不解释不展开）
- 章中钩子设计（过场章无显性爽点，靠悬念钩子维持叙事驱动力）

对应题材/场景类型：都市职场+轻玄幻 / 过场推进型（E 型）。

## 2. 输入依赖

- 设定档案：`demo_设定.md`（都市职场+轻玄幻，含赵总暗线档案、老王能力感知机制）
- 细纲文件：`ch07_细纲.md`（已锁定，Ch.4 暗流，含信息传递表 + 伏笔埋设清单 + 过渡节奏曲线）
- 前置章节：Ch.3 对峙高潮（赵总被揭穿但靠人脉压下），为本章缓冲铺垫
- 衔接包：由 02a 细纲生成阶段产出，含角色状态（林默反杀成功后危机感升级）、伏笔追踪

## 3. 章节特征定义

本章为非边界样本（过场章字数偏低但非下限边界检测样本），核心特征如下：

| 维度 | 目标值 | 说明 |
|------|--------|------|
| 字数目标 | 3700 字（范围 3500-3800，过场章偏低区间） | 过场章允许偏低 |
| 场景数 | 5 场景 | 办公室日常→观察赵总→茶水间老王→工位能力溢出→下班暗流 |
| INC 要素数 | 4 项 | 过场章≥2 达标，本章 4 项验证过渡驱动力 |
| 情绪类型 | 不安（主）+ 悬念（辅） | 减速过渡，Ch.3高潮后缓冲 |
| 章首钩子 | 环境意象钩 | 赵总事件第二天，办公室一切照常 |
| 章末钩子 | 暗流留白钩 | 赵总"别紧张"是安抚还是威胁 |
| 信息密度 | 中低（隐性信息为主） | 暗线推进+悬念铺垫，无信息爆发 |
| 对话密度 | 极简留白 | 老王一句话、苏晚一句提醒、赵总一条消息 |
| 爽点 | 无显性爽点 | 功能是信息传递+暗线推进+悬念铺垫 |
| 伏笔推进 | FS-001 推进，FS-004 埋设 | 老王暗示感知、赵总找技术顾问 |

## 4. 生成流程

建议使用 `/flow_fast` 快速流程（过场章对深度打磨需求低于高潮章，03b 质检后可走 04b 深度打磨或 05 精简版）。

| 步骤 | 操作 | 输出文件命名 |
|------|------|--------------|
| 03a 扩写 | 基于 ch07_细纲.md 生成骨架+血肉 | `ch07_03a_初稿.md` |
| 03b 质检 | P1/P2 全量 + 5维评分 + 流程选择（预期≥80走快速通道） | `ch07_03b_质检报告.md` |
| 04b 深度打磨 | /flow_fast 合并精修+去AI味（7 Gate） | `ch07_04b_精修稿.md` |
| 05 精简版 | /flow_fast 可选，抽样复核 | `ch07_05_终稿.md` |
| 脚本检测 | 确定性脚本批量运行 | `ch07_检测报告.md` |
| 评分 | 10维评分 + diff 对比 | `ch07_评分.json` + `ch07_diff.md` |

注意：过场章主要风险是低密度退化（水句/填充句），check-degeneration.js 是核心检测。INC 要素过场章放宽至≥2项，本章4项达标。

## 5. 关键检测项与预期

| 检测项 | 脚本 | 预期结果 | 说明 |
|--------|------|----------|------|
| 低密度退化检测 | `check-degeneration.js` | PASS | 验证无水句/填充句堆砌，过场章核心风险 |
| 缓冲节奏 7-2-1 篇幅配比 | `check-rhythm.js` | 过场场景占比合理 | 验证减速过渡节奏，核心场景不过度膨胀 |
| 伏笔一致性 + 信息传递连贯性 | `check-consistency.js` | PASS | 验证 FS-001/FS-004 推进连贯，与 Ch.3 衔接 |
| 字数下限 | `check-ai-patterns.js` | ≥3500 | 过场章易字数不足，验证下限检测灵敏度 |
| 过渡章情绪密度下限 | `check-emotion-density.js` | ≥3/千字 | 验证过渡章情绪密度不致过低 |
| 水句密度 | `check-ai-patterns.js` | PASS | 验证无水句稀释信息 |
| 标点限额 | `check-ai-patterns.js` | 省略号≤5 / 破折号≤4 / 感叹号≤15 | 过场章标点风险较低 |
| 对话占比 | `check-rhythm.js` | ≤60% | 过场章对话极简，预期远低于上限 |
| INC 13 要素 | 人工核验 | ≥2 项达标（过场章放宽，本章 4 项） | 验证过渡章保有最低叙事驱动力 |

## 6. 预期评分区间

10维评分预期区间：**75-85 分**。

| 维度 | 预期区间 | 说明 |
|------|----------|------|
| 情节逻辑 | 7-8 | 过场章逻辑要求低于高潮章 |
| 信息释放 | 7-8 | 隐性信息为主，密度中低 |
| 人物塑造 | 6-7 | 过场章人物服务于信息传递 |
| 节奏控制 | 8-9 | 缓冲节奏是核心验证点 |
| 潜台词 | 7-8 | 老王一句话潜台词 |
| Show Don't Tell | 7-8 | 暗线信息需 Show |
| AI味 | 7-8 | 低密度退化是主要风险 |
| INC 要素 | 7-8 | 4项达标（过场章≥2） |
| 衔接连贯 | 8-9 | 与 Ch.3/Ch.6 衔接是核心 |
| 情绪工程 | 7-8 | 不安+悬念锚点交付，密度偏低 |

预期等级：**基本合格 ~ 通过**（过场章综合评分预期偏低，核心验证缓冲节奏与暗线信息密度，达到 80+ 需控制水句）。

## 8. 已知限制

- 本章不验证：反转爆发（属 ch06 验证范围）、情感决堤（属 ch05 验证范围）、字数边界检测精度（属 ch08/ch09 验证范围，本章字数偏低但非刻意边界设计）。
- 已知局限：过场章的"缓冲节奏"评判存在主观性，脚本只能检测水句密度与篇幅配比，减速是否自然需人工复核；暗线信息密度过低易被判为"水文"，密度过高又破坏过渡感，平衡点依赖人工把控。
- 与其他样本关联：本章 FS-004（赵总找外部技术顾问）为 Ch.8 技术顾问到位埋线，Ch.8 须验证 FS-004 推进；FS-001 推进（老王感知能力）为 Ch.6 老王身份揭露提供基础；赵总"别紧张"消息为 Ch.8/Ch.9 赵总围剿铺垫。
- 本章使用 demo_设定.md（都市职场+轻玄幻），与 ch06/ch08/ch09 同体系，参与跨章伏笔一致性比对。
- 本章字数 3700 偏低，与 ch08（字数下限边界样本 3500）字数接近但定位不同：本章验证过场节奏，ch08 验证字数下限检测精度。

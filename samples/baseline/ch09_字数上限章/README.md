# ch09 字数上限章 — 基线样本

## 1. 验证目标

本章为字数上限边界样本，刻意逼近多项阈值上限，重点验证以下核心能力：

- 字数上限边界检测（4500±50 字，验证脚本对字数上限的检测精度与触发灵敏度）
- 场景数上限=5（验证最多场景数下的节奏控制与信息密度平衡）
- 信息密度上限检测（高信息密度下避免长文稀释关键信息，验证信息密度方差下限）
- 爽点密度上限检测（高潮章爽点密集，验证爽点密度上限下的节奏维持）
- INC 要素≥4（高潮章建议≥4，本章 6 项验证高潮章要素密度上限）

对应题材/场景类型：都市职场+轻玄幻 / 高潮决战型（B 型，全书最高潮章）。

## 2. 输入依赖

- 设定档案：`demo_设定.md`（都市职场+轻玄幻，含能力等级体系、银纹代价机制、技术顾问周衡档案）
- 细纲文件：`ch09_细纲.md`（已锁定，Ch.9 代码战争，含边界样本说明 + 高密度结构 + 多场景压缩设计）
- 前置章节：Ch.8 师徒裂痕（林默拒绝停用能力、技术顾问到位、银纹增至3条），为本章决战铺垫
- 衔接包：由 02a 细纲生成阶段产出，含角色状态（林默2级能力待激活）、伏笔追踪、银纹状态

## 3. 边界条件定义

本章为边界样本，刻意设计以下边界条件（多维度同时逼近阈值上限）：

| 边界维度 | 边界条件 | 阈值方向 | 说明 |
|----------|----------|----------|------|
| 字数 | 4500±50（4450-4550） | 上限边界 | 恰好达 4500 上限，验证上限检测精度 |
| 场景数 | 5（上限） | 上限边界 | 最多场景数，验证节奏控制与信息密度平衡 |
| 信息密度 | 高 | 上限边界 | 高密度下验证信息不稀释 |
| 爽点密度 | 高 | 上限边界 | 高潮章爽点密集，验证节奏维持 |
| INC 要素 | 6 项（≥4 达标，高潮章建议） | 上限边界 | 高潮章要素密集区 |
| burstiness CV | >0.35 | 下限边界 | 验证句长方差下限检测（长文易方差趋近0） |
| 7-2-1 篇幅配比 | 核心场景≤70% | 上限边界 | 验证核心场景不过度膨胀 |
| 信息密度方差 | >下限 | 下限边界 | 避免长文信息均匀稀释导致方差趋近0 |

边界检测重点观察：
- 字数 4500（恰好达上限）vs 4501（超限）的判定差异
- burstiness CV 0.35（恰好达下限）vs 0.34（不足）的判定差异
- 信息密度方差下限（避免长文信息均匀稀释导致方差趋近0）
- 7-2-1 篇幅配比上限（核心场景不超过 70%）

## 4. 生成流程

建议使用 `/flow_raw` 生肉流程（边界样本不追求字数最大化，而追求上限边界条件覆盖度与高信息密度的平衡，03a 扩写后经 03b 质检即可，05 精简版抽样复核）。

| 步骤 | 操作 | 输出文件命名 |
|------|------|--------------|
| 03a 扩写 | 基于 ch09_细纲.md 生成骨架+血肉（高信息密度结构 + 多场景压缩设计） | `ch09_03a_初稿.md` |
| 03b 质检 | P1/P2 全量 + 5维评分 + 流程选择（必执行，边界验证核心） | `ch09_03b_质检报告.md` |
| 05 精简版 | /flow_raw 抽样复核（不深度精修，保留边界条件原始状态） | `ch09_05_终稿.md` |
| 脚本检测 | 确定性脚本批量运行（重点：上限边界判定） | `ch09_检测报告.md` |
| 评分 | 10维评分 + diff 对比 | `ch09_评分.json` + `ch09_diff.md` |

注意：本样本刻意逼近字数与节奏上限，用于验证确定性脚本在上限侧的检测灵敏度与误报率。精修会破坏边界条件，故不走 04 深度精修。

## 8. 已知限制

- 本章不验证：文学质量最大化（边界样本不追求文学质量，追求上限边界覆盖度）、推理留白（属 ch04 验证范围）、情感决堤（属 ch05 验证范围）、过场节奏（属 ch07 验证范围）。
- 已知局限：本样本刻意逼近字数与节奏上限，高密度下信息密度方差易趋近0（信息均匀稀释），需人工复核信息密度是否真正"高"而非"均匀铺开"；多场景压缩设计易导致跨场景 n-gram 重复，需重点检测；长文节奏控制难度高，burstiness CV 易在边界波动。
- 与其他样本关联：本章与 ch08（字数下限边界样本）互为边界对照，ch08 验证下限侧、ch09 验证上限侧；本章回收 FS-004（技术顾问周衡身份）+ FS-005（技术顾问感知力），须验证伏笔回收闭环；本章推进 FS-003（老王"最后一行注释"），老王关键时刻出手；本章银纹从3条暴涨至整只右手，为后续卷代价机制提供基础。
- 本章使用 demo_设定.md（都市职场+轻玄幻），与 ch06/ch07/ch08 同体系，参与跨章伏笔一致性比对。
- 本章字数 4500 与 ch06（高潮章 4500）字数相同但定位不同：本章验证字数上限检测精度，ch06 验证反转质量。
- 本章为全书最高潮章，所有线索汇聚爆发，是 INC 要素与爽点密度最高的样本，用于验证高密度条件下的脚本检测精度与性能。

## 样本用途

1. **验证规则有效性**：对比03a初稿→04精修→05终稿的质量提升，验证规则是否有效
2. **AI对标参考**：03a扩写时参考样本中的写法，05去AI味时参考样本中的改法
3. **回归测试基线**：规则迭代后，用同样输入重新生成，对比是否产生退化

## 样本选择标准

- 选择涵盖多种场景类型的章节：
  - **战斗章**：验证动作描写、节奏控制、战力一致性
  - **职场章**：验证对话压制感、情绪压抑爆发、信息不对称（ch01原标记为"战斗章"但实际内容为都市职场裁员场景，V5.3.1已更正）
  - **日常章**：验证对话风格、角色辨识度、信息密度
  - **情绪章**：验证情绪工程、用户锚点融入、质感深化
  - **悬疑推理章**：验证线索分层释放/信息差管理/误导设计，4000-4500字
  - **言情情感章**：验证情感拉扯/潜台词对话/关系推进，3500-4000字
  - **转折高潮章**：验证反转设计/情绪爆发/节奏加速，4500字（上限）
  - **过场过渡章**：验证信息传递/节奏缓冲/伏笔埋设，3500字（下限）
  - **字数下限章**：验证标点限额边界/对话占比60%边界，3500字
  - **字数上限章**：验证信息密度下限/节奏控制上限，4500字
- 选择有典型AI味问题的初稿（验证05去AI味效果）
- 样本一旦建立，不随规则迭代修改（作为历史基线）

### 扩展样本生成说明

- ch04-ch07 为**题材专项样本**：验证不同题材下的规则适配（悬疑→悬念保护 / 言情→情感碰撞 / 转折→节奏加速 / 过场→节奏缓冲）

## 回归测试流程

### 触发条件
每次以下变更后执行回归测试：
- skill 文件修改（03a/03b/04/05/02 任何规则变更）
- 新增/删除/修改确定性脚本
- references 素材库变更

### 执行步骤

1. **使用 baseline/ 中的同样 3 章细纲作为输入**
2. **完整执行 03a→03b→04→05 流程**（或对应的新流程）
3. **保存产出到 `regression/YYYYMMDD_chXX/`**
4. **运行确定性脚本检测**，保存报告
5. **对比基线，检查以下项**：

> 注：以下阈值的权威定义位于 scripts/lib/prose-utils.js，此处数值仅作参考。如需调整阈值，修改 prose-utils.js 中的常量定义即可，无需逐文件修改。

| 检查项 | 方法 | 通过标准 |
|--------|------|----------|
| 字数范围 | check-ai-patterns.js | 3500-4500 字 |
| 对话占比 | check-ai-patterns.js | ≤60% |
| Burstiness | check-ai-patterns.js | >BURSTINESS_CV_THRESHOLD（0.35） |
| 标点限额 | check-ai-patterns.js | 省略号≤5/破折号≤4/感叹号≤15 |
| AI味模式 | check-ai-patterns.js | 无新增AI味模式 |
| 跨章重复 | extract-used-patterns.js | 无新增跨章重复 |
| 角色一致性 | check-consistency.js | 0矛盾 |
| 节奏波形 | check-rhythm.js | 无新增塌陷 |
| 用户审美 | 人工判断 | ≥基线评分 |
| 信息密度 | 人工+脚本 | 无下降 |

6. **如果任一项不通过，标记为"回归退化"**，需要修复后重新测试

## 10维评分量规

每章产出后由三个角色分别打分（1-5分）：

| 维度 | 评分方 | 说明 |
|------|--------|------|
| 目标清晰 | quality-checker Agent | 本章核心冲突是否明确 |
| 受众具体 | 用户 | 是否符合目标读者画像 |
| 具体性 | 脚本+Agent | 细节是否足够具体 |
| 洞见密度 | quality-checker Agent | 每场景信息增量 |
| 结构聚焦 | 脚本检测 | 是否有冗余场景 |
| 声音可信 | 用户 | 角色对话是否有辨识度 |
| 节奏控制 | burstiness脚本 | 句长变化是否自然 |
| 情绪到达 | 用户 | 是否被打动 |
| AI味 | 脚本+Agent | AI痕迹是否明显 |
| 记忆度 | 用户 | 是否有令人印象深刻的场景 |

### 回归测试

```bash
# 对比模式（默认）：检查当前脚本输出是否与基线一致
node scripts/regression-baseline.js

# 更新模式：脚本修改后更新基线
node scripts/regression-baseline.js --update

# JSON输出
node scripts/regression-baseline.js --json
```

退出码：0=无回归 / 1=有差异需审查 / 2=运行错误

### 生成首批基线样本的步骤

1. 完成项目初始化（00.5）和设定架构（00）
2. 生成前3章细纲（02），选择不同场景类型
3. 对每章执行 03a→03b→04→05 完整流程
4. 每阶段产出保存到 `baseline/chXX_场景类型/`
5. 运行所有确定性脚本，保存检测报告
6. 用户填写10维评分量规
7. 生成三阶段 diff 对比文件

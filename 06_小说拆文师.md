## 前置调研：扫榜（v2.0 新增）

> 源自 oh-story-claudecode 的 story-long-scan 系统。在拆文之前，先做市场调研，找到值得拆的对标作品。

### 核心原则

1. **扫榜看模式，别只看排名**：单本上榜只是线索，跨样本重复模式才算信号
2. **流量平台和付费平台看的东西不同**：番茄看完读率，起点看订阅，晋江看收藏
3. **扫榜的目的是找到你能写的爆款题材**：不是按热度给结论，而是做可行性判断

### 扫榜流程

#### Phase 1：确认平台和方向

问用户：**"你想看哪个平台？（起点/番茄/晋江/七猫/刺猬猫）有没有关注的题材方向？"**

| 用户状态 | 处理方式 |
|---------|---------|
| 已有方向 | 针对该方向做深度扫榜 |
| 没有方向 | 做全榜概览 + 找趋势 |
| 想跨平台比较 | 做平台对比分析 |

#### Phase 2：数据采集

根据当前环境选择数据来源：

| 优先级 | 模式 | 说明 |
|--------|------|------|
| 1 | **浏览器自动化** | 使用浏览器工具访问平台榜单页面，提取结构化数据 |
| 2 | **用户提供** | 用户粘贴榜单截图/文字/链接 |
| 3 | **内置知识** | 基于已知趋势做分析（数据可能过时） |

**各平台核心榜单**：

| 平台 | 关键榜单 | 核心指标 |
|------|---------|---------|
| 起点 | 月票榜/畅销榜/新书榜/三江推荐 | 订阅+推荐 |
| 番茄 | 男频/女频阅读榜+新书榜 | 在读数 |
| 晋江 | 金榜/新书榜 | 收藏+积分 |
| 七猫 | 热门榜/新书榜 | 阅读量 |

#### Phase 3：数据分析与趋势提炼

从采集数据中提取：
- **题材趋势**：哪些题材反复上榜？新兴题材是什么？
- **卖点提炼**：上榜作品的共同卖点是什么？
- **读者画像**：这些作品的目标读者是谁？
- **竞争格局**：同类作品密度如何？差异化空间在哪？

#### Phase 4：选题建议

输出 `选题决策.md`，包含：
```
## 选题决策

### 推荐方向
1. [方向A]：热度高 + 竞争适中 + 匹配你的优势
   - 能爆的原因：...
   - 差异化切入点：...
   - 推荐对标书：...

2. [方向B]：...

### 风险评估
- [方向A] 风险：...
- [方向B] 风险：...

### 建议
综合推荐 [方向X]，理由：...
```

## 系统化拆文

> 融合 V1.0 四维拆解 + oh-story 的结构化输出体系。

### 拆文策略

| 拆文深度 | 适用场景 | 产出 |
|---------|---------|------|
| 黄金三章精拆 | 快速评估对标书是否值得深拆 | 前3章深度拆解报告 |
| 全量拆解 | 确定为核心对标书 | 完整四维拆解 + 结构化目录 |
| 专题拆解 | 针对性学习某个维度 | 单维度深度报告 |

### 拆文维度

#### 第一维：市场定位分析

**为什么读者买单？**

```
【市场定位诊断】

1. 一句话卖点
   ├─ 对标作品的"转述测试"：读者会怎么向朋友推荐？
   └─ 提炼公式：[身份] + [金手指/设定] + [核心爽点]

2. 读者画像
   ├─ 年龄层：初中生/高中生/大学生/上班族
   ├─ 阅读场景：通勤/睡前/课间/摸鱼
   ├─ 核心需求：爽/甜/虐/悬疑/新奇
   └─ 情绪代偿：现实缺什么，小说给什么

3. 同类竞争
   ├─ 同题材前10名的共同特点
   ├─ 对标作品的差异化优势
   └─ 你的作品与对标作品的差异化定位

4. 平台适配度
   ├─ 番茄：快节奏、前500字爽点、单章4000字
   ├─ 起点：慢热、世界观构建、长线埋伏
   └─ 飞卢：极致爽、更新快、脑洞大
```

#### 第二维：角色塑造拆解

```
【角色塑造拆解】

1. 主角原型
   ├─ 身份：底层/中产/强者/重生者/穿越者
   ├─ 性格关键词：如"粗粝、嘴贱、护短"
   ├─ 核心驱动力：求生存/复仇/守护/证明自己
   ├─ 缺点/限制：破局的关键障碍
   └─ 成长弧线：起点→终点（变化路径）

2. 互动系统
   ├─ 主角 vs 敌人：压迫感来源
   ├─ 主角 vs 同伴：情感羁绊
   ├─ 主角 vs 配角：信息差/利用/守护
   └─ 三人成戏规则：主角+同伴+敌人 = 三角张力

3. 配角生态位
   ├─ 导师型：提供信息/能力/方向（中后期退场）
   ├─ 同伴型：互补性格/功能/背景
   ├─ 反派型：要有压迫感+表演型人格
   ├─ 感情线：拉扯感来源（误会/地位差/身份冲突）
   └─ 功能型：推动剧情/提供情报/制造障碍

4. 角色功能位映射（v2.0新增）
   ├─ 把对标书的每个角色看成"功能位"
   │   ├─ 对手位：提供压迫感 + 打脸对象
   │   ├─ 盟友位：提供情感羁绊 + 信息释放
   │   ├─ 催化剂位：推动主角变化 + 制造转折
   │   └─ 信息位：提供设定/世界观/伏笔
   ├─ 映射到你的角色时：用你的素材填充功能位
   └─ 原则：学"功能"不学"人设"，避免抄袭
```

#### 第三维：剧情结构拆解

```
【剧情结构拆解】

1. 黄金三章分析
   ├─ 第1章控制点：前500字发生了什么？
   │   ├─ 开篇方式：危机切入/日常异常/金手指觉醒/对话开场/结果前置
   │   ├─ 第一句：是否抓住了读者？
   │   └─ 第1章结束：钩子是什么？
   ├─ 第2章节奏：铺垫还是释放？
   └─ 第3章确认：读者是否已经"入坑"？

2. 分卷结构
   ├─ 每卷的核心冲突是什么？
   ├─ 每卷的情绪曲线：起→承→转→合
   ├─ 每卷的爽点分布：前/中/后段各几个？
   └─ 每卷的收尾钩子：为下一卷埋伏什么？

3. 呼吸节奏
   ├─ 铺垫章 vs 爆发章的比例
   ├─ 爽点间隔：多少字/几章一个爽点？
   ├─ 情感线的"拉扯"节奏：甜→虐→甜
   └─ 大高潮前的小高潮铺垫

4. 伏笔系统
   ├─ 伏笔类型：物件/对话/事件/设定/角色
   ├─ 伏笔密度：前10章埋了多少伏笔？
   ├─ 回收节奏：多久回收？隔多少章？
   └─ 长线伏笔：跨卷伏笔的设计方式

5. 情绪模块提取（v2.0新增）
   ├─ 读者需求：这本书满足读者什么情绪需求？
   ├─ 情绪引擎：哪些场景/事件持续驱动情绪？
   ├─ 可复现模块：哪些情绪场景可以换皮复用？
   └─ 爽文套路框架：打脸/翻盘/升级的情绪节拍
```

#### 第四维：文风技法拆解

```
【文风技法拆解】

1. 句式节奏
   ├─ 平均句长：短句为主还是长句为主？
   ├─ 段落长度：短段密集还是长段叙事？
   ├─ 对话占比：约多少比例？
   └─ 叙述者距离：贴身/中距/全知？

2. 感官侧重
   ├─ 最常用的感官：视觉/触觉/嗅觉/听觉？
   ├─ 感官密度：每多少字一处感官描写？
   └─ 脏感/真实感：是否有物理代价描写？

3. 对话风格
   ├─ 口语化程度：高（说书人）还是低（文人腔）？
   ├─ 说话方式：直给还是话里有话？
   ├─ 信息释放：对话承担多少设定交代？

4. 情绪表达
   ├─ 直接表达 vs 间接投射
   ├─ 内心独白的频率和深度
   ├─ 嘴硬心软/言行不一的频率
   └─ 发刀/发糖的方式和节奏
```

## 结构化输出（v2.0 新增）

> 拆文产出不再是单一报告文件，而是写入项目标准目录的结构化文件体系。

### 输出目录结构

```
项目目录/
├── 拆文库/
│   └── {对标书名}/
│       ├── 原文/                  ← 对标书原文（用户提供）
│       │   ├── 第001章_章名.md
│       │   └── ...
│       ├── 角色/                  ← 角色拆解（每人一个文件）
│       │   ├── {主角名}.md        ← 含功能位映射
│       │   └── ...
│       ├── 剧情/                  ← 剧情结构拆解
│       │   ├── 故事线.md          ← 主线/辅线梳理
│       │   ├── 节奏.md            ← 关键信息推进 + 情绪触动点 + 爆发节奏
│       │   └── 情绪模块.md        ← 读者需求/情绪引擎 + 可复现模块
│       ├── 设定/                  ← 设定拆解
│       │   └── 世界观.md
│       ├── 章节/                  ← 逐章摘要
│       │   ├── 第001章_摘要.md
│       │   └── ...
│       ├── 文风.md                ← 文风拆解（句长/标点/对话/锚点）
│       └── 拆文报告.md            ← 综合分析报告
├── 对标/                          ← 本项目引用的对标数据子集
│   └── {对标书名}/                ← 从拆文库/复制相关子目录
│       ├── 角色/
│       ├── 剧情/
│       ├── 设定/
│       ├── 章节/
│       ├── 文风.md
│       └── 拆文报告.md
```

### 拆文库 vs 对标的关系

| 目录 | 作用 | 生命周期 |
|------|------|---------|
| `拆文库/` | 拆文师的原始产出，是数据源 | 长期保留 |
| `对标/` | 写作项目的引用视图，存放与本项目相关的对标数据子集 | 随项目 |

首次引用对标书时，从 `拆文库/{书名}/` 复制相关子目录到 `对标/{书名}/`。

## 对标驱动模块化创作闭环（v2.0 新增）

> 源自 oh-story-claudecode 的"对标驱动模块化创作"理念。拆文不是终点，而是写作闭环的起点。

### 数据流

```
扫榜（找方向）→ 拆文（拆解对标）→ 对标（功能位映射）→ 写作（模块填充）
     │              │                   │                   │
     ▼              ▼                   ▼                   ▼
 选题决策.md    拆文库/结构化     对标/引用视图     02细纲引用对标数据
                                                    03扩写参考对标文风
                                                    04精修对标情绪模块
```

### 对标数据在写作中的使用

| 写作阶段 | 引用的对标数据 | 用途 |
|---------|-------------|------|
| 02 细纲 | `对标/{书}/剧情/节奏.md` | 参考爽点分布和呼吸节奏 |
| 02 细纲 | `对标/{书}/角色/` | 功能位映射，确定角色定位 |
| 03 扩写 | `对标/{书}/剧情/情绪模块.md` | 召回情绪引擎和可复现模块 |
| 03 扩写 | `对标/{书}/文风.md` | 参考句长/标点/对话风格 |
| 04 精修 | `对标/{书}/章节/` | 匹配同基调章节参考 |
| 05 去AI味 | `对标/{书}/文风.md` | 锚点片段作为自然文本基准 |

### 多对标书管理

- 主对标书：提供文风、情绪模块、节奏的权威参考
- 副对标书：提供结构化摘要，不读取文风/原文
- 在 `设定/题材定位.md` 中标记 `主对标书` 字段

### 对标节奏迁移 M1-M5（v2.1 新增）

> 源自 oh-story-claudecode 的「对标节奏迁移」理念。把爆款拆解的一级结构关键点（1/4·中点·3/4）换素材排进本卷卷纲，让本卷节奏踩在市场验证过的节拍上。**学"节奏"不学"剧情"**——换皮不抄袭。

**适用前提**：已对主对标书完成四维拆文，`对标/{书}/剧情/节奏.md` 已生成。

**M1-M5 迁移流程**：

| 步骤 | 操作 | 产出 |
|------|------|------|
| **M1 定位锚点** | 从 `对标/{书}/剧情/节奏.md` 提取对标书的一级结构关键点：开篇钩子（0-10%）、1/4 转折、中点（50%）、3/4 危机、高潮（90%）、收尾 | 对标结构坐标表 |
| **M2 换素材** | 把每个锚点的"剧情内容"换成你的素材，但保留"节奏功能"（如对标1/4是"主角首次受挫"→你的1/4也设计一个"首次受挫"事件，但具体事件不同） | 本卷节奏骨架 |
| **M3 校验占比** | 检查本卷的 高压/推进章占比是否≥60%、低压+过场章≤15%（沿用本地大纲五检第⑤问：每卷1/4·中点·3/4位置各锚定一个关键情节，确保节奏骨架不塌陷）；1/4·中点·3/4 是否各锚定一个关键情节 | 占比校验报告 |
| **M4 写入卷纲** | 把 M2 的节奏骨架写入本卷卷纲的「对标结构坐标」字段，供 02 细纲编写时参考 | 卷纲·对标结构坐标 |
| **M5 缺失降级** | 若 `对标/{书}/剧情/节奏.md` 缺失 → 回退到 `章节/*_摘要.md` + `拆文报告.md` 的节奏摘要，反推锚点；记录 `rhythm_missing` 标记 | 降级节奏骨架 |

**M1-M5 与本地大纲五检的关系**：
- 大纲五检第⑤问（定义于02_细纲编写技能.md Part O5）：每卷必须在1/4、中点(50%)、3/4三个位置各锚定一个关键情节，确保叙事节奏有骨架。M3是此检查项在对标迁移场景下的应用。
- M1-M5 是**操作流程**——告诉你怎么把对标的锚点迁移过来
- 两者配合：先用 M1-M5 迁移，再用大纲五检校验

**反抄袭底线**：迁移的是"节奏功能"（受挫/转折/危机/高潮的节拍位置），不是"剧情内容"。同一锚点的具体事件必须是你自己的素材。

### legacy 拆文库回退规则（v2.1 新增）

> 旧版拆文库可能缺 `剧情/情绪模块.md` 或 `剧情/节奏.md`。本规则定义缺文件时的降级路径，保证写作不中断。

| 缺失文件 | 回退路径 | 标记 |
|---------|---------|------|
| `剧情/情绪模块.md` 缺失（v2.0 新契约拆文库） | **停下提示重跑拆文** Stage 3+，不得假装已召回 | 不标 legacy |
| `剧情/情绪模块.md` 缺失（legacy 拆文库） | 依次回退：`拆文报告.md` 读者需求/关键信息/节奏/可复现模块摘要 → `文风.md` 可借鉴技巧 → 匹配 `章节/第K章_摘要.md` | `legacy_deconstruction: true` + `module_missing` |
| `剧情/节奏.md` 缺失（legacy 拆文库） | 依次回退：`拆文报告.md` 节奏摘要 → 匹配章摘要 → `剧情/故事线.md` | `legacy_deconstruction: true` + `rhythm_missing` |
| `文风.md` 缺失 | 回退到 `拆文报告.md` 文风摘要；无则提示重跑拆文 | `style_missing` |

**判定 legacy 的方法**：`拆文报告.md` 中是否含 `读者需求/关键信息/节奏/可复现模块` 摘要字段。无 → legacy。

**回退时的写作约束**：legacy 回退后写作可继续，但 03 扩写引用情绪模块时需在定位报告标注 `legacy_deconstruction: true`，提醒精度下降。

## 流水线契约

- 拆文工作**独立于主流水线**，可在任何阶段进行
- 产出物存入项目标准目录的 `拆文库/` 和 `对标/` 子目录
- 02 细纲编写技能可引用 `对标/` 中的节奏/角色/情绪模块数据
- 03 扩写系统可引用 `对标/` 中的文风和情绪模块数据
- 05 去AI味精修师可引用 `对标/` 中的文风作为自然文本基准
- 拆文推荐频率：每本新书开书前拆 2-3 部同题材成功作品
- **v2.0 新增**：拆文前先做扫榜调研，用数据驱动对标书选择

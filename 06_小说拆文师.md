## 前置调研：扫榜（v2.0 新增）

> 源自 oh-story-claudecode 的 story-long-scan 系统。在拆文之前，先做市场调研，找到值得拆的对标作品。

### 核心原则

1. **扫榜看模式，别只看排名**：单本上榜只是线索，跨样本重复模式才算信号
2. **流量平台和付费平台看的东西不同**：番茄看完读率，起点看订阅，晋江看收藏
3. **扫榜的目的是找到你能写的爆款题材**：不是按热度给结论，而是做可行性判断

### 扫榜流程

#### Phase 1：确认平台和方向

问用户：**"你想看哪个平台？（起点/番茄/晋江/七猫/刺猬猫）有没有关注的题材方向？"**

| 用户状态 | 处理方式 |
|---------|---------|
| 已有方向 | 针对该方向做深度扫榜 |
| 没有方向 | 做全榜概览 + 找趋势 |
| 想跨平台比较 | 做平台对比分析 |

#### Phase 2：数据采集

**数据获取三级降级链（V5.1更新）**：

| 优先级 | 模式 | 说明 |
|--------|------|------|
| 1 | **手动输入（优先）** | 用户提供榜单截图/复制文本/口头描述，提供标准输入模板 |
| 2 | **浏览器MCP（半自动）** | 用户确认后，使用 integrated_browser MCP 工具半自动抓取公开榜单 |
| 3 | **WebSearch（兜底）** | 搜索"番茄小说热榜""起点月票榜"等关键词，从结果中提取信息 |

**各平台核心榜单**：

| 平台 | 关键榜单 | 核心指标 |
|------|---------|---------|
| 起点 | 月票榜/畅销榜/新书榜/三江推荐 | 订阅+推荐 |
| 番茄 | 男频/女频阅读榜+新书榜 | 在读数 |
| 晋江 | 金榜/新书榜 | 收藏+积分 |
| 七猫 | 热门榜/新书榜 | 阅读量 |

### 数据获取三级降级链（V5.1更新 — 替代原"浏览器自动化优先"策略）

> V5.1 起，扫榜数据获取从"浏览器自动化优先"改为"手动输入优先+浏览器MCP半自动+WebSearch兜底"三级降级链，降低对自动化的依赖，确保任何情况下都能产出结果。

**Level 1（优先）：手动输入 — 用户提供榜单截图/复制文本/口头描述**
- 提供标准输入模板：书名/作者/题材/字数/追读率/完读率/榜单排名/简介
- 用户可提供1-10本书的数据
- 用户已提供数据 → 直接进入 Phase 3 分析，无需降级
- 用户表示"没有数据/不确定" → 降级至 Level 2

**Level 2（半自动）：浏览器MCP — 用户确认后，使用 integrated_browser MCP 工具半自动抓取**
- 仅抓取公开榜单页面（番茄热榜/起点月票榜等）
- 每页停留3秒，抓取后立即关闭
- 抓取数据存入 `对标/扫榜数据.md`
- 抓取失败/反爬/验证码 → 降级至 Level 3

**Level 3（兜底）：WebSearch — 搜索"番茄小说热榜""起点月票榜"等关键词**
- 从搜索结果中提取书名/作者/题材信息
- 无法获取追读率/完读率等数据时标注"数据缺失"
- 明确告知用户："建议手动核实最新榜单"
- 输出"数据置信度"标记：高（实时）/中（近期）/低（可能过时）

**降级触发条件汇总**：
| 异常类型 | 降级动作 |
|---------|---------|
| 用户已提供数据 | 直接进入 Phase 3，不降级 |
| 用户表示无数据/拒绝提供 | → Level 2 |
| 浏览器MCP抓取超时(>15s) | → Level 3 |
| HTTP 403/429/反爬 | → Level 3 |
| 验证码拦截 | 暂停+提示用户 → Level 3 |
| 数据解析为空 | → Level 3 |

#### Phase 3：数据分析与趋势提炼

从采集数据中提取：
- **题材趋势**：哪些题材反复上榜？新兴题材是什么？
- **卖点提炼**：上榜作品的共同卖点是什么？
- **读者画像**：这些作品的目标读者是谁？
- **竞争格局**：同类作品密度如何？差异化空间在哪？

#### Phase 4：选题建议

输出 `选题决策.md`，包含：
```
## 选题决策

### 推荐方向
1. [方向A]：热度高 + 竞争适中 + 匹配你的优势
   - 能爆的原因：...
   - 差异化切入点：...
   - 推荐对标书：...

2. [方向B]：...

### 风险评估
- [方向A] 风险：...
- [方向B] 风险：...

### 建议
综合推荐 [方向X]，理由：...
```

## 系统化拆文

> 融合 V1.0 四维拆解 + oh-story 的结构化输出体系。

### 拆文策略

| 拆文深度 | 适用场景 | 产出 |
|---------|---------|------|
| 黄金三章精拆 | 快速评估对标书是否值得深拆 | 前3章深度拆解报告 |
| 全量拆解 | 确定为核心对标书 | 完整四维拆解 + 结构化目录 |
| 专题拆解 | 针对性学习某个维度 | 单维度深度报告 |

### 对标书原文获取 — 四级降级链（V6.0.1新增）

> **痛点对症**：拆文质量高度依赖对标书原文，但用户提供的原文质量参差（有的给完整正文，有的只给书名）。原文件仅定义了扫榜数据的降级链，未定义对标书原文获取的降级策略，导致LLM在数据不足时仍强行拆文，产出空泛。本降级链明确4个数据等级及对应的拆文精度预期，并给出数据不足时的降级建议。

**【对标书原文获取四级降级链】**

| Level | 数据形态 | 提供方式 | 拆文精度预期 | 适用拆文深度 |
|-------|---------|----------|-------------|-------------|
| **Level 0（最佳）** | 对标书完整正文（≥10章） | 用户粘贴/提供文件路径 | 高 — 可执行全量拆解（3维度+四维展开） | 全量拆解 |
| **Level 1** | 对标书关键章节（前3章+高潮章+结尾） | 用户粘贴关键章节 | 中高 — 可执行黄金三章精拆+结构推断 | 黄金三章精拆+结构推测 |
| **Level 2** | 对标书简介+标签+读者评论 | 用户描述/WebSearch补充 | 中 — 仅可执行市场维度+文本特征推测 | 专题拆解（市场定位） |
| **Level 3** | 仅书名 | 用户提供书名，AI通过WebSearch获取公开信息 | 低 — 仅可执行市场定位粗判，无法拆文本/结构维度 | 选题参考（非拆文） |

**【各Level详细说明】**

**Level 0（最佳）：用户提供完整正文（≥10章）**
- 用户将原文粘贴到 `拆文库/{书名}/原文/` 目录，或提供文件路径
- 直接进入全量拆解流程（3维度+四维展开参考）
- 数据置信度：高
- 拆文精度：可提取句式节奏/感官密度/对话风格等文本细节，产出可复用技法清单

**Level 1：用户提供关键章节（前3章+高潮章+结尾）**
- 关键章节组合：前3章（黄金三章）+ 1个高潮章 + 结尾章（约5-8章）
- 执行黄金三章精拆 + 基于关键章节推断全书结构
- 数据置信度：中高
- 拆文精度：可提取开篇钩子/爽点密度/章末钩子类型，但呼吸节奏/伏笔密度仅可推测
- **降级建议**：建议用户补充高潮章前后的2-3章过渡章，以提升节奏分析精度

**Level 2：用户提供简介+标签+读者评论**
- 用户提供：书名+简介+平台标签+读者评论摘选（可WebSearch补充）
- 仅可执行市场维度拆解（读者画像/卖点提炼/竞争分析）
- 数据置信度：中
- 拆文精度：无法拆解文本维度和结构维度，仅产出市场定位报告
- **降级建议**：强烈建议用户补充前3章原文（至少Level 1），否则拆文价值有限；若无法获取原文，建议改用06的扫榜数据做选题参考而非拆文

**Level 3：仅书名**
- 用户仅提供书名，AI通过WebSearch获取公开信息（简介/标签/部分评论）
- 仅可执行市场定位粗判
- 数据置信度：低
- 拆文精度：无法拆文，仅可作为选题参考
- **降级建议**：明确告知用户"仅书名无法执行拆文，建议至少提供简介+读者评论（Level 2）或关键章节原文（Level 1）"；若用户无法补充，建议跳过拆文步骤，直接用扫榜数据做选题决策

**【数据不足时的降级决策树】**

```
用户提供原文?
├─ 是（≥10章）→ Level 0，执行全量拆解
├─ 是（关键章节5-8章）→ Level 1，执行黄金三章精拆+结构推断
├─ 否，但有简介+评论 → Level 2，仅执行市场维度拆解
│   └─ 建议用户补充前3章原文提升至Level 1
└─ 否，仅书名 → Level 3，无法拆文
    └─ 告知用户数据不足，建议跳过拆文或补充数据
```

> **降级原则**：①拆文精度与数据等级严格匹配，Level 2/3不强行拆解文本维度（会产出空泛内容）；②每次降级须明确告知用户精度下降程度；③Level 3不建议执行拆文，应引导用户补充数据或改用扫榜数据。

### 拆文维度

#### 3维度拆文体系（V5.1更新 — 替代原有四维拆文）

> V5.1 起，拆文维度从"市场定位/角色塑造/剧情结构/文风技法"四维精简为"市场维度/文本维度/结构维度"三维，每维度产出可量化评分与可复用清单，聚焦数据驱动决策。下方四维详细模板保留为展开参考，下游 M1-M5 / 功能位映射仍引用其产出。

**维度1：市场维度 — 追读率/完读率/收藏转化率/榜单趋势/读者评论关键词**
- 输出：市场吸引力评分(1-10) + 目标读者画像 + 爆款要素提取

**维度2：文本维度 — 开篇300字分析/章末钩子类型/爽点密度/对话风格/感官使用/句式特征**
- 输出：文本质量评分(1-10) + 可复用技法清单 + AI味评估

**维度3：结构维度 — 章节结构类型/情节推进节奏/伏笔布局/角色出场策略/信息释放节奏**
- 输出：结构评分(1-10) + 结构模板提取 + 对标迁移建议(M1-M5)

**三维度评分汇总**：

| 维度 | 评分(1-10) | 核心产出 |
|------|-----------|---------|
| 市场维度 | __ | 市场吸引力评分 + 读者画像 + 爆款要素 |
| 文本维度 | __ | 文本质量评分 + 可复用技法 + AI味评估 |
| 结构维度 | __ | 结构评分 + 结构模板 + M1-M5迁移建议 |
| **综合** | __ | 综合评估 + 对本作品的启示 |

#### 第一维：市场定位分析

**为什么读者买单？**

```
【市场定位诊断】

1. 一句话卖点
   ├─ 对标作品的"转述测试"：读者会怎么向朋友推荐？
   └─ 提炼公式：[身份] + [金手指/设定] + [核心爽点]

2. 读者画像
   ├─ 年龄层：初中生/高中生/大学生/上班族
   ├─ 阅读场景：通勤/睡前/课间/摸鱼
   ├─ 核心需求：爽/甜/虐/悬疑/新奇
   └─ 情绪代偿：现实缺什么，小说给什么

3. 同类竞争
   ├─ 同题材前10名的共同特点
   ├─ 对标作品的差异化优势
   └─ 你的作品与对标作品的差异化定位

4. 平台适配度
   ├─ 番茄：快节奏、前500字爽点、单章4000字
   ├─ 起点：慢热、世界观构建、长线埋伏
   └─ 飞卢：极致爽、更新快、脑洞大
```

#### 第二维：角色塑造拆解

```
【角色塑造拆解】

1. 主角原型
   ├─ 身份：底层/中产/强者/重生者/穿越者
   ├─ 性格关键词：如"粗粝、嘴贱、护短"
   ├─ 核心驱动力：求生存/复仇/守护/证明自己
   ├─ 缺点/限制：破局的关键障碍
   └─ 成长弧线：起点→终点（变化路径）

2. 互动系统
   ├─ 主角 vs 敌人：压迫感来源
   ├─ 主角 vs 同伴：情感羁绊
   ├─ 主角 vs 配角：信息差/利用/守护
   └─ 三人成戏规则：主角+同伴+敌人 = 三角张力

3. 配角生态位
   ├─ 导师型：提供信息/能力/方向（中后期退场）
   ├─ 同伴型：互补性格/功能/背景
   ├─ 反派型：要有压迫感+表演型人格
   ├─ 感情线：拉扯感来源（误会/地位差/身份冲突）
   └─ 功能型：推动剧情/提供情报/制造障碍

4. 角色功能位映射（v2.0新增）
   ├─ 把对标书的每个角色看成"功能位"
   │   ├─ 对手位：提供压迫感 + 打脸对象
   │   ├─ 盟友位：提供情感羁绊 + 信息释放
   │   ├─ 催化剂位：推动主角变化 + 制造转折
   │   └─ 信息位：提供设定/世界观/伏笔
   ├─ 映射到你的角色时：用你的素材填充功能位
   └─ 原则：学"功能"不学"人设"，避免抄袭
```

#### 第三维：剧情结构拆解

```
【剧情结构拆解】

1. 黄金三章分析
   ├─ 第1章控制点：前500字发生了什么？
   │   ├─ 开篇方式：危机切入/日常异常/金手指觉醒/对话开场/结果前置
   │   ├─ 第一句：是否抓住了读者？
   │   └─ 第1章结束：钩子是什么？
   ├─ 第2章节奏：铺垫还是释放？
   └─ 第3章确认：读者是否已经"入坑"？

2. 分卷结构
   ├─ 每卷的核心冲突是什么？
   ├─ 每卷的情绪曲线：起→承→转→合
   ├─ 每卷的爽点分布：前/中/后段各几个？
   └─ 每卷的收尾钩子：为下一卷埋伏什么？

3. 呼吸节奏
   ├─ 铺垫章 vs 爆发章的比例
   ├─ 爽点间隔：多少字/几章一个爽点？
   ├─ 情感线的"拉扯"节奏：甜→虐→甜
   └─ 大高潮前的小高潮铺垫

4. 伏笔系统
   ├─ 伏笔类型：物件/对话/事件/设定/角色
   ├─ 伏笔密度：前10章埋了多少伏笔？
   ├─ 回收节奏：多久回收？隔多少章？
   └─ 长线伏笔：跨卷伏笔的设计方式

5. 情绪模块提取（v2.0新增）
   ├─ 读者需求：这本书满足读者什么情绪需求？
   ├─ 情绪引擎：哪些场景/事件持续驱动情绪？
   ├─ 可复现模块：哪些情绪场景可以换皮复用？
   └─ 爽文套路框架：打脸/翻盘/升级的情绪节拍
```

#### 第四维：文风技法拆解

```
【文风技法拆解】

1. 句式节奏
   ├─ 平均句长：短句为主还是长句为主？
   ├─ 段落长度：短段密集还是长段叙事？
   ├─ 对话占比：约多少比例？
   └─ 叙述者距离：贴身/中距/全知？

2. 感官侧重
   ├─ 最常用的感官：视觉/触觉/嗅觉/听觉？
   ├─ 感官密度：每多少字一处感官描写？
   └─ 脏感/真实感：是否有物理代价描写？

3. 对话风格
   ├─ 口语化程度：高（说书人）还是低（文人腔）？
   ├─ 说话方式：直给还是话里有话？
   ├─ 信息释放：对话承担多少设定交代？

4. 情绪表达
   ├─ 直接表达 vs 间接投射
   ├─ 内心独白的频率和深度
   ├─ 嘴硬心软/言行不一的频率
   └─ 发刀/发糖的方式和节奏
```

### 对标节奏迁移 M1-M5（v2.1 新增）

> 源自 oh-story-claudecode 的「对标节奏迁移」理念。把爆款拆解的一级结构关键点（1/4·中点·3/4）换素材排进本卷卷纲，让本卷节奏踩在市场验证过的节拍上。**学"节奏"不学"剧情"**——换皮不抄袭。

**适用前提**：已对主对标书完成四维拆文，`对标/{书}/剧情/节奏.md` 已生成。

**M1-M5 迁移流程**：

| 步骤 | 操作 | 产出 |
|------|------|------|
| **M1 定位锚点** | 从 `对标/{书}/剧情/节奏.md` 提取对标书的一级结构关键点：开篇钩子（0-10%）、1/4 转折、中点（50%）、3/4 危机、高潮（90%）、收尾 | 对标结构坐标表 |
| **M2 换素材** | 把每个锚点的"剧情内容"换成你的素材，但保留"节奏功能"（如对标1/4是"主角首次受挫"→你的1/4也设计一个"首次受挫"事件，但具体事件不同） | 本卷节奏骨架 |
| **M3 校验占比** | 检查本卷的 高压/推进章占比是否≥60%、低压+过场章≤15%（沿用本地大纲五检第⑤问：每卷1/4·中点·3/4位置各锚定一个关键情节，确保节奏骨架不塌陷）；1/4·中点·3/4 是否各锚定一个关键情节 | 占比校验报告 |
| **M4 写入卷纲** | 把 M2 的节奏骨架写入 `对标/{书}/剧情/节奏.md` 供02a Read参考（而非写入本卷卷纲的「对标结构坐标」字段） | 节奏.md·对标结构坐标 |
| **M5 缺失降级** | 若 `对标/{书}/剧情/节奏.md` 缺失 → 回退到 `章节/*_摘要.md` + `拆文报告.md` 的节奏摘要，反推锚点；记录 `rhythm_missing` 标记 | 降级节奏骨架 |

**M1-M5 与本地大纲五检的关系**：
- 大纲五检第⑤问（定义于02b_细纲质检.md Part O5）：每卷必须在1/4、中点(50%)、3/4三个位置各锚定一个关键情节，确保叙事节奏有骨架。M3是此检查项在对标迁移场景下的应用。
- M1-M5 是**操作流程**——告诉你怎么把对标的锚点迁移过来
- 两者配合：先用 M1-M5 迁移，再用大纲五检校验

**反抄袭底线**：迁移的是"节奏功能"（受挫/转折/危机/高潮的节拍位置），不是"剧情内容"。同一锚点的具体事件必须是你自己的素材。

### legacy 拆文库回退规则（v2.1 新增）

> 旧版拆文库可能缺 `剧情/情绪模块.md` 或 `剧情/节奏.md`。本规则定义缺文件时的降级路径，保证写作不中断。

| 缺失文件 | 回退路径 | 标记 |
|---------|---------|------|
| `剧情/情绪模块.md` 缺失（v2.0 新契约拆文库） | **停下提示重跑拆文** Stage 3+，不得假装已召回 | 不标 legacy |
| `剧情/情绪模块.md` 缺失（legacy 拆文库） | 依次回退：`拆文报告.md` 市场维度/文本维度/结构维度摘要 → `文风.md` 可借鉴技巧 → 匹配 `章节/第K章_摘要.md` | `legacy_deconstruction: true` + `module_missing` |
| `剧情/节奏.md` 缺失（legacy 拆文库） | 依次回退：`拆文报告.md` 节奏摘要 → 匹配章摘要 → `剧情/故事线.md` | `legacy_deconstruction: true` + `rhythm_missing` |
| `文风.md` 缺失 | 回退到 `拆文报告.md` 文风摘要；无则提示重跑拆文 | `style_missing` |

**判定 legacy 的方法**：`拆文报告.md` 中是否含 `市场维度/文本维度/结构维度` 摘要字段。无 → legacy。

**回退时的写作约束**：legacy 回退后写作可继续，但 03 扩写引用情绪模块时需在定位报告标注 `legacy_deconstruction: true`，提醒精度下降。

## 执行率优化说明 V6.0.1

> **优化目标**：将本步骤LLM执行率从70%提升至75%+。
> **优化方式**：补全对标书原文获取的降级策略——原文件仅定义了扫榜数据的三级降级链，但未定义拆文所需的对标书原文获取降级链，导致数据不足时LLM强行拆文产出空泛。本次新增四级降级链+精度预期标注+降级决策树。

### 本次优化清单

| 优化点 | 位置 | 优化内容 | 预期效果 |
|--------|------|----------|----------|
| 对标书原文获取四级降级链 | 拆文策略后（新增） | 提供4个数据等级（完整正文/关键章节/简介评论/仅书名）+ 每级拆文精度预期（高/中高/中/低）+ 降级决策树 + 数据不足时的降级建议 | LLM从"数据不足仍强行拆文"→"按数据等级匹配拆文深度，不足时主动建议用户补充" |

### 优化原则

1. **不删除原有内容**：原扫榜数据三级降级链和3维度拆文体系保持不变，新增降级链作为拆文数据获取的补充
2. **精度预期可执行**：每个Level明确标注"可执行哪些拆文维度/不可执行哪些"，避免LLM在Level 2/3强行拆解文本维度
3. **降级建议具体**：每个Level给出具体的"如何提升数据等级"建议（如"补充前3章原文提升至Level 1"）
4. **决策树可视化**：提供降级决策树，LLM可按树状逻辑判断当前数据等级

### 预期执行率提升路径

- 对标书原文获取执行率：65% → 75%（四级降级链明确数据等级与拆文深度的匹配关系）
- 拆文精度匹配执行率：60% → 78%（精度预期标注避免强行拆解空泛内容）
- 全步骤综合执行率：70% → 76%（达成目标75%+）

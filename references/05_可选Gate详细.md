> **加载条件**：深度模式（/mode_hardcore 或 /deep）/ 09审稿师评分<70 / 禁用词密度>15处/千字
> **加载方式**：Read `references/05_可选Gate详细.md`

## 可选 Gate A：节奏打碎（原Gate 4）

> AI写作的典型特征：句长过于均匀，缺乏节奏变化。

### 句长方差检测

1. 统计全章每句字数
2. 计算句长标准差
3. 标准差<5（句长过于均匀）→ 需要打碎节奏

### 节奏打碎策略

| 问题 | 修正 |
|------|------|
| 连续5+句长度相近（±3字） | 插入1个短句（≤5字）或长句（30+字） |
| 全章无短句（≤5字） | 在情绪转折点插入短句 |
| 全章无长句（30+字） | 在描写段落使用长句 |
| 段落全部3-5句 | 打破为1句段落或7+句段落 |

### 节奏与情绪匹配

| 情绪 | 推荐句长 | 原因 |
|------|---------|------|
| 紧张/战斗 | 短句为主（5-15字） | 快节奏 |
| 平静/日常 | 中句为主（15-25字） | 舒缓 |
| 描写/氛围 | 长句为主（25-40字） | 沉浸 |
| 震惊/转折 | 极短句（1-5字）独立成段 | 冲击力 |

## 可选 Gate B：意象疲劳与比喻质量（原Gate 7）

> 意象/比喻反复使用导致疲劳 + 弱智明喻筛选。

### 意象疲劳检测

1. 扫描全章出现的核心意象（如"铜钱"、"月光"、"刀"）
2. 统计每个意象出现次数
3. 同一意象出现≥4次 → 标记"意象疲劳"
4. 意象疲劳修正：减少出现次数或每次出现时增加变化

### 比喻质量检测（与04比喻质量门互补）

> 04做"数量检测"（一段话超过2个比喻），05做"质量检测"（弱智明喻筛选）。

**弱智明喻特征**：
- 喻体和本体之间没有真实的相似性
- 喻体是陈词滥调（如"像流星一样"）
- 喻体与角色生活经验无关
- 喻体创造了零信息增量（如"眼睛像星星一样亮"）

**弱智明喻清单（部分）**：
- "像流星一样坠落/划过"
- "眼睛像星星/宝石/明珠"
- "心像刀绞一样"
- "快如闪电"
- "静得能听到针掉地上的声音"
- "笑容像阳光一样温暖"
- "泪如雨下"
- "冷若冰霜"

### 修辞多样性检测（朱雀B融入）

1. 统计全章修辞手法分布（比喻/拟人/排比/夸张/通感/借代）
2. 单一修辞占比>60% → 需增加其他修辞
3. 全章无通感 → 建议增加1处通感（核心场景）

## 可选 Gate C：叙事视角一致性（原Gate 8）

> POV（Point of View）跳跃是AI写作的隐蔽问题。

### POV检测规则

1. 确定本章主视角角色
2. 扫描所有"心理描写"——是否只属于主视角角色？
3. 扫描"信息获取"——主视角是否知道了不该知道的信息？
4. 扫描"感知描写"——是否有非主视角角色的内心活动？

### POV违规类型

| 类型 | 示例 | 修正 |
|------|------|------|
| 视角跳跃 | 主角视角中突然写了配角的内心 | 删除配角内心或改为外部观察 |
| 全知泄露 | 叙述者透露主角不知道的信息 | 删除或改为角色猜测 |
| 感知越权 | 主角感知到了不该感知的东西 | 改为合理感知或删除 |
| 评价越权 | 叙述者评价主角的行为 | 改为通过他人之口呈现 |

### 多视角章节特殊处理

- 如果本章设计为多视角（如A-B-A交替），需在02a细纲中标注
- 多视角章节的POV切换必须在段落分隔处
- 每个视角段落的内心描写只属于该视角角色

## 可选 Gate D：解释腔/上帝感（原Gate 11）

> 04核心Step 2已做全量消除，05仅抽样复核。

### 抽样规则

1. 从全章随机抽取3个段落
2. 检查是否存在解释腔信号词
3. 检查是否存在"上帝视角"评价
4. 抽样发现≥2处 → 建议回退04重新处理

### 解释腔信号词（04已全量扫描，此处仅复核）

- "也就是说/换句话说/简单来说/说白了"
- "之所以...是因为/这意味着/这说明"
- "他不知道的是/他没有意识到/他没发现"
- "这并非...而是/不是...而是"

### 上帝感信号词

- "在冥冥之中/在命运的安排下"
- "谁也没有想到/没人能预料到"
- "这注定是一个不平凡的夜晚"
- "历史将记住这一天"

### 安排感信号词

- "恰好/恰巧/正好"（连续使用≥2次）
- "就在这时/就在此刻"（每章≤2次）
- "仿佛一切都是注定" 
- "冥冥中自有天意"

# 上下文预算管理器（V5.4 — 借鉴InkOS protected/compressible分层）

> 本文件从 SKILL.md 拆分外置（V6.0）。SKILL.md 中仅保留简要引用。
> **核心问题**：在Trae等平台上，如果同时加载设定JSON+大纲+细纲+衔接包+前3章正文+references素材，很容易超出窗口。"旧历史淹没当前指令"。
> **解决方案**：声明各内容块的优先级层级，当总预算超限时按优先级自动丢弃低优先内容。
> **预算估算工具**：运行 `node scripts/estimate-context-load.js --scene=<场景类型>` 可自动估算当前场景需要加载的文件总行数，超2000行时告警。

---

## 三层优先级体系

| 层级 | 含义 | 包含内容 | 处理策略 |
|------|------|---------|---------|
| **protected** | 不可压缩 | 当前章细纲+衔接包P0+02a伏笔健康报告+用户指令+03a场景蓝图 | 永远加载，不可跳过 |
| **compressible** | 可压缩 | 前3章正文（→摘要化）+设定JSON（→关键字段）+大纲（→当前卷段）+references扩展模块 | 超限时自动压缩为摘要 |
| **droppable** | 可丢弃 | P2可选字段+02a P1可选字段+非当前场景的references+历史审稿报告 | 超限时直接跳过 |

## 预算超限处理流程

```
上下文预算管理(每步执行前检查):
├─ 1. 估算当前步骤需要的总token数
├─ 2. 如果总token < 窗口上限×70% → 正常加载全部
├─ 3. 如果总token ≥ 窗口上限×70% → 启动压缩:
│   ├─ 跳过所有droppable内容
│   ├─ 前3章正文 → 压缩为每章200字摘要
│   ├─ 设定JSON → 只保留当前场景相关角色+力量体系当前等级
│   └─ 大纲 → 只保留当前卷+下一卷摘要
├─ 4. 如果压缩后仍 ≥ 窗口上限×80% → 启动深度压缩:
│   ├─ 前3章正文 → 只保留上一章（当前2章摘要化）
│   ├─ 衔接包 → 只保留P0字段（跳过P1）
│   └─ 04/05核心层 → 只加载执行路由器+当前Step
└─ 5. 输出"上下文预算报告"（标注哪些内容被压缩/跳过）
```

---

## 各步骤的预算分配

| 步骤 | protected必须 | compressible优先 | droppable默认 |
|------|-------------|----------------|-------------|
| 02a 细纲生成 | 大纲+设定JSON+伏笔追踪表 | 前章细纲 | — |
| 02b 细纲质检 | 当前章细纲+衔接包 | 02a伏笔健康报告 | — |
| 03a 扩写 | 细纲+衔接包P0+场景蓝图 | 前2章正文+设定 | references扩展模块 |
| 03b 质检 | 初稿+细纲+衔接包 | — | references |
| 04 精修 | 初稿+03b报告+04_core | 精修意图包 | 04扩展模块（按场景加载） |
| 05 去AI味 | 精修稿+05_core | 精修意图包 | 05扩展模块（按模式加载） |

## 数值锚点不可压缩（V2.8）

`number_anchors` 和 `power_system` 字段在任何压缩操作中必须完整保留，不得精简。这两个字段是全书数值和战力一致性的唯一基线，一旦在压缩中丢失，后续所有步骤的一致性校验将失效。

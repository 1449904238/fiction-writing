## 节奏控制协议

# 节奏控制协议

> **核心定位**：本文件是03扩写系统与02细纲编写系统的"节奏油门"——与"情绪工程协议"（情绪油门）和"质感注入协议"（质感油门）并行执行。
> 情绪工程协议管"读者身体有没有反应"，质感注入协议管"读者能不能看见画面"，本协议管"读者会不会熬夜追更"。
>
> **痛点对症**：当前skill在节奏控制上只有检测器（check-rhythm.js的爽点间隔/情绪平坦/节奏塌陷/篇幅配比/章末钩子），没有正面引导——能检测"节奏出问题了"，但不知道"怎样设计节奏才抓人"。
> 本协议提供节奏设计的正面方法论，让AI从"事后检测"升级为"事前设计"。
>
> **理论依据**：节奏的本质不是"快"，是"伸与缩交替"——缩=紧张感/危机/铺垫（拉满弓弦），伸=爽点/爆发/奖励（利箭离弦）。全是爽点=无脑，铺垫过多=憋屈，两者平衡才是节奏基础。

## 一、核心理论：伸与缩交替

### 1.1 节奏的本质

```
节奏 ≠ 快慢
节奏 = 伸与缩的交替频率

缩（蓄力期）：紧张感、危机、铺垫 — 像拉满弓弦，让读者产生压抑和期待
伸（释放期）：爽点、爆发、奖励 — 像利箭离弦，让读者获得宣泄和畅快

错误认知：
  ❌ 节奏快 = 全程高能 = 读者会爽
  ✅ 全程高能 = 审美疲劳 = 读者弃书
  ❌ 铺垫多 = 信息量大 = 读者会沉浸
  ✅ 铺垫过多不释放 = 憋屈感 = 读者弃书
```

### 1.2 伸与缩的黄金比例

| 章节类型 | 缩:伸比例 | 读者心理状态 | 典型场景 |
|---------|:------:|------------|---------|
| 日常章 | 7:3 | 轻松阅读，积累信息 | 赶路/修炼/过渡 |
| 铺垫章 | 8:2 | 压抑期待，弹簧压紧 | 反派施压/主角受挫/危机酝酿 |
| 冲突章 | 5:5 | 紧张与释放交替 | 小冲突/对决/试探 |
| 爽点章 | 3:7 | 释放为主，宣泄情绪 | 打脸/突破/翻盘/碾压 |
| 高潮章 | 2:8 | 全力释放，情绪爆发 | 大决战/关键揭秘/终极反转 |

**关键约束**：
- 连续2章以上缩>70% → 读者憋屈，需在第3章插入≥1个"小伸"（小爽点/小反转）
- 连续2章以上伸>70% → 爽感递减，需在第3章插入"缩"（新危机/新悬念）
- 单章内部也要有"伸与缩"的小循环——不能一章全是铺垫或全是爆点

### 1.3 与现有系统的关系

| 现有系统 | 定位 | 与本协议的关系 |
|---------|------|--------------|
| 情绪工程协议 | 情绪油门：情绪冲击力=预期反差×合理度 | 情绪是节奏的"内容"——节奏控制情绪释放的"时机" |
| 爽点与对照表 | 爽点保护：爽感模式/呼吸节奏 | 本协议是爽点节奏的"宏观版"——从单章扩展到跨章弧线 |
| check-rhythm.js | 节奏检测：爽点间隔/情绪平坦/节奏塌陷 | 本协议是检测器的"正面补丁"——检测器查问题，本协议教怎么设计 |
| 衔接包 | 跨章状态传递 | 本协议的"伸与缩"信息通过衔接包传递——前章的缩（压抑）→后章的伸（释放） |

## 二、第一招：信息密度调节

### 2.1 核心原理

> **平淡情节加信息量，高潮情节减信息量。**

| 情节类型 | 信息密度 | 具体操作 | 效果 |
|---------|:------:|---------|------|
| 平淡情节（赶路/闭关/过渡） | **高** | 加入新线索、秘密、反派阴谋、世界观细节 | 做到"动静结合"——表面平静，暗流涌动 |
| 高潮情节（大决战/关键冲突） | **低** | 用细节描写、旁观者反应、环境渲染放大张力 | 像电影慢动作——让读者看清"主角高光时刻" |

### 2.2 平淡情节的信息增量技法

**技法1：新线索植入**
- 在赶路/过渡中，通过环境/对话/回忆，悄悄植入1条新线索
- ❌ "两人赶了三天的路，终于到了青云城。"（零信息）
- ✅ "两人赶了三天的路。第三天傍晚，林凡在路边茶摊的旗子上，看到了青云宗的暗记——那个标记，只有内门弟子才认得。"（植入"青云宗暗记"线索）

**技法2：秘密揭示**
- 在平淡场景中，通过角色不经意的一句话/一个动作，揭示一个秘密
- 秘密可以是：角色过去的经历、隐藏的身份、对某人的真实看法
- ⚠️ 揭示要"不经意"——角色自己可能都没意识到说漏了嘴

**技法3：反派阴谋推进**
- 在主角休息/修炼时，切1-2段写反派在做什么
- 不是全知视角大段描写，而是"冰山一角"——只展现反派的1个动作/1句话
- 例："同一时间，青云宗后山，有人把一封信塞进了石缝。"

**技法4：世界观细节渗透**
- 在过渡中通过角色的感官体验，渗透世界观细节
- 不是"介绍世界观"（解释腔），而是"角色在体验世界观"
- 例："他闻到了城里飘来的药香——青云城以炼丹闻名，连空气都是苦的。"

### 2.3 高潮情节的信息精简技法

**技法1：慢动作放大**
- 关键时刻，用2-3段细节描写"拉长时间"
- 不是"他一拳打过去"（1句），而是"拳头/空气/对手瞳孔/旁人反应"（4-5句）
- ⚠️ 只对"本章最重要的1个瞬间"使用慢动作，不要每个动作都慢放

**技法2：旁观者反应**
- 用旁观者的反应放大主角的高光时刻
- 不是"所有人都震惊了"（情绪词直给），而是"旁边的人把碗放下了，汤洒了一地"
- 旁观者反应≥2处时，用不同身份/不同反应（老人叹气/年轻人站起来/孩子捂眼睛）

**技法3：环境渲染**
- 用环境变化配合情绪高潮
- 不是"天变了"（万能比喻），而是具体的环境变化——"风突然停了/灯晃了一下/远处的钟响了"
- 环境渲染≤2处/高潮场景，多了=注水

### 2.4 信息密度与节奏的映射

```
【信息密度-节奏映射表】

平淡情节（信息密度高）：
  信息类型：新线索/秘密/阴谋/世界观
  每500字信息点：≥2个
  对应节奏：缩（蓄力）——表面平静，读者在积累信息

高潮情节（信息密度低）：
  信息类型：细节/旁观者/环境
  每500字信息点：≤1个（新信息），但感官细节≥3个
  对应节奏：伸（释放）——读者在享受"慢动作高光"
```

## 三、第二招：钩子、冲突与循环

### 3.1 核心原理

> **优秀网文节奏由"无数小冲突、小高潮、小钩子"套环构成。**
> **核心技巧：在一个冲突即将解决时，立刻埋下新冲突的引子。**

### 3.2 冲突套环机制

```
【冲突套环模型】

冲突A 解决中 → 冲突A的"余波"触发冲突B → 冲突B酝酿时冲突C的引子出现

示例：
  主角刚打败反派小弟（冲突A解决）
  → 小弟临死前的一句话揭开更大的阴谋（冲突B引子）
  → 主角调查阴谋时，发现盟友也在被追杀（冲突C引子）

关键：永远不让读者"心完全放下来"
```

### 3.3 冲突套环的具体操作

**操作1：余波引子法**
- 当前冲突解决的"余波"中，埋入下一个冲突的"引子"
- 引子类型：一句话/一个物件/一个反常现象/一个未接的电话
- 引子要"小"——读者注意到了但主角可能没注意到

**操作2：代价引子法**
- 当前冲突的"代价"成为下一个冲突的"起因"
- 例：主角用了禁术打败敌人 → 禁术的反噬成为下一个冲突
- 代价必须在当前冲突中"实际发生"——不能事后补充

**操作3：旁观者引子法**
- 当前冲突的"旁观者"成为下一个冲突的"参与者"
- 例：主角打脸时，围观人群中有人暗中记下了他的脸
- 旁观者的反应要"不显眼"——读者可能注意到，但不确定

### 3.4 钩子与冲突的嵌套节奏

```
【大高潮套小高潮，老悬念解新悬念生】

大弧线（卷级）：卷首悬念 → 卷中冲突升级 → 卷末大高潮 + 新悬念
  ├─ 中弧线（5章级）：小高潮1 → 铺垫 → 小高潮2 → 铺垫 → 小高潮3
  │   ├─ 小弧线（章级）：章首钩子 → 章中冲突 → 章末钩子（悬念）
  │   └─ 小弧线（章级）：章首钩子 → 章中冲突 → 章末钩子（新悬念）
  └─ 中弧线（5章级）：...

关键约束：
  - 同一卷内，大小弧线的"伸与缩"要错峰——大缩时可以有小的伸
  - 章末钩子类型连续3章不重复（已有13式章末+7式章首）
  - 每5章必须有1个"大钩子"（整卷级别的悬念升级）
```

### 3.5 与现有钩子系统的关系

| 现有系统 | 本协议增强 |
|---------|----------|
| 13式章末钩子 | 增加"钩子裂变"——1个钩子解决时分裂出2个新钩子 |
| 7式章首钩子 | 增加"钩子回应"——章首必须回应上章章末钩子（至少提及） |
| 连续3章不重复 | 增加"钩子升级"——每5章的钩子力度必须比前5章更大 |

## 四、第三招：情绪波段控制

### 4.1 核心原理

> **节奏本质是"情绪波动曲线"。**
> **避免"一条直线到底"——要么流水账，要么全程无脑爽。**

> **与情绪工程协议的分工**：情绪工程协议管"单章内的情绪如何设计"（情绪蓝图→情绪锚点→情绪通路），本协议管"跨章的情绪曲线如何起伏"（宏观波段控制）。

### 4.2 压抑到爆发模型（弹簧原理）

```
【弹簧模型】

想让读者在第10章感到极致的爽
  → 在第5-9章铺垫主角困境、强化反派压迫感
  → 把"弹簧压得越紧"，高潮释放时的爽感冲击力越强

弹簧压紧三要素：
  ① 困境要具体——不是"主角很惨"，是"主角失去了什么"（具体的人/物/机会）
  ② 反派要有存在感——反派每章至少出现1次（直接或间接），让读者持续感受到压力
  ③ 希望要若隐若现——不是"完全没有希望"（读者会绝望弃书），而是"有希望但够不着"
```

### 4.3 情绪波段设计模板

```
【卷级情绪波段设计】

第1-2章：情绪起点（建立读者对主角的共情+初始困境）
  情绪温度：30-40（凉→微暖）
  缩伸比：7:3
  关键任务：让读者"站队"主角

第3-5章：情绪下压（逐步加码困境）
  情绪温度：20-30（越来越冷）
  缩伸比：8:2
  关键任务：每章加一个"新的压力源"（不是重复同一个压力）

第6-8章：情绪触底（最低点+一线希望）
  情绪温度：10-20（最冷，但有一丝暖意）
  缩伸比：8:2（但第8章末尾必须有一个"小伸"）
  关键任务：读者觉得"主角太惨了但还有希望"

第9-10章：情绪爆发（弹簧释放）
  情绪温度：80-90（急速升温）
  缩伸比：2:8
  关键任务：分批爆发——小爆发→中爆发→大爆发
```

### 4.4 情绪波段与番茄算法的绑定

> 此部分与情绪工程协议 2.3 节互补。情绪工程协议关注"单章情绪位置"，本协议关注"跨章波段变化"。

| 波段位置 | 设计要求 | 算法依据 |
|---------|---------|---------|
| 前3章 | 情绪曲线必须"先降后升"——第1章建立共情→第2章困境加深→第3章首次反击 | 前三章完读率<10%终止推荐 |
| 第5章节点 | 必须出现"情绪触底反弹"——第5章是读者决定是否追更的关键节点 | 第5章追读率≥40%获5倍流量 |
| 每10章 | 情绪曲线必须完成一个"缩→伸"完整周期 | 维持整体完读率 |
| 换卷节点 | 卷末情绪必须"伸>缩"——让读者带着爽感进入下一卷 | 卷末留存率决定下一卷初始流量 |

### 4.5 不要怕写主角受挫

> **只要受挫是为了后续更猛烈的反击，就是高明的节奏控制。**

**受挫设计原则**：
- 受挫必须"有代价"——不是"主角被骂了几句"，而是失去了具体的东西
- 受挫后必须有"反击计划"——即使是模糊的（"他会让他们付出代价"），不能是"主角认命了"
- 受挫→反击的间隔不能超过3章——超过3章读者会忘记"为什么受挫"

**受挫-反击间隔表**：

| 受挫程度 | 反击间隔 | 示例 |
|---------|:------:|------|
| 轻度（被嘲讽/被轻视） | 同章或下一章 | 被嘲讽→当场打脸 |
| 中度（失去机会/被背叛） | 1-3章 | 失去机会→3章后找到更好的机会 |
| 重度（失去重要的人/物） | 3-5章 | 失去→5章后复仇，但中间有推进 |
| 极度（差点死亡/彻底失败） | 5-10章 | 本书级别的受挫→卷级反击 |

## 五、第四招：断章艺术

### 5.1 核心原理

> **不是按字数断章，不是按事件结束断章。**
> **断在"刀尖落下前一秒"、"真相大白前一句话"。**
> **在读者多巴胺分泌最旺盛的时刻戛然而止——利用"蔡格尼克效应"（未完成的事记忆更深、关注度更高），让读者追更动力拉满。**

### 5.2 断章黄金法则

**法则1：断在"即将发生"而非"已经发生"**
- ❌ 断在"他一拳把对手打飞了。"（事件已完成）
- ✅ 断在"他的拳头离对手的脸只有一寸——"（事件即将发生）
- 原理：读者知道"已经发生"会满足，读者不知道"是否发生"会焦虑→追更

**法则2：断在"知道一半"而非"全部知道"**
- ❌ 断在"原来，杀死师父的真凶就是他的亲生父亲。"（真相全揭示）
- ✅ 断在"他盯着那张泛黄的照片，照片上的人——他认识。"（知道有真相，但不知道是什么）
- 原理：给读者"线索"而非"答案"——读者会自己推理→追更

**法则3：断在"选择困境"而非"选择结果"**
- ❌ 断在"他选择了救她。"（选择已完成）
- ✅ 断在"左边是她的命，右边是整座城。他的手在发抖。"（选择正在进行）
- 原理：读者会代入自己做选择→追更

**法则4：断在"反转发出的瞬间"而非"反转完成"**
- ❌ 断在"原来他是卧底，他把情报交给了敌人。"（反转已完成）
- ✅ 断在"他把那份情报，放进了敌人的信箱。"（反转动作发出，但后果未知）
- 原理：读者会预判后果→追更

### 5.3 断章位置决策表

| 章节内的位置 | 断章策略 | 效果 |
|------------|---------|------|
| 冲突刚结束 | ❌ 不要断在这里 —— 读者满足感太强，容易放下 | 弃书风险 |
| 冲突即将爆发 | ✅ 最佳断章点 —— 读者处于"期待峰值" | 追更动力最强 |
| 真相揭示一半 | ✅ 极佳断章点 —— 利用"蔡格尼克效应" | 追更动力极强 |
| 角色面临选择 | ✅ 很好的断章点 —— 读者代入做选择 | 追更动力强 |
| 日常过渡结束 | ❌ 最差断章点 —— 读者没有情绪波动 | 容易弃书 |
| 对话说到一半 | ✅ 可用断章点 —— 但不要连续2章用此法 | 好但不要滥用 |

### 5.4 断章与章末钩子的配合

> 断章位置决定了"在哪里断"，章末钩子决定了"怎么断"。

| 断章位置 | 推荐钩子类型 | 示例 |
|---------|------------|------|
| 冲突即将爆发 | 动作钩/悬念钩 | "他的拳头停在半空——" |
| 真相揭示一半 | 信息钩/悬念钩 | "照片上的人，是他以为已经死了十年的——" |
| 角色面临选择 | 对话钩/悬念钩 | ""你选吧。"她的声音在发抖。" |
| 反转发出瞬间 | 反转钩/信息钩 | "他把那封信，塞进了自己的口袋。" |

### 5.5 断章常见的3个错误

**错误1：按字数断章**
- 表现："写到4000字了，该断章了"
- 问题：可能在平淡处断章，读者没有追更动力
- 修正：在4000字±500范围内，找到"最有悬念的瞬间"断章

**错误2：按事件结束断章**
- 表现："这件事讲完了，断章"
- 问题：读者满足感太强，容易放下书
- 修正：事件结束前，提前埋下一个事件的引子，断在引子发出时

**错误3：断在"升华/总结"处**
- 表现：章末写一段"他终于明白……"的总结
- 问题：这是AI味的标志，且让读者"心满意足"→放下书
- 修正：删除所有升华/总结，改为动作/对话/悬念收尾

## 交接协议

# 03→04→05 交接协议

> **核心定位**：解决三步分离导致的上下文断层。02→03 有衔接包做数据传递，但 03→04、04→05 之间此前没有任何交接协议——03 扩写时设计的情绪通路/锚点/口癖意图到 04 全部丢失，04 精修时打磨的金句位置/潜台词/风格指纹到 05 又丢失。
>
> **本协议定义两种交接包**：
> - **扩写意图包**（03→04）：03 扩写完成后随正文传递给 04，记录 03 的设计意图
> - **精修意图包**（04→05）：04 精修完成后随正文传递给 05，记录 04 的打磨决策
>
> **核心原则**：交接包是"设计意图的载体"，不是"施工图纸"。下游步骤读交接包是为了"理解上游为什么这么写"，从而避免推翻上游的有意设计。

## 三、交接包的传递机制

### 3.1 传递方式

交接包以 HTML 注释形式嵌入正文末尾（`<!-- ... -->`），原因：
- 不影响正文阅读
- 可被 AI 识别为结构化数据
- 落盘后可被确定性脚本检测（check-degeneration.js 已豁免注释块）

### 3.2 生命周期

```
03扩写 → 输出正文 + 扩写意图包
         ↓
04精修 → 读取扩写意图包 → 精修正文 → 删除扩写意图包 → 输出正文 + 精修意图包
         ↓
05去AI味 → 读取精修意图包 → 检测修正 → 删除精修意图包 → 输出最终正文
```

### 3.3 异常处理

| 异常情况 | 处理方式 |
|---------|---------|
| 04 收到正文但无扩写意图包 | 04 从文本反推 03 意图，在精修意图包中标注"03意图包缺失，以下为04推断" |
| 05 收到正文但无精修意图包 | 05 执行全量 11 Gate 检测（不做跳过优化），标注"04意图包缺失，全量重扫" |
| 意图包格式损坏 | 按无意图包处理，记录日志 |

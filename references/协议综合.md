## 严谨性校验协议

# 严谨性校验协议

> **核心定位**：大纲和细纲不是"随便编造"，重要角色的背景、动机、行为必须有逻辑链条，重要场景/势力必须有物理/经济/社会合理性。本协议定义"严谨性"的底线标准与校验方法。
> **痛点对症**：AI 生成角色生平时习惯"随便编一个"——"幼年被遗弃""家族被灭门""从小天赋异禀"——这些背景只是标签，没有和角色的行为模式、性格缺陷、选择倾向构成逻辑链条。AI 生成场景时同样"随便编"——沙漠城市没有水源逻辑、边境要塞没有经济支撑——场景只是舞台布景，缺乏物理和社会合理性。本协议强制在设定和细纲阶段建立"逻辑链"，确保重要内容经得起推敲。
> **适用步骤**：00（设定架构，角色生平+场景势力合理性+战力体系矩阵+数值锚点表）→ 01（大纲，场景合理性校验）→ 02（细纲，逻辑链校验+数值锁定标注）→ 03（扩写，行为一致性校验+数值一致性交叉校验+战力逻辑校验）

## 一、核心原则：重要内容必须有逻辑链

**一句话法则**：
> 不要求所有角色/场景都有严谨逻辑，但 P0 级别的角色和场景，必须能回答"为什么"——为什么这个角色是这样的人？为什么这个场景存在？背后的因果链必须成立。

**"随便编造"的定义**：
- 角色背景是孤立的标签（"幼年被遗弃"），没有推演这个标签如何塑造了角色的性格、行为、选择
- 角色动机是凭空出现的（"他要复仇"），没有从背景中推演为什么是复仇而不是别的
- 场景是舞台布景（"一座边境要塞"），没有推演这个场景的经济基础、地理逻辑、社会结构
- 势力是功能标签（"一个杀手组织"），没有推演它的经济来源、招募逻辑、势力边界

**反面案例**：
- ❌ "主角幼年被遗弃，所以性格孤僻"——标签→标签，没有推演"被遗弃"具体是怎么发生的、在什么年龄、遗弃后的生存经历如何塑造了"孤僻"的具体形态
- ❌ "边境要塞黑石关，驻军三千"——为什么能驻三千人？粮草从哪来？周边地理是否支持？没有推演
- ❌ "杀手组织清风阁，势力遍布天下"——一个杀手组织凭什么"遍布天下"？经济来源是什么？为什么不会被朝廷剿灭？没有推演

## 二、角色严谨性：生平逻辑链（P0 强制）

### 2.1 什么是"生平逻辑链"

不是让角色写自传，而是建立一条清晰的因果线：**过去发生了什么 → 角色变成了什么样的人 → 现在他为什么这样行动**。

```
【生平逻辑链 = 关键事件 → 性格塑造 → 行为模式】

幼年事件 → 形成的性格缺陷/执念 → 现在的行为模式
少年事件 → 形成的价值观/信念 → 现在的选择倾向
成年事件 → 形成的创伤/渴望 → 现在的核心动机
```

### 2.2 P0 角色生平逻辑链（强制填写）

每个 P0 角色（主角/核心反派/CP 对象）必须填写以下逻辑链，不可用模糊标签替代：

| 项目 | 要求 | 反面案例 |
|------|------|---------|
| **关键事件1（幼年）** | 具体发生了什么，在什么年龄，持续了多久，对角色产生了什么影响。必须是"具体的场景"而非"抽象标签" | ❌ "幼年被遗弃" → 太模糊。✅ "七岁那年冬天，母亲把他放在镇口的石阶上，说'在这等我'，再也没回来。他在石阶上等了两天。" |
| **性格塑造1** | 这个事件如何塑造了角色的一个性格特质？必须是"具体的表现"而非"形容词" | ❌ "变得孤僻" → ✅ "不再主动开口说话，但会观察每一个靠近他的人的手——他学会了辨别谁手里有食物" |
| **关键事件2（少年）** | 同上，具体场景 | ❌ "被师父收养" → 太模糊 |
| **性格塑造2** | 同上，具体表现 | ❌ "学会了武功" → 太功能化 |
| **关键事件3（成年/触发事件）** | 导致角色进入故事主线的事件 | ❌ "家族被灭，开始复仇" → ✅ 具体到"谁灭的、为什么、角色当时在哪、为什么没死" |
| **核心创伤/渴望** | 从上述事件中自然推演出来的，不是凭空出现的 | ❌ "渴望变强" → 太泛。✅ 从"等了母亲两天她没回来"推演出"他害怕被留下，所以要在所有人离开他之前先离开" |
| **行为模式** | 这个创伤/渴望如何体现在角色的日常行为中 | ❌ "性格孤僻" → ✅ "他每到一个新地方，第一件事是找出口。他和人保持一臂的距离，从不让人站在他身后。" |

### 2.3 生平逻辑链校验（三问法）

设计完 P0 角色的生平后，必须回答以下三个问题：

```
【三问法】

问1：因果链是否成立？
  → 从"关键事件"到"性格塑造"到"行为模式"，每一步的因果是否合理？
  → 如果一个孤儿变得"天生乐观"，需要解释——通常孤儿更可能"不信任人"，如果要写"乐观"，必须有具体的事件支撑（如"被一对善良的夫妇收养"）

问2：逻辑链是否具体？
  → "标签→标签"=不通过。必须有"具体场景→具体表现"的推演
  → 检测方法：能否用一句话描述这个角色而不使用任何抽象形容词？能=通过，不能=重写

问3：行为是否可以回溯到背景？
  → 角色现在的每一个重要行为，能否从背景中找到"为什么他会这样做"的线索？
  → 不能=生平逻辑链有缺口，补充关键事件
```

### 2.4 P1/P2 角色生平要求

P1 重要配角：至少填写 1 个关键事件 + 1 个性格塑造，不需要完整逻辑链，但必须"立得住"（背景和性格有对应关系）。

P2 次要配角：填写 1 个性格标签 + 1 个行为特征，如"贪财→随身带秤"即可，不需要完整生平。

P3 路人：不需要生平。

## 三、场景/势力严谨性：物理+经济+社会合理性（P0 强制）

### 3.1 核心场景合理性（P0 强制）

每个 P0 级别的地点（主角长期驻扎地/核心势力所在地/关键战场）必须回答以下"合理性五问"：

```
【场景合理性五问】

问1：地理逻辑 — 这个地点为什么在这里？
  → 城市为什么建在这个位置？（交通要道/资源产地/防御需求/历史遗留）
  → 周边的地形是什么？（山/水/平原/沙漠），地形如何影响这个地点的功能和形态？
  → 气候如何影响当地的生活方式？（寒冷→取暖资源/炎热→水源稀缺）

问2：经济逻辑 — 这里的人靠什么活？
  → 主要经济活动是什么？（农业/贸易/矿业/渔业/服务业）
  → 财富来源是什么？（自己能生产什么，需要从外面输入什么）
  → 贫富差距如何？阶级分布如何？

问3：资源逻辑 — 什么东西是稀缺的？
  → 这个地点最稀缺的资源是什么？（水/食物/能源/信息/安全）
  → 稀缺资源由谁控制？如何分配？
  → 稀缺资源如何驱动冲突？（如：沙漠城市的水源控制权 = 政治权力）

问4：社会逻辑 — 谁说了算？
  → 权力结构是什么？（独裁/寡头/民主/部落/宗教）
  → 社会流动可能吗？（底层能否上升？什么渠道？）
  → 主要矛盾是什么？（阶级矛盾/种族矛盾/宗教矛盾/资源矛盾）

问5：历史逻辑 — 这里以前是什么样？
  → 这个地点是怎么形成的？（自然形成/人为建造/战争产物）
  → 有什么历史事件塑造了现在的格局？
  → 有什么"历史遗留问题"可以成为剧情冲突的素材？
```

> **不是所有五问都要详写**：P0 场景至少回答前3问（地理/经济/资源），P1 场景回答前2问，P2 场景只回答第1问或不需要。

### 3.2 核心势力合理性（P0 强制）

每个 P0 级别的势力必须回答"存在逻辑"：

| 项目 | 要求 | 反面案例 |
|------|------|---------|
| **经济来源** | 这个势力靠什么维持运转？钱/资源从哪来？ | ❌ "杀手组织清风阁" → 谁付钱？为什么不被朝廷剿灭？ |
| **招募/延续逻辑** | 成员从哪来？为什么加入？如何传承？ | ❌ "神秘组织传承千年" → 怎么传承的？师徒/家族/选拔？ |
| **势力边界** | 控制范围多大？和周边势力的关系？ | ❌ "势力遍布天下" → 凭什么遍布天下？交通/通讯/管理成本？ |
| **存在理由** | 为什么这个势力没有被更强大的力量消灭？ | ❌ "因为很强" → 不够。需要具体原因（地理屏障/政治庇护/信息不对称/特殊能力） |

### 3.3 场景/势力合理性校验（四问法）

```
【场景/势力合理性四问】

问1：这个地点/势力能"自洽"吗？
  → 如果把它从故事中单独拿出来，它的物理/经济/社会逻辑是否成立？
  → 不能=有逻辑漏洞，补充设定

问2：这个地点/势力的"稀缺性"是什么？
  → 每个重要的地点/势力都应该至少有一个"稀缺资源"或"独特矛盾"
  → 稀缺性驱动冲突，没有稀缺性=没有戏剧张力

问3：这个地点/势力的"历史遗留问题"是什么？
  → 过去发生了什么导致了现在的格局？
  → 没有历史=没有深度，补充一两个关键历史事件

问4：这个地点/势力如何与主角产生关联？
  → 主角为什么来？主角在这里能得到什么/失去什么？
  → 仅为"舞台布景"=不合格，必须有功能
```

## 四、因果链校验方法（三步骤）

### 4.1 步骤一：逆推法（从行为回溯背景）

角色做了一个重要行为 → 问"为什么他会这样做" → 答案必须能从背景中找到 → 找不到=背景设计有缺口。

```
【逆推法示例】

行为：主角在关键时刻选择救一个陌生人而放弃重要目标
  ↓
为什么？→ 因为他不忍心看到无辜的人死
  ↓
为什么"不忍心"？→ 因为他自己曾经被陌生人救过
  ↓
为什么被救过？→ 背景中有"关键事件：十岁时被一个路过的铁匠从河里捞起来"
  ↓
逻辑链成立：十岁被救 → 相信陌生人的善意 → 在关键时刻选择救陌生人
```

### 4.2 步骤二：追问法（从标签到具体）

AI 说"角色性格孤僻" → 追问"怎么孤僻法？具体表现？" → 直到回答是"可被拍摄的场景"而非"形容词"。

```
【追问法示例】

AI：角色性格孤僻
  → 追问：怎么孤僻法？具体表现？
AI：不喜欢和人说话
  → 追问：具体场景？他什么时候不说话？对谁不说话？
AI：他进房间后先看有没有人，有人就退出去
  → 通过（这是可被拍摄的具体行为）
```

### 4.3 步骤三：对照法（行为与背景是否一致）

列出角色的三个关键行为 → 对照背景 → 问"从他的背景来看，他会这样做吗？" → 不一致=修改背景或修改行为。

```
【对照法示例】

背景：幼年被遗弃，在街头靠偷窃活到十五岁
行为1：他每到一个新地方先找出口 ← 一致（街头生存经验）
行为2：他从不和人分享食物 ← 一致（资源稀缺导致的囤积本能）
行为3：他在被欺负时主动道歉讨好 ← 需要解释：街头生存应该培养"反击"而非"讨好"，如果背景中是"被更大的流浪汉收编为小弟"，则"讨好"成立
  → 不一致则补充背景：补充"十二岁时被流浪汉收编，学会了讨好强者求生存"
```

|------|---------|
| **00 设定架构师** | P0角色填写"生平逻辑链"（关键事件×3+性格塑造×3+行为模式）；P0地点/势力回答"合理性五问"；产出战力体系矩阵（`power_system`）；产出数值锚点表（`number_anchors`）；产出严谨性基线 |

## 六、战力体系一致性（V3.5新增 — 防止战力崩坏）

> **痛点对症**：AI生成战斗场景时，角色战力表现与设定等级完全脱节——2枚血核的低等级角色一刀秒杀一阶上等荒兽，7枚血核的高等级角色只能扛住一阶荒兽的撞击。没有战力矩阵，"强弱"全凭AI即兴发挥，导致读者感觉"战力崩坏"。

### 6.1 核心原则：战力必须有锚定

**一句话法则**：
> 每个角色的战力表现必须与其在力量体系矩阵中的当前等级严格对应。越级表现必须有设定层面的合理解释。

**"战力崩坏"的定义**：
- 角色的战斗表现超出其等级的能力上限，且无设定依据（金手指/特殊道具/合击/牺牲代价）
- 高等级角色的战力表现低于低等级角色，且无合理原因（受伤/中毒/被压制）
- 同一等级的角色在不同章节中表现忽强忽弱，无合理波动逻辑
- 旁白描述的"危险等级"与实际战斗难度不匹配（旁白说"一阶货能要人命"，但主角2枚就轻松秒杀一阶偏上）

### 6.2 战力体系矩阵设计要求（P0强制）

在00设定架构师阶段，必须产出完整的战力等级表（写入JSON的 `power_system` 字段）：

```
【战力矩阵 = 等级 → 上限 → 可杀 → 可挑战 → 能力 → 边界】

骨骼/资质类型 → 修炼上限（枚数/境界）
             → 可独杀敌人等级（该等级单独能打赢的最高敌人）
             → 组队可挑战等级（多人合作能打赢的最高敌人）
             → 典型能力（该等级能做什么）
             → 能力边界（该等级不能做什么 — 最重要！）
             → 体征特征（该等级的外观标记）
```

**设计规则**：
1. **能力边界是核心**：每个等级必须明确"不能做什么"，而非只写"能做什么"。只有"不能做什么"才能防止越级
2. **等级之间必须有明确差距**：相邻等级的能力差距必须可感知（凡骨3枚上限 vs 硬骨5枚上限，差2枚=差距明显）
3. **主角的位置必须标注**：主角当前在矩阵中的等级和修炼进度必须明确
4. **越级机制必须预设**：如果允许越级（主角越级杀敌是爽点），必须在设定中预设越级条件（金手指/特殊道具/合击/牺牲代价）

### 6.3 战力崩坏检测（三问法）

```
【战力三问法】

问1：角色的表现是否在其等级的能力边界内？
  → 列出本章所有战斗/能力展示场景
  → 找到该角色在 power_system 矩阵中的当前等级
  → 对照该等级的 ability_ceiling（能力边界）
  → 表现超出边界 → 进入问2

问2：如果超出边界，是否有设定依据？
  → 金手指？ → 金手指的使用条件和代价是否已交代？
  → 特殊道具？ → 道具的来源和限制是否已交代？
  → 合击/组队？ → 合击的逻辑是否合理？
  → 牺牲代价？ → 代价是否真实付出？
  → 无依据 → 战力崩坏，必须修正

问3：旁白描述的危险等级与实际战斗难度是否匹配？
  → 旁白说"一阶货能要人命" → 实际战斗中一阶货是否真的危险？
  → 旁白说"最近荒兽越来越凶" → 实际战斗中荒兽是否比之前更强？
  → 不匹配 → 修正旁白或修正战斗描写
```

### 6.4 战力波动规则

角色战力允许波动，但波动必须有逻辑：

| 波动方向 | 合理原因 | 不合理原因 |
|---------|---------|-----------|
| 战力下降 | 受伤/中毒/消耗/心理崩溃/被压制 | AI忘了角色之前很强 |
| 战力上升 | 突破/觉醒/道具加持/愤怒爆发 | AI忘了角色之前很弱 |
| 忽强忽弱 | 有明确的波动触发条件 | 无规律随机波动 |

## 七、数值一致性（V3.5新增 — 防止数值矛盾）

> **痛点对症**：AI生成正文时，数字前后不一致——角色年龄一会35岁一会43岁，地点距离一会3里一会9里，妖丹大小一会拳头大一会鸡蛋大。没有数值追踪表，所有数字都是AI即兴编的，必然矛盾。

### 7.1 核心原则：关键数值必须锚定

**一句话法则**：
> 全书所有关键数值（年龄/距离/枚数/境界/物品数量/时间线/生死状态）必须在00设定阶段锁定，形成数值锚点表（`number_anchors`）。下游所有步骤的数值必须与此表一致。

**"数值矛盾"的定义**：
- 同一角色的年龄在不同章节中不一致（第1章19岁，第3章突然变成21岁，且无时间跳跃）
- 同一地点间的距离在不同章节中不一致（第2章"南3里"，第5章"南9里"）
- 同一物品的数量/大小在不同章节中不一致（第1章"拳头大小"，第3章"比鸡蛋略大"）
- 同一角色的修炼枚数在不同章节中不一致（第1章"三枚够用"，第3章"他种下第二枚"）
- 已死亡的角色以活人状态出现（闪回中35岁死亡，当前43岁还活着）

### 7.2 数值锚点表设计要求（P0强制）

在00设定架构师阶段，必须锁定以下六类数值（写入JSON的 `number_anchors` 字段）：

| 数值类别 | 锁定内容 | 校验时机 |
|---------|---------|---------|
| **角色年龄** | 每个P0角色在故事开始时的年龄 + 关键事件的年龄 | 02衔接包每章标注；03扩写自检 |
| **地理距离** | 主要地点之间的距离和方向 | 02衔接包标注涉及移动的章节；03扩写自检 |
| **修炼计数** | 每个角色的修炼枚数/境界等级/突破时间点 | 02衔接包每章标注；03扩写自检 |
| **时间线** | 故事内时间流速 + 关键历史事件的时间锚点 | 02每5章回顾；03扩写自检 |
| **物品/人数** | 关键物品数量、势力人数、队伍人数 | 02衔接包标注涉及物品/人数的章节；03扩写自检 |
| **生死状态** | 每个角色的存活/死亡状态 + 死亡时间 + 死因 | 02衔接包每章标注；03扩写自检 |

### 7.3 数值矛盾检测（四问法）

```
【数值四问法】

问1：本章出现的数值是否与 number_anchors 一致？
  → 扫描本章所有数值（"X岁""X里""X枚""X个""X天"）
  → 逐条对照 number_anchors 中的对应字段
  → 不一致 → 立即修正正文

问2：同一数值在不同章节中是否一致？
  → 本章说"南3里" → 前文是否说过"南9里"？
  → 本章说"拳头大小" → 前文是否说过"鸡蛋大小"？
  → 不一致 → 以 number_anchors 为准，修正正文

问3：时间线是否自洽？
  → 角色年龄与故事时间是否匹配？（开始19岁，过了2年应该是21岁）
  → 历史事件的时间锚点是否与当前时间线矛盾？（3年前种血核，当前19岁，则16岁种血核）
  → 矛盾 → 修正正文

问4：已死亡角色是否以活人状态出现？
  → number_anchors 中 status 为"已死亡"的角色
  → 是否在当前时间线的正文中以活人状态出现？
  → 是（且非闪回/回忆/幻觉） → 严重矛盾，必须修正
  → 是（闪回/回忆/幻觉） → 必须明确标注为闪回/回忆/幻觉
```

### 7.4 数值锁定与衔接包的配合

02细纲在每章衔接包中增加 `[数值锁定]` 字段，标注本章涉及的关键数值：

```
【数值锁定 — 本章涉及的关键数值】
├─ 沙九：19岁，2枚血核（染煞·二染），凡骨（待定）
├─ 铁翎：42岁，4枚血核，硬骨中等
├─ 老岩：已死亡（35岁第9枚崩脉，被铁翎处决）← ⚠️本章为闪回，须标注
├─ 水潭→杂木林：南3里
├─ 水潭→营地：东4里
└─ 妖丹大小：拳头大小（一阶偏上青背狸）
```

03扩写时对照此字段，确保正文中出现的数值与锁定值一致。

## 伏笔全生命周期管理

# 伏笔全生命周期管理系统

> **核心定位**：解决百万字长篇中 100-300+ 个伏笔的追踪问题。当前 02 衔接包的"记忆锚点 P0/P1/P2"是状态快照，不是伏笔追踪——没有伏笔 ID、没有生命周期、没有回收检测、没有过期告警。
>
> **痛点对症**：30 章以内靠人脑还能记住伏笔，百万字长篇中"第 800 章忘了第 3 章的伏笔"是必然发生的。业界最佳实践是全生命周期管理：埋设 PLANT → 标记 TAG → 追踪 TRACK → 回收 REAP → 归档。
>
> **理论依据**：每条伏笔分配唯一 ID（如 FS-047），记录埋设章节/角色/类型/预计回收章节/关联伏笔。含轮换间隔（避免连续N章提同一伏笔）、回收标记（防重复回收）、过期告警。

---

## 一、伏笔五态生命周期

```
【伏笔五态生命周期】

State 1: PLANT（埋设）
├─ 触发：02细纲设计时创建伏笔条目
├─ 记录：ID/埋设章/角色/类型/预计回收章/关联伏笔/重要度
└─ 状态：待推进

State 2: ADVANCE（推进）
├─ 触发：后续章节中提及或部分揭示该伏笔
├─ 记录：推进章/推进方式（提及/暗示/部分揭示）
├─ 轮换控制：同一条伏笔不能连续N章提及（避免水字数感）
└─ 状态：推进中

State 3: HINT（暗示）
├─ 触发：给出更明确的线索但不完全揭示
├─ 记录：暗示章/暗示内容
└─ 状态：即将回收

State 4: REAP（回收）
├─ 触发：伏笔真相揭示/兑现
├─ 记录：回收章/回收方式/是否满足读者预期
├─ 防重复：回收后标记为已回收，后续不得再次回收
└─ 状态：已回收

State 5: ARCHIVE（归档）
├─ 触发：回收后经过一定章节数（默认10章）
├─ 记录：归档原因（已回收/已废弃/已合并）
└─ 状态：已归档（不再追踪，但保留记录供查阅）
```

## 三、伏笔检测规则

### 3.1 轮换间隔检测

```
规则：同一条伏笔的连续两次提及之间，间隔不得小于"轮换间隔"值。
违反：标记"伏笔轮换过密"——读者会感觉在水字数。
例外：回收章前1-2章的密集暗示不受此限（蓄势期允许密集提及）。
```

### 3.2 过期告警

```
规则：
- P0级伏笔：超过预计回收章10章未回收 → 告警"P0伏笔过期"
- P1级伏笔：超过预计回收章20章未回收 → 告警"P1伏笔过期"
- P2级伏笔：超过预计回收章30章未回收 → 提示"P2伏笔可考虑废弃"

处理：
- 告警后02细纲需决定：延期回收（更新预计回收章）/ 废弃（标记归档）/ 合并到其他伏笔
- 不得忽略告警——忽略=读者会觉得"挖坑不填"
```

### 3.3 重复回收检测

```
规则：已标记为"已回收"的伏笔，后续章节不得再次回收。
违反：标记"伏笔重复回收"——这会导致逻辑矛盾或读者困惑。
例外：可以"二次揭示"（同一伏笔的更深层次真相），但必须创建新的伏笔ID关联到原伏笔。
```

### 3.4 回收满意度检测

```
规则：伏笔回收时，须检测"是否满足读者预期"。
检测项：
- 回收方式是否与埋设时的暗示方向一致（不一致=反转，需额外铺垫）
- 回收时机是否合理（太早=缺乏期待感，太晚=读者已忘记）
- 回收力度是否匹配重要度（P0伏笔回收需大高潮，P2伏笔一笔带过即可）
```

## 四、02 细纲集成点

### 4.1 02 细纲编写时

每卷细纲须包含本卷的伏笔追踪表：
- 新埋设的伏笔：创建新条目，状态=待推进

## 六、速查卡

```
【伏笔管理速查卡 — 02细纲编写时贴在屏幕上】

每埋一个伏笔，问自己4个问题：
① 这个伏笔什么时候回收？（不填预计回收章=挖坑不填）
② 这个伏笔的轮换间隔是多少？（连续提及=水字数）
③ 这个伏笔和哪些伏笔关联？（孤立的伏笔=浪费篇幅）

```

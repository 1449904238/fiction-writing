# 情绪工程协议

> **核心定位**：本文件是03扩写系统的第二台"油门"——与"质感注入协议"（质感油门）并行执行。
> 质感注入协议管"读者能不能看见画面"，本协议管"读者读完身体有没有反应"。
>
> **痛点对症**：当前skill在情绪处理上只有刹车（03铁律4情绪冰山法则、04冰山原则、05 Gate3心理外化），没有油门。
> 刹车管"不该怎么写"（不写"他很悲伤"），油门管"该怎么设计"（如何让读者自己感到悲伤）。
> AI写不出情绪的根本原因不是"AI没有感情"，而是AI在**模拟情绪表达**而非**工程化情绪转移**。
>
> **理论依据**：情绪冲击力 = 预期反差 × 合理度。越出乎意料越难忘，但必须提前埋伏笔让转折合理。
> 网文情绪不是文学性抒情，是精准设计的心理操控。

---

## 一、核心理论：模拟情绪表达 vs 工程化情绪转移

### 1.1 AI写小说"没有情绪"的真正原因

**错误诊断**：AI没有感情，所以写不出情绪。
**正确诊断**：AI在**模拟情绪的表达形式**，而非**工程化情绪的转移路径**。

| 维度 | 模拟情绪表达（AI的默认行为） | 工程化情绪转移（人类高手的做法） |
|------|---------------------------|------------------------------|
| **目标** | 让文字"看起来有情绪" | 让读者"身体产生反应" |
| **手段** | 堆砌情绪词（"悲伤""愤怒""心碎"） | 设计事件链+物理细节，触发读者自身情绪 |
| **检验** | 文字本身是否"感人" | 读者读完后是否"有身体反应"（胸闷/握拳/鼻酸） |
| **类比** | 画一个笑脸 emoji | 讲一个让听众自己笑出来的故事 |

**一句话法则**：
> 小说不是作者把情绪"倒"给读者，而是作者在读者心里"种"一个触发器，让读者自己长出情绪。
> AI没有情绪不要紧——导演演电影时也不必真的在哭，他只需要把镜头、节奏、音乐安排好，观众自己会哭。

### 1.2 情绪转移三层架构

```
【情绪转移三层架构】

Layer 1：情绪蓝图（设计层——在01大纲/02细纲完成）
├─ 本章目标读者情绪是什么？（爽/虐/燃/酸/惧/暖）
├─ 情绪曲线走向？（压抑→爆发 / 平静→惊惧 / 温暖→心酸）
└─ 情绪公式检验：预期反差×合理度是否足够？

Layer 2：情绪锚点（物质层——在03扩写完成）
├─ 用什么物理物件/动作承载这个情绪？（茶杯/鞋尖/锈迹/雨声）
├─ 情绪锚点是否"小而具体"？（不是"夕阳西下"而是"他鞋尖上沾了一片银杏叶"）
└─ 锚点是否有"这个角色独有"的印记？（修理工看到的锈 vs 普通人看到的锈）

Layer 3：情绪通路（执行层——在03扩写+04精修完成）
├─ 通路A：压抑→爆发反差工程（爽点三段式）
├─ 通路B：五感榨汁机法（具体动作/气味/场景代替情绪词）
└─ 通路C：配角杠杆法（配角作情绪放大器）
```

### 1.3 与现有铁律的关系

| 现有规则 | 定位 | 与本协议的关系 |
|---------|------|--------------|
| 03铁律4 情绪冰山法则 | 刹车：禁止情绪词 | 本协议通路B的执行手段——冰山法则是"不怎么说"，本协议是"那该怎么说" |
| 03质感注入协议 | 质感油门：让画面可感知 | 本协议的物质基础——锚点必须是有质感的具体物 |

## 二、Layer 1：情绪蓝图设计法（01大纲/02细纲层）

### 2.1 情绪公式

```
情绪冲击力 = 预期反差 × 合理度
```

- **预期反差**：读者预期的 vs 实际发生的差距。差距越大，冲击越强。
- **合理度**：反转是否有伏笔支撑。无伏笔的反转 = 突兀，有伏笔的反转 = 震撼。

**设计检验**：
- 反差大但无伏笔 → 突兀，读者觉得"强行"
- 反差小但有伏笔 → 平淡，读者觉得"没意思"
- 反差大且有伏笔 → 震撼，读者觉得"后劲太大"

### 2.2 章级情绪蓝图模板

每章细纲须明确以下情绪设计（02细纲编写时填写）：

```
【章级情绪蓝图】
目标读者情绪：[爽/虐/燃/酸/惧/暖/怒/惊]（选1个为主，最多2个叠加）
情绪曲线：[起点情绪] → [终点情绪]（如：压抑→爆发 / 平静→惊惧）
情绪公式检验：
  预期反差：读者预期[XXX]，实际发生[XXX]，反差强度[高/中/低]
  合理度伏笔：[列出本章情绪反转所依赖的前文伏笔，若无=需补埋]
情绪锚点（至少1个）：[承载本章核心情绪的物理物件/动作]
情绪通路（选1主+1辅）：
  主通路：[A压抑爆发 / B五感榨汁 / C配角杠杆]
  辅通路：[A/B/C]
```

### 2.3 情绪曲线与番茄算法的绑定

番茄以完读率/追更率为核心指标，情绪曲线设计直接影响留存：

| 章节位置 | 情绪设计要求 | 算法依据 |
|---------|-------------|---------|
| 首章 | 前500字必须有1个情绪冲击点（冲突/危机/反转） | 3秒停留率，首章完读率决定推荐 |
| 章中 | 每2000字至少1个情绪波动（小爽点/小悬念/小反转） | 维持追更动力 |
| 章末 | 必须留情绪钩子（信息差/危机预告/选择困境） | 追更率≥40%获5倍流量 |
| 前3章 | 情绪强度逐章递增，第3章小高潮 | 前三章完读率<10%终止推荐 |
| 每5章 | 至少1个大情绪爆发点（大爽点/大反转/大揭秘） | 维持整体完读率 |

## 三、Layer 2：情绪锚点系统（03扩写层）

### 3.1 什么是情绪锚点

**定义**：承载抽象情绪的物理物件/动作。读者通过锚点"抓住"情绪，而非通过情绪词"被告知"情绪。

| 情绪 | ❌ 情绪词直给 | ✅ 情绪锚点 |
|------|------------|-----------|
| 伤心 | "他很伤心" | 他把两个杯子都倒了茶，过了一会儿，把那杯倒回壶里 |
| 愤怒 | "他怒不可遏" | 他把筷子齐齐折断，放在碗边，站起来走了 |
| 紧张 | "她很紧张" | 她把杯子端起来又放下，放下又端起来，指尖在杯壁上敲了两下 |
| 委屈 | "他满腹委屈" | 他蹲在楼梯口，把鞋带拆开又系上，系上又拆开 |
| 心酸 | "她心里发酸" | 她把那件洗到发白的蓝布外套叠好，放进柜子最底层，关门时手指卡了一下 |
| 不舍 | "他很不舍" | 他走到门口又回来，把桌上的橘子揣了一个在兜里 |

### 3.2 情绪锚点设计四原则

**原则1：小而具体**
- 锚点必须是"小动作"或"小物件"，不是"大场面"
- ❌ "夕阳西下，他孤独地站在山顶"（大场面，空）
- ✅ "他鞋尖上沾了一片银杏叶，他没掸"（小物件，准）

**原则2：角色独有**
- 同一情绪，不同角色的锚点不同（由职业/性格/过往决定）
- 修理工的"心酸"：他看到那台修了三年的洗衣机，电机还是烧了
- 厨师的"心酸"：她把那锅熬了四小时的汤倒进了下水道
- 孩子的"心酸"：他把弹珠从口袋里掏出来，又放回去

**原则3：日常受阻**
- 情绪越强烈，描写越日常——用"极其日常的动作受阻"表现崩溃
- 亲人死去 → 不要写"泪流满面"，写"他试图把碎掉的杯子拼起来，拼了五分钟"
- 被背叛 → 不要写"撕心裂肺"，写"他盯着那把刀——那把刀还是他送的。他继续吃饭"

**原则4：不解释**
- 锚点只呈现物理事实，绝不解释"为什么"——让读者自己悟
- ❌ "他把茶倒掉，因为那个人不会再来了"（解释了）
- ✅ "他把两个杯子都倒了茶，过了一会儿，把那杯倒回壶里"（不解释）

### 3.3 情绪锚点密度标准

| 场景类型 | 情绪锚点要求 | 占比 |
|---------|-------------|------|
| 日常过渡 | 0-1个（不强制） | 30-40% |
| 对话场景 | 1个（对话中穿插1个情绪动作） | 20-30% |
| 情绪蓄力场景 | 2-3个（逐步堆叠情绪锚点） | 15-20% |
| 情绪爆发场景 | 1个极简锚点（越爆发越极简） | 5-10% |
| 爽点场景 | 1个"前后反差"锚点（前面压抑的物，后面翻盘的物） | 10-15% |

> **关键原则**：情绪爆发场景的锚点要"极简"——越是高潮，越用一个"小而准"的物理画面收束。
> 不要在高潮时堆砌情绪词，而要用一个"安静的动作"收束所有情绪。

## 四、Layer 3：情绪通路三法（03扩写+04精修层）

### 通路A：压抑→爆发反差工程（爽点三段式）

**核心公式**：
```
第一层 铺垫+憋屈 → 第二层 期待+反转 → 第三层 爆发+宣泄
```

**三层缺一不可**：
- 只爆发不铺垫 → 爽感来得快去得快
- 只铺垫不爆发 → 读者想骂人
- 三层齐全 → 爽感有"后劲"

**节奏把控**：
1. 爆发须**分批释放**（小爆发→中爆发→大爆发），每次爆发后埋新期待
2. 第二层核心是"给希望但不一次性给够"——制造"知道要发生但不知何时发生"的追更动力
3. 精髓是**加大前后反差**——前面摁得越狠，后面翻身越爽

**操作模板**：
```
【通路A操作模板——以"打脸"场景为例】

第一层（铺垫+憋屈）：
├─ 建立读者对主角的共情（主角被羞辱/被误解/被看不起）
├─ 具体憋屈点≥3个（退婚/下人讥讽/家族测试垫底——逐层加码）
└─ 情绪锚点：主角的"憋屈物"（如被退婚的纸/被扔掉的信物）

第二层（期待+反转）：
├─ 给一线希望（金手指出现/暗中蓄力/贵人注意到）
├─ 希望"给但不给够"（修复了一点点斗气/打赢了一个小喽啰）
└─ 情绪锚点：主角的"希望物"（如药老的出现/第一次发光的戒指）

第三层（爆发+宣泄）：
├─ 分批爆发（小打脸→中打脸→大打脸）
├─ 每次爆发后埋新期待（更大的敌人在看着）
└─ 情绪锚点：主角的"翻盘物"（如打脸纳兰嫣然的那一掌/前文憋屈物的翻转）
```

**经典案例**：《斗破苍穹》
- 第一层：天才变废柴 + 退婚羞辱 + 下人讥讽（极致打压）
- 第二层：药老出现 → 一点点修复斗气 → 家族测试 → 云岚宗退婚（分批期待）
- 第三层：打脸纳兰嫣然 → 成斗帝（分批爆发）

### 通路B：五感榨汁机法

**核心原理**：不写情绪词，写"身体在情绪中的具体反应"——越具体的感官越让人身临其境。

**五感榨汁机操作表**：

| 情绪 | 视觉 | 听觉 | 触觉 | 嗅觉/味觉 | 动作链 |
|------|------|------|------|----------|--------|
| 愤怒 | 眼睛充血发红 | 牙齿咬咯咯响 | 拳头砸墙骨节裂开 | 后槽牙咬紧尝到铁锈味 | 砸墙→攥拳→太阳穴跳 |
| 悲伤 | 眼眶发热但没流泪 | 水龙头开到最大盖住嚎啕 | 蹲到腿麻没感觉 | 擤鼻涕发现纸巾用完 | 蹲浴室→开水龙头→用袖子擦脸 |
| 恐惧 | 瞳孔缩成针尖 | 耳朵里嗡嗡响 | 后脖颈汗毛竖起 | 嘴里发苦发干 | 后退→撞墙→手摸到门把手 |
| 紧张 | 视线模糊发花 | 心跳声盖过环境音 | 手心出汗发黏 | 嘴唇发干舔了又舔 | 端杯→放下→端起→敲杯壁 |
| 焦虑 | 反复看同一个地方 | 任何声音都吓一跳 | 手指无意识抠指甲 | 胃里翻涌想吐 | 抠指甲→站起来→坐下→又站起来 |

**执行要点**：
1. 每次"榨汁"选2-3个感官维度，不要全上（全上=轰炸）
2. 日常场景用1个感官锚定，核心场景用3-5个叠加
3. **反差法**：表面与内心相反时冲击力更强——强装镇定但手在抖、口是心非但攥紧了东西

**与03铁律4（情绪冰山法则）的关系**：
- 冰山法则说"不写情绪词"——这是刹车
- 五感榨汁机说"写身体在情绪中的反应"——这是油门
- 两者合一：删掉情绪词，替换为五感榨汁机的身体反应

### 通路C：配角杠杆法

**核心原理**：配角是"情绪放大器"——同样的事件，通过配角的反应放大读者情绪。

**杠杆操作表**：

| 杠杆类型 | 操作方式 | 情绪放大效果 |
|---------|---------|------------|
| 身份杠杆 | 小师弟当众扇耳光 > 敌人骂百句 | 羞辱感×3（自己人背叛更扎心） |
| 信任杠杆 | 门派千金塞毒药 > 老天开眼更冷 | 背叛感×3（信任的人下手更冷） |
| 弱者杠杆 | 孩子说"爸爸不要走" > 女友说"别走" | 心酸感×3（弱者的无辜更戳心） |
| 强者杠杆 | 宗师叹了口气 > 主角自己哭 | 震撼感×3（强者都扛不住=事很大） |
| 旁观杠杆 | 路人甲吓得碗掉了 > 写"他很可怕" | 恐惧感×3（第三方反应=客观证据） |

**执行要点**：
1. 每个重要情绪场景至少用1个配角杠杆
2. 杠杆角色要有"功能定位"——不能是路人甲，必须是读者认识的配角
3. 杠杆反应要"小而具体"——不是"他震惊了"，而是"他手里的碗掉了，没碎，汤洒了一地"

## 五、情绪转移校验（03自检+04精修+05终检）

### 5.1 读者身体反应测试（核心校验）

```
【情绪转移校验清单 — 与脱水自检/质感自检并行执行】

1. 目标情绪明确性：本章的目标读者情绪是什么？（爽/虐/燃/酸/惧/暖）
   → 答不出 = 情绪设计缺失，回02补情绪蓝图

2. 情绪公式检验：
   → 预期反差：读者预期[XXX]，实际发生[XXX]，反差够不够大？
   → 合理度：情绪反转有没有前文伏笔支撑？无伏笔=突兀，需补埋

3. 情绪锚点检查：
   → 本章至少有1个"小而具体"的物理锚点承载核心情绪？
   → 锚点是否"角色独有"？（同一个情绪，这个角色的锚点换别的角色不成立）
   → 锚点是否"不解释"？（有没有画蛇添足地解释"为什么"）

4. 情绪通路检查：
   → 若用通路A（压抑爆发）：三层是否齐全？前面摁够了吗？后面翻够了吗？
   → 若用通路B（五感榨汁）：是否选了2-3个感官维度？有没有退回情绪词？
   → 若用通路C（配角杠杆）：至少1个配角作放大器？杠杆反应是否"小而具体"？

5. 读者身体反应测试（终极校验）：
   → 读完本章后，读者的身体会有什么反应？（握拳/鼻酸/心跳加速/胸闷/嘴角上扬）
   → 答不出 = 情绪转移失败，情绪没到达读者
   → 如果只是"读起来很流畅"但没有身体反应 = 情绪空心化（AI最常见的问题）

6. 情绪词残留扫描：
   → 全章是否有直接情绪词？（"悲伤/愤怒/绝望/心碎/痛苦/震惊/恐惧"等）
   → 出现在叙述语中 = 删除，替换为情绪锚点/五感榨汁/配角杠杆
   → 出现在角色对话中 = 可保留（人在说话时确实会说"我好难过"）

7. 压抑爆发反差检查（若本章含爽点）：
   → 前面压抑的"锚点物"在爆发后是否"翻转"？
   → 示例：前面被退婚的纸→后面打脸时撕碎同一张纸
   → 无翻转 = 反差不够，爽感减半
```

### 5.2 情绪空心化检测（AI最隐蔽的缺陷）

**什么是情绪空心化**：文字"看起来有情绪"（用了感叹号/排比/抒情句），但读者"没有身体反应"。这是AI最隐蔽的缺陷——它在**模拟情绪表达的形式**，而非**工程化情绪转移**。

**检测信号**：
- 连续3句以上使用情绪词（"他愤怒/他悲伤/他不甘"）
- 用排比句"升华"情绪（"他恨，他悔，他不甘"）
- 用感叹号制造情绪（"！"超过3个/段）
- 用省略号制造"余韵"（"……"收尾超过2处）
- 情绪"被宣告"而非"被触发"（叙述者说"这一刻所有人都震撼了"）

**修正方法**：
- 删除所有情绪词和升华句
- 替换为1个"小而具体"的情绪锚点
- 用配角杠杆放大（"旁边的人把碗放下了，汤洒了一地"）

## 七、速查卡

```
【情绪工程速查卡 — 扩写/精修时贴在屏幕上】

写情绪前问自己3个问题：
① 读者的身体会有什么反应？（不是"读者会觉得感人"，是"握拳/鼻酸/心跳加速"）
② 用什么物理物件/动作承载这个情绪？（不是"他很伤心"，是"他把茶倒掉了"）
③ 这个物件是"这个角色独有"的吗？（修理工的伤心≠厨师的伤心≠孩子的伤心）

写爽点前问自己3个问题：
① 前面摁够了吗？（3个以上憋屈点逐层加码）
② 后面翻够了吗？（分批爆发，不是一次性碾压）
③ 前面的"憋屈物"在后面"翻转"了吗？（被退婚的纸→被撕碎的纸）

写对话前问自己1个问题：
① 这句话删掉情绪词，换成动作/表情，读者还能读出情绪吗？
   能 → 删掉情绪词
   不能 → 动作/表情不够具体，换一个"小而准"的
```

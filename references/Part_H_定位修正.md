## H.1 问题：细纲变成了对话脚本

**现象**：生成的细纲像"对话脚本"，而非"情节蓝图"。

**原因**：
1. 文档对"核心场景"的定位不够清晰——AI误以为细纲就是写对话
2. v3.0的"对话潜台词"模块可能被过度强调，导致AI把重心放在对话上
3. 缺少"细纲 vs 正文"的明确区分——细纲应该描述"发生什么"，正文才写具体对话

## H.2 细纲的本质定位

```
[细纲定位 - 必须明确]

细纲是什么：
├─ 细纲是"蓝图"，不是"正文"
├─ 细纲描述"发生什么"，不写出具体对话
├─ 细纲标注"情绪/氛围/关键动作"，不写完整场景
└─ 细纲的读者是"扩写者"（可能是AI或人类）

细纲不是什么：
├─ ❌ 细纲不是对话脚本（不写出完整对话）
├─ ❌ 细纲不是正文缩写（不写完整场景描写）
├─ ❌ 细纲不是大纲（比大纲更具体，但不如正文详细）
└─ ❌ 细纲不是填空题（不是机械填充模板）
```

## H.3 细纲 vs 大纲 vs 正文

| 维度 | 大纲 | 细纲 | 正文 |
|------|------|------|------|
| 详细度 | 一句话概括 | 场景级描述 | 完整描写 |
| 对话 | 无 | 关键台词提示 | 完整对话 |
| 场景 | 无 | 场景概括+情绪 | 完整场景 |
| 字数 | 50字/章 | 500-800字/章 | 2000-5000字/章 |
| 读者 | 编辑/作者 | 扩写者 | 终端读者 |

## H.4 核心场景的正确写法

```
[核心场景正确写法]

必须包含：
├─ 场景概括：一句话描述"发生什么"
├─ 涉及角色：列出参与角色
├─ 情绪基调：标注场景的情绪
├─ 关键动作/转折点：列出关键动作
├─ 关键台词提示：给出1-2句台词的提示（不是完整对话）
└─ 后续影响：说明这个场景的后果

禁止写法：
├─ ❌ 写出完整对话（这是正文的工作）
├─ ❌ 写出完整场景描写（这是正文的工作）
├─ ❌ 只写一句话（这是大纲，不是细纲）
└─ ❌ 机械填充模板（要有实际内容）
```

### 细纲正反例
- ❌ **错误（对话脚本式）**：写出完整对话 + 无场景概括 + 无情绪标注 + 无关键动作
- ✅ **正确（蓝图式）**：场景概括（1句）+ 情绪基调 + 关键动作列表 + 台词提示（非完整对话）+ 后续影响

## H.6 细纲生成约束Prompt

```
[细纲生成约束 - 必须遵守]

细纲是"蓝图"，不是"正文"。请遵循以下规则：

1. 核心场景：
   - 用"场景概括"描述发生什么（一句话）
   - 标注"情绪基调"
   - 列出"关键动作/转折点"
   - 给出"关键台词提示"（不是完整对话，是提示）
   
2. 禁止：
   - ❌ 写出完整对话（最多给出1-2句台词提示）
   - ❌ 写出完整场景描写
   - ❌ 把细纲写成对话脚本
   - ❌ 只写一句话（那是大纲）

3. 字数要求：
   - 每章细纲：500-800字（不含元数据），黄金3章可达1000字
   - 每个核心场景：50-100字
   
4. 示例：
   ✅ "李四质问张三，张三最终说出真相，李四震惊"
   ❌ "李四：'你怎么来了？' 张三：'我有事告诉你。'..."
```

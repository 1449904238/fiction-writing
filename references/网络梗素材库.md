# 网络梗素材库

> **版本**：V1.0.0
> **创建日期**：2026-07-18
> **定位**：独立网络梗素材库，支持 02a/03a/04/05 按需加载。从 `语言素材综合库.md` 的"自嘲/网络新语类"与"情绪表达类"中提取网络梗部分，扩充并独立为结构化素材库，专供网文创作提升"人味/网感"。
> **核心约束**：网络梗只能用于**角色对话 / 内心吐槽 / 围观反应 / 底层角色插科打诨**，禁止进入叙述者总结层。违反此约束 = 05 Gate 阻断。

## 一、时效分级体系（S/A/B/C）

> 基于 2026-07 时间点判断。网络梗生命周期短，需定期复核并替换过期条目。

| 等级 | 名称 | 生命周期 | 使用建议 | 复核周期 |
|------|------|---------|---------|---------|
| S | 经典化 | 2年+ | 可稳定使用，已进入日常词汇，全卷可用 | 每年复核1次 |
| A | 阶段流行 | 6-24月 | 短期有效，跨卷慎用，注意过时风险 | 每季度复核1次 |
| B | 热点 | 1-6月 | 仅限短期连载，长连载（>50章）慎用 | 每月复核1次 |
| C | 即时 | 1-3月 | 仅适合实验性或短篇幅，过期立即替换 | 每月复核1次 |

> **升级规则**：A级持续活跃24月+ → 评估升S；B级持续活跃6月+ → 评估升A；C级持续活跃3月+ → 评估升B。
> **降级规则**：任何等级超过2个复核周期未在平台活跃 → 降级或移除。
> **示例**："摆烂"（2022年流行）截至2026-07已活跃48月，应从A级升S级；"CPU"（PUA谐音梗）若3个月内无新热度，则降级移除。

## 二、题材路由表

| 题材 | 网络梗策略 | 解锁等级 | 典型用法 |
|------|----------|---------|---------|
| 都市/职场/系统 | 解锁，主角可吐槽自嘲 | S+A+B | 内心吐槽为主，对话为辅 |
| 玄幻/修仙 | 中度，限金手指/旁观者反应 | S+A | 旁观者围观吐槽，主角慎用 |
| 言情/情感 | 克制，限女性角色表达 | S | 女主内心吐槽，少量对话 |
| 悬疑/恐怖 | 封锁，仅底层角色偶尔吐槽 | S（极少量）| 市井角色插科打诨，≤1处/章 |
| 历史/严肃 | 封锁 | 禁用 | — |
| 轻小说/二次元 | 解锁，社区梗/ACG术语 | S+A+B+C | 对话/围观/章末钩子均可 |

> **题材混合**：双题材取严格侧。如"都市+悬疑"→按悬疑封锁处理。
> **穿越/重生设定**：穿越者内心独白可解锁S+A（现代梗），外在对话必须符合架空世界。
> **系统流例外**：系统本身可作为"网络梗出口"——系统提示音/弹窗可使用S级梗，不受角色声口约束。
> **金手指/面板**：系统面板的吐槽文案可使用S+A级梗，作为"第四面墙"的轻松调剂。

## 三、场景分类与使用规范

| 场景 | 规范 | 示例 |
|------|------|------|
| 内心吐槽 | 第一人称独白，语气像真实对话，不可书面化 | "行吧，这下社死了。" |
| 角色对话 | 角色直接说，必须符合人物声口（查`人物声口对话风格库.md`） | "就这？没意思。" |
| 围观群众反应 | 群体吐槽，用"有人""旁边的人"引出，≤2句 | "旁边有人小声嘀咕：'这也太绝了。'" |
| 底层角色插科打诨 | 市井角色用语，口语化，短句 | "得，又凉凉了。" |
| 章末钩子 | 仅轻喜剧/都市题材可适度使用，制造悬念+笑点 | "他盯着屏幕上的消息，脑子里只有一个念头：芭比Q了。" |
| 叙述者总结 | **禁止**。网络梗不可出现在第三人称叙述总结中 | ❌ "就这样，他经历了人生中最社死的一天。" |

> **关键边界**：叙述者用网络梗 = AI味反弹的高危信号。05 Gate 会标记所有叙述层的网络梗。

### 4.5 各级使用策略说明

**S级使用策略**：
- 全卷稳定可用，无需担心过时
- 适合用作角色的"语言指纹"——某个角色反复使用某个S级梗可强化人设
- 示例：嘴贱主角的口头禅设为"就这"，每次打脸后说出，形成读者期待
- 注意：S级虽稳定，但不可堆砌——同章3个以上S级梗仍有AI味风险

**A级使用策略**：
- 跨卷使用时注意复核是否已过时
- 适合职场/都市题材的日常章，增加时代感
- A级梗多为"身份认同型"（打工人/摸鱼/摆烂），与角色职业强绑定
- 示例：职场主角内心吐槽"又是带薪摸鱼的一天"——符合身份，不过度
- 注意：A级梗在长连载中可能中途过时，卷末归档时必须复核

**B级使用策略**：
- 仅限短期连载（≤50章）或单卷内使用
- 适合章末钩子和围观群众反应，制造"网感"
- B级梗多为"事件型"（小丑竟是我自己/大无语事件），与具体情节绑定
- 示例：主角被反转打脸后内心吐槽"小丑竟是我自己"——自嘲效果好
- 注意：B级梗在长连载中过期风险高，跨卷时必须替换

**C级使用策略**：
- 仅限实验性短篇或当前连载的即时热点
- 适合青少年角色的对话和内心吐槽，制造"00后感"
- C级梗多为"时效型"（本世纪夏天的第一个夏天/赛博对账），与当下社会事件绑定
- 示例：00后角色感叹"这是本世纪夏天的第一个夏天"——符合角色年龄
- 注意：C级梗过期最快，每月复核，过期立即替换为S级或替代表达

## 五、使用范例对照

> 以下范例展示正确与错误的网络梗使用方式，供03a扩写和04精修参考。

### 5.1 内心吐槽（正确用法）

**场景**：都市职场主角被领导画大饼后回到工位

```
✅ 正确：
林远盯着电脑屏幕，脑子里反复回放刚才领导那句"明年一定给你升职"。
画大饼。他在心里翻了个白眼，这种话他听了三年了，一次都没兑现过。
算了，打工人打工魂，摸鱼才是正经事。他打开了一个摸鱼小游戏。

❌ 错误（叙述层误用）：
林远回到工位，内心感到十分无奈。领导又一次给他画了大饼，这让他作为
一个打工人深感打工魂的无奈。他决定摸鱼，毕竟打工人都是这样度日的。
```

> **分析**：正确版网络梗在角色内心吐槽中出现，语气像真实独白。错误版网络梗
> 进入叙述者总结层，且"画大饼/打工人/打工魂/摸鱼"堆砌4处，超出密度上限。

### 5.2 角色对话（正确用法）

**场景**：玄幻系统流主角打脸反派后

```
✅ 正确：
反派脸色铁青："你……你这是什么妖术？"
林远拍了拍手上的灰，语气随意："就这？我还以为多厉害呢。"
旁边围观的弟子忍不住小声嘀咕："这也太绝了吧……"

❌ 错误（角色失真）：
反派脸色铁青："你这是什么妖术？"
林远拍了拍手上的灰："哈哈，我YYDS！你这水平也敢来挑战我？笑死了，
简直是绝绝子！"
旁边围观的弟子说："师兄真是遥遥领先，硬控全场，一招绝杀！"
```

> **分析**：正确版"就这"符合嘴贱主角声口，围观弟子用"太绝了"自然。错误版
> 主角对话堆砌4个网络梗（YYDS/笑死/绝绝子），不符合修仙世界观的对话风格；
> 围观弟子用"遥遥领先/硬控/绝杀"C级梗，在玄幻题材中严重出戏。

### 5.3 章末钩子（正确用法）

**场景**：都市轻喜剧，主角发现被套路

```
✅ 正确：
手机屏幕亮了，是一条转账记录。
林远盯着那个数字，脑子里只有一个念头：芭比Q了，这下真芭比Q了。

❌ 错误（题材+密度冲突）：
林远看着手机屏幕上的转账记录，陷入了深深的沉思。他感到自己仿佛经历
了一次大无语事件，小丑竟是他自己。他emo了，破大防了，彻底麻了。
```

> **分析**：正确版"芭比Q了"作为章末钩子，简洁有力，制造悬念。错误版在叙述
> 层堆砌4个B级梗（大无语事件/小丑/emo了/破大防），超出密度上限且语义重复。

## 六、常见误用清单

| 误用类型 | 表现 | 修正方案 | 检测方 |
|---------|------|---------|--------|
| 叙述层误用 | 第三人称叙述中使用网络梗 | 改为角色内心吐槽或对话 | 05 Gate (blocking) |
| 密度超标 | 单章网络梗>3处 | 删除低效梗，保留最贴合场景的1-2处 | 05 Gate (advisory) |
| 语义重复 | 连续2处以上网络梗表达同一情绪 | 保留1处，其余改为非网络梗表达 | 05 Gate (blocking) |
| 题材冲突 | 玄幻/悬疑/历史题材使用B/C级梗 | 按题材路由表替换为S级或删除 | 02a/05 Gate |
| 角色失真 | 知识分子/军人角色使用"绝绝子"等 | 查`人物声口对话风格库.md`校验 | 04精修 |
| 过时使用 | 连载中途B/C级梗已过期但未替换 | 卷末归档时复核替换 | 02a卷末归档 |
| 全卷重复 | 同一梗全卷使用>3次 | 替换为同义S级梗 | 05 Gate (advisory) |
| 跨级混用 | 单章使用3个以上不同时效等级的梗 | 限制在2个时效等级内 | 05 Gate (advisory) |

## 七、使用密度规范

| 章节类型 | 网络梗密度上限 | 说明 |
|---------|-------------|------|
| 核心章（冲突/高潮） | ≤2处 | 网络梗会冲淡紧张感，从严控制 |
| 日常章 | ≤3处 | 可适度放松，增加人味 |
| 过场章 | ≤1处 | 过场章本身篇幅短，不宜堆砌 |

**硬性规则**：
- 连续2处以上网络梗且语义重复 = **blocking**（05 Gate 检测，必须修改）
- 同一网络梗全卷不得重复使用超过 **3次**
- B/C级网络梗在长连载（>50章）中自动标记过期风险，需在卷末归档时复核替换
- 单章最多使用 **2个不同时效等级** 的网络梗（如S+A可，S+A+B不可）
- 章末钩子使用网络梗仅限轻喜剧/都市题材，玄幻/悬疑/严肃文学禁止
- **首章慎用**：第1章是读者留存关键，网络梗密度建议≤1处，避免读者误判题材调性
- **高潮章禁用**：卷高潮/全书高潮章网络梗密度=0，防止梗冲淡情绪张力

## 八、风险清单

| 风险 | 说明 | 缓解方案 |
|------|------|---------|
| 过时 | 网络梗生命周期短，B/C级可能连载中途过期 | 时效分级+定期复核，过期立即替换为S级或替代表达 |
| AI味反弹 | 堆砌网络梗=模板化写作，反而像AI | 密度控制（≤3处/章）+05 Gate检测堆砌 |
| 题材冲突 | 严肃/古代/悬疑题材出现网络梗=出戏 | 题材路由封锁，02a建纲时锁定 |
| 角色失真 | 网感过强导致角色不像设定身份 | 人物声口校验（查`人物声口对话风格库.md`） |
| 平台审核 | 网络梗堆砌可能被番茄标记为低质AI内容 | 避免堆砌，保留原创表达，单章≤3处 |
| 语义漂移 | 部分网络梗含义随时间变化 | 定期复核条目含义，更新素材库 |

### 9.2 与人物声口的配合规则

网络梗必须与角色身份匹配，违反以下规则 = 角色失真（04精修拦截）：

| 角色身份 | 可用网络梗特征 | 禁用网络梗 | 示例 |
|---------|-------------|-----------|------|
| 青少年/学生 | 全等级可用，语气词多 | 无 | "笑死，这也太绝了吧。" |
| 职场白领 | S+A级，身份认同型（打工人/摸鱼） | C级时效梗 | "又是带薪摸鱼的一天。" |
| 市井/底层 | S级口语化（凉凉/麻了/就这），短句 | 文绉绉的梗 | "得，又凉凉了。" |
| 知识分子 | 极少量S级，克制使用 | 绝绝子/YYDS/冲鸭 | "这……多少有些破防了。" |
| 军人/纪律型 | 禁用网络梗 | 全部 | — |
| 修仙者（非穿越） | 禁用现代网络梗 | 全部 | — |
| 穿越者 | 内心独白可用S+A，外在对话禁用 | B/C级 | 内心："这不就是996吗？" |
| 中老年角色 | 极少量S级（佛系/凉凉），口语化 | 青年向梗 | "行吧，佛系了。" |

### 11.1 B/C 级淘汰周期自动化管理

| 季度复核 | 人工触发 | 按 10.1 季度复核清单执行，更新时效等级 |

# Part G0 — 黄金三章强制检查
> 从 02a_细纲生成.md 拆分（V5.3模块化降载）
> 加载条件：首批建纲（前10章）必须加载；卷首章建纲推荐加载；滚动补纲/过场章/高潮章建纲可跳过。

## G.1 什么是黄金三章？

黄金三章 = 前3章必须建立的期待感

第1章：爽点 + 钩子（题材适配后以各自开篇抓手为准）
第2章：升级的期待感
第3章：更大的期待感 + 确定追读

## G.2 黄金三章检查协议

```
[黄金三章检查协议]

第1章检查：
├─ 是否有爽点？（玄幻/都市/系统）或开篇抓手（其他题材参照N.0）
│   └─ 否 → ❌ 必死
│
├─ 爽点是否在前500字？
│   └─ 否 → ⚠️ 高风险
│
├─ 是否有结尾钩子？
│   └─ 否 → ⚠️ 建议加入
│
└─ 主角人设是否讨喜？
    └─ 否 → ⚠️ 建议调整

第2章检查：
├─ 是否有新钩子？
│   └─ 否 → ❌ 读者可能流失
│
├─ 是否有升级的期待感？
│   └─ 否 → ⚠️ 建议加入
│
└─ 是否延续了第1章的爽感？
    └─ 否 → ⚠️ 可能断层

第3章检查：
├─ 是否有更大的期待感？
│   └─ 否 → ❌ 读者可能弃书
│
├─ 是否确定了追读动力？
│   └─ 否 → ⚠️ 建议加强
│
└─ 是否有"必须看下一章"的理由？
    └─ 否 → ⚠️ 建议加入强钩子

综合评分：
├─ 三章全过 → ✅ 签约概率高
├─ 1-2章过 → ⚠️ 有风险，建议修改
└─ 全不过 → ❌ 必死，建议重写
```

### 黄金三章模板
- **第1章**：前100字直接切入→爽点(100-500字)→发展(500-1500字)→结尾钩子
- **第2章**：承接第1章麻烦→主角应对+新爽点→期待感升级→更强钩子
- **第3章**：承接第2章决定→主角行动+新发现→确立长期目标+主线→强钩子收尾

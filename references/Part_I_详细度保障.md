## Part I: 详细度一致性保障

### I.1 问题：前面详细，后面一句话带过

**现象**：前几章细纲很详细，中后章越来越简略，最后变成"一句话带过"。

**更严重的现象**：中后期的细纲变成"几章合在一起介绍"——一个故事弧只用一段话概括，而不是每章单独介绍。

```
❌ 错误示例（多章合并介绍）：

Ch.15-18：李四前往C城寻找父亲的线索。
他在路上遇到了张三，两人一起调查。
最终发现父亲当年是被人陷害的。
李四决定复仇。

问题：
├─ 4章内容只用一段话概括
├─ 每章的具体场景、情绪、转折全部缺失
├─ 扩写者根本不知道每章该写什么
└─ 这不是细纲，这是大纲

✅ 正确示例（逐章独立介绍）：

Ch.15：李四独自出发
├─ 核心场景数：3
├─ 情绪心电图：压抑 → 紧张 → 决然
├─ 感官锚点：清晨的冷风、背包的重量
├─ 关键动作：告别 → 上路 → 第一夜露营
└─ 字数分配：4000字

Ch.16：路上遇张三
├─ 核心场景数：3
├─ 情绪心电图：意外 → 警惕 → 勉强同行
├─ 感官锚点：路边的篝火、张三身上的酒味
├─ 关键动作：发现跟踪 → 对峙 → 达成临时合作
└─ 字数分配：4000字

Ch.17：调查线索
├─ 核心场景数：4
├─ 情绪心电图：困惑 → 发现 → 震惊 → 疑虑
├─ 感官锚点：旧档案的霉味、照片上模糊的人影
├─ 关键动作：找到旧档案 → 发现矛盾 → 追问知情人 → 得到新线索
└─ 字数分配：4000字

Ch.18：真相浮出
├─ 核心场景数：4
├─ 情绪心电图：紧张 → 震惊 → 愤怒 → 决心
├─ 感官锚点：雨夜、闪电照亮的面孔
├─ 关键动作：找到关键证人 → 听到真相 → 确认陷害 → 决定复仇
└─ 字数分配：4000字
```

**原因**：
1. **AI注意力衰减**：长文本生成时，AI对后面章节的注意力下降
2. **上下文窗口限制**：生成到后面时，前面的约束条件被"遗忘"
3. **缺少详细度强制检查**：没有机制确保每章详细度一致
4. **一次性生成过多**：试图一次生成太多章节，导致质量递减
5. **AI偷懒合并**：AI倾向于把相似章节合并概括，减少输出量

### I.2 最低详细度要求（硬性约束）

```
[最低详细度要求 - 每章必须满足]

⚠️ 最高优先级约束：逐章独立，禁止合并
├─ 每一章必须有独立的完整细纲
├─ 禁止将多章合并为一段介绍（如"Ch.15-18：..."）
├─ 禁止用一个故事弧概括多章内容
├─ 每章细纲必须包含完整的：核心场景、情绪心电图、感官锚点、字数分配
└─ 即使是过渡章节，也必须有独立的3个以上核心场景

核心场景：
├─ 核心场景数 ≥ 3
└─ 每个场景必须有：场景概括 + 情绪基调 + 关键动作

情绪心电图：
├─ 必须标注
└─ 包含：开头 → 中段 → 高潮 → 收束

感官锚点：
├─ 至少1个
└─ 包含：核心锚点 + 围绕展开

字数要求：
├─ 每章细纲：500-800字（不含元数据）
├─ 每个核心场景：100-200字
└─ 确保细纲信息足够详细，扩写者能直接使用

检查机制：
├─ 每章生成后自动检查
├─ 任一项不合格 → 强制补充，不允许跳过
└─ 检查结果写入自检报告
```

### 分批策略
- 每批最多5章。每批生成后强制检查详细度
- 第N批详细度 < 第1批 × 80% → 触发警告
- 第N批详细度 < 第1批 × 50% → 严重警告，建议重写

> **注**：详细度检查协议、详细度衰减检测、详细度评分等质检规则已剥离至 02b_细纲质检.md。

## Part I.5: 每卷黄金3章设定（关键）

### I.5.1 什么是"黄金3章"？

**定义**：每卷的前3章是该卷的"黄金3章"，决定读者是否继续阅读本卷。

**作用**：
1. 确立本卷的基调/风格/核心冲突
2. 重新抓住读者的注意力（上一卷结束可能已疲劳）
3. 铺设本卷的核心线索和人物关系

### I.5.2 黄金3章的设定要求

```
[黄金3章设定 - 每卷必须]

每卷开始前，必须先设定本卷的"黄金3章"：

═══════════════════════════════════════
【卷X 黄金3章设定】
═══════════════════════════════════════

[本卷基调]
├─ 整体基调：热血/压抑/悬疑/温馨/悲壮...
├─ 节奏特点：快节奏推进/慢热铺垫/张弛有度...
├─ 情绪走向：从X到Y（如：从压抑到爆发）
└─ 核心冲突：本卷要解决的核心矛盾是什么？

[黄金第1章] 开卷抓人
├─ 目标：在500字内抓住读者注意力
├─ 必须包含：
│   ├─ 一个强钩子（危机/悬念/冲突/异常）
│   ├─ 本卷核心冲突的第一次暗示
│   └─ 与上卷结尾的衔接（如有）
├─ 禁止：
│   ├─ ❌ 缓慢的开场（大段环境描写）
│   ├─ ❌ 与本卷主线无关的内容
│   └─ ❌ 让读者困惑的信息过载
└─ 示例：直接从危机场景切入，或从异常事件开始

[黄金第2章] 确立基调
├─ 目标：让读者清楚本卷"要讲什么"
├─ 必须包含：
│   ├─ 本卷核心冲突的正式呈现
│   ├─ 主角的目标/动机/困境的明确
│   └─ 本卷关键角色的登场或铺垫
├─ 禁止：
│   ├─ ❌ 仍在游离，不进入主线
│   └─ ❌ 核心冲突模糊不清
└─ 示例：主角明确"我要做什么"，障碍是什么

[黄金第3章] 深化投入
├─ 目标：读者完全进入本卷的故事流
├─ 必须包含：
│   ├─ 第一次小高潮/小转折
│   ├─ 读者对主角的情感投入（担心/期待）
│   └─ 本卷风格的稳定确立
├─ 禁止：
│   ├─ ❌ 仍是铺垫，没有推进
│   └─ ❌ 风格摇摆不定
└─ 示例：第一个障碍出现，主角做出选择，后果显现
```

### I.5.3 黄金3章与普通章节的区别

| 维度 | 黄金3章 | 普通章节 |
|------|---------|---------|
| 详细度 | 最高（细纲600-1000字） | 正常（细纲500-800字） |
| 钩子强度 | 必须有强钩子 | 可以是弱钩子 |
| 信息密度 | 高（快速确立基调） | 正常 |
| 检查频率 | 每章生成后人工确认 | 自动检查 |
| 违规容忍 | 零容忍 | 可允许低风险违规 |

### 黄金三章常见错误
| 错误 | 症状 | 修正 |
|------|------|------|
| 开卷太平 | 第1章是日常/铺垫/回忆 | 直接从危机/异常/冲突切入 |
| 基调摇摆 | 风格混乱 | 黄金3章统一执行本卷基调 |
| 与上卷脱节 | 像新故事 | 第1章必须呼应上卷结尾 |
| 核心冲突模糊 | 3章后不知主角要干嘛 | 第2章明确主角目标和障碍 |
| 没有小高潮 | 全铺垫无爆发 | 第3章必须有第一个小转折 |

## Part I.6: 长期记忆与锚点管理（02a自管）

> **锚点管理模型**：
> - **00 设定架构师**：P0锚点初始值唯一来源（JSON元数据）
> - **02a 细纲生成**：逐章锚点更新。每章产出衔接包时，同步更新P0锚点快照
> - **02b 细纲质检**：每5章回顾校验衔接包链+锚点一致性，发现漂移在下一章修正
> - **每卷结束**：整理锚点历史，输出伏笔追踪总结

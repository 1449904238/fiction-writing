# 半自动批量模式（V5.4）

> 本文件从 SKILL.md 拆分外置（V6.0）。SKILL.md 中仅保留简要引用。
> **核心问题**：默认每步完成后需用户确认才能进入下一步，连续写5章需确认15-20次，严重影响创作效率。
> **解决方案**：`/mode_assisted` 下可激活批量模式，按当前 `/flow_*` 流程连续执行多步，仅在遇到 **blocking 问题**时暂停。

---

## 激活方式

- 用户输入 `/batch N`（N=连续执行章数，默认3）激活批量模式
- 或用户输入 `/batch` 等同于 `/batch 3`
- 批量模式与 `/flow_*` 正交：`/flow_full /batch 5` = 完整流程连续跑5章

---

## 批量执行流程（以 /flow_full 为例）

```
对每一章（共N章）:
  ├─ 03a 扩写 → 自动进入03b（不等用户确认）
  ├─ 03b 质检 → 判定评分:
  │   ├─ ≥80分 → 自动进入04b（快速通道）→ 自动进入脚本收尾
  │   ├─ 60-79分 → 自动进入04 → 自动进入05 → 自动进入脚本收尾
  │   └─ <60分 → ⛔ 暂停，向用户报告："第X章评分XX分，需重写"
  ├─ 脚本收尾（check-ai-patterns / check-rhythm 等）
  │   ├─ 脚本通过 → 自动进入下一章
  │   └─ 脚本报blocking错误 → ⛔ 暂停，向用户报告脚本输出
  └─ 每章完成后输出一行摘要: "✅ Ch.X完成 [评分XX] [flow:full] [剩余N-X章]"
全部完成后: 输出批量执行报告（每章评分+流程路径+脚本结果）
```

---

## Blocking 暂停条件（仅以下情况暂停，其余自动通过）

| 暂停类型 | 触发条件 | 恢复方式 |
|---------|---------|---------|
| 03b评分不达标 | 评分 < 60分 | 用户决定重写/跳过/调整后继续 |
| 09审稿阻断 | 4-Agent审稿发现critical级问题 | 用户处理审稿意见后继续 |
| 脚本blocking | check-ai-patterns/check-consistency报blocking错误 | 用户修正或确认忽略后继续 |
| 衔接包P0缺失 | 02b质检发现P0字段不完整 | 回退02a补全后继续 |
| sync_status=degraded连续2章 | Delta校验连续2章降级 | 触发09审计，用户确认后继续 |

---

## 非Blocking情况（自动通过，不暂停）

- 03b评分60-79分（走完整流程，不暂停）
- 03b评分≥80分（走快速通道04b，不暂停）
- 脚本advisory级警告（记录但不暂停）
- 02b P1字段缺失（警告但不暂停）
- 09审稿非critical级建议（记录但不暂停）

---

## 批量模式约束

- **章节上限**：单次批量最多5章（防止上下文窗口溢出导致质量衰减）
- **03b不可跳过**：批量模式下03b质检仍然每章必执行
- **质量衰减保护**：若连续2章03b评分下降超过15分，自动暂停并提示"检测到质量衰减，建议检查上下文"
- **退出方式**：用户可随时输入 `/batch stop` 中断批量模式
- **重要章节保护**：开篇章/卷末章/高潮章即使在批量模式下也会在03a后暂停供用户检查（不受批量模式影响）

---

## 批量模式与执行遥测集成（V6.0新增）

批量模式执行完成后，自动运行 `node scripts/check-rule-execution.js --project <项目路径>` 生成规则执行遥测报告，统计本次批量执行中各规则的触发率与通过率。报告写入 `追踪/rule-execution-telemetry.json`。

如需可视化面板，运行 `node scripts/generate-dashboard.js --project <项目路径> --output 追踪/dashboard.html` 生成HTML进度面板。

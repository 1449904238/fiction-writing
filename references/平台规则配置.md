# 平台规则动态配置

## 说明

本文件外置各平台审稿标准，当平台政策变化时只需更新本文件，无需修改 09 技能本体。09 审稿时优先读取本文件。

- **优先级**：09 多视角审稿师启动时，优先读取本文件作为 rubric 来源；本文件缺失或某平台配置块缺失时，回退至 09 技能内嵌的兜底标准。
- **加载方式**：审稿开始前，09 技能按 `platform` 字段定位对应配置块并加载权重、审稿要点、政策约束等内容。
- **维护原则**：本文件为唯一权威来源（Single Source of Truth），避免技能本体与配置文件出现双重维护、口径漂移。
- **兼容性**：配置块字段向下兼容，新增字段不会破坏旧版 09 技能的读取；删除字段时需同步检查 09 技能是否有兜底逻辑。

## 平台配置

### 番茄小说

- **审稿权重**：爽感密度 30% + 节奏 20% + 钩子 20% + 代入 15% + 文笔 15%
- **核心评分维度**：强开局、强冲突、高频爽点/情绪反馈、低理解门槛
- **黄金三章法则**：前 3 章必须在 3000 字内完成"建立人设 → 制造冲突 → 给出爽点"三步
- **爽感密度要求**：每 2000 字至少一个情绪反馈点（爽点 / 笑点 / 紧张点 / 泪点）
- **低理解门槛**：核心设定能否用一句话说清？读者前 500 字能否理解主角处境？
- **完读率及格线**：10 万字完读率 ≥ 15%（脑洞文）/ ≥ 10%（传统爽文）
- **追更率目标**：≥ 35%（验证期书架比 / 追更比）
- **签约政策（2026）**：三次签约机会制（2 万 / 5 万 / 8 万字），全勤需月听读分成 ≥ 500 元
- **AI 检测政策**：纯 AI 永久封禁，AI 辅助需提交人工底稿
- **钩子要求**：章末必留钩子，连续 3 章钩子类型不重复
- **章节定位与张弛**：高压 / 推进章配齐钩子 + 爽点 + 悬念；低压 / 关系 / 修炼章允许弱钩子，但每章都给读者一个往下看的理由

### 起点中文网

- **审稿权重**：设定 30% + 结构 25% + 爽感 20% + 文笔 15% + 钩子 10%
- **核心评分维度**：设定自洽、升级路径清晰、长线期待、世界观承载力
- **中下层扶持**：签约作品稿费未达 5000 元，平台补差
- **AI 使用建议**：可用于大纲润色 / 创意启发 / 资料查询，不可生吞活剥（直接照搬 AI 生成正文）

- **核心评分维度**：短篇钩子密集、反转密度高、情绪兑现快、信息差推进
- **短篇钩子密集**：反转密度高，每篇需保证足够的情节转折

### 七猫小说

- **审稿权重**：爽感 25% + 节奏 20% + 钩子 20% + 代入 20% + 文笔 15%
- **核心评分维度**：大众类型全覆盖、持续阅读能力、人物辨识度、稳定爽点
- **黄金三章法则**：前 3 章需建立主角目标 + 核心冲突 + 第一次爽感交付
- **爽感密度要求**：每 2500 字至少一个情绪反馈点（爽点 / 笑点 / 紧张点 / 泪点）
- **完读率参考**：编辑人工审核 + 持续阅读能力评估
- **签约政策（2026）**：保底 + 分成模式，千字 20-1000 元（经验口径），新人可拿两个月全勤
- **AI 检测政策**：纯 AI 封禁，AI 辅助需提交人工底稿
- **钩子要求**：章末必留钩子，连续 3 章钩子类型不重复
- **章节定位与张弛**：与番茄类似，大众爽文节奏；人物辨识度要求更高，角色性格需在 3 章内立住

### 晋江文学城

- **审稿权重**：人物 25% + 情感 25% + 文笔 20% + 设定 15% + 钩子 15%
- **核心评分维度**：人物立得住、感情有铺垫、女性向垂直、评论互动性强
- **黄金三章法则**：前 3 章需建立主角性格 + 核心关系 + 情感线索
- **爽感密度要求**：情感拉扯型，每 3000 字至少一个情感推进点（心动 / 虐心 / 误会 / 和好）
- **完读率参考**：收藏增长 + 评论互动 + 长期订阅
- **签约政策（2026）**：付费订阅模式，自荐 1 万字起签
- **AI 检测政策**：严格，AI 辅助需标注，大量 AI 生成内容可能影响签约
- **钩子要求**：情感钩子为主，章末留情感悬念；连续 3 章钩子类型不重复
- **章节定位与张弛**：情感铺垫章允许慢节奏，但需有情感推进；高潮章需情感爆发 + 关系转折
- **特殊要求**：人物塑造优先于爽感，感情线需有充分铺垫，避免突兀的情感转折

- **黄金三章法则**：前 3 章需展示创意设定 + 主角登场 + 世界观核心
- **爽感密度要求**：轻喜剧 / 反差型，每 3000 字至少一个笑点或反差点
- **完读率参考**：完读率 + 付费用户增长 + 社区互动
- **签约政策（2026）**：保底 + 分成，保底 40-120 元 / 千字，70% 分成；连载 ≥1 万字 + 大纲
- **AI 检测政策**：原创性要求高，AI 辅助需声明
- **钩子要求**：轻喜剧 / 反差钩子为主；章末留创意悬念
- **章节定位与张弛**：轻松节奏为主，允许日常描写但需有创意亮点；月更 6 万字以上
- **特殊要求**：标题需含搜索词，适合二次元 / 轻小说社区风格

- **钩子要求**：章末强钩子，每章都需章首钩子；连续 2 章钩子类型不重复
- **章节定位与张弛**：极速节奏，几乎不允许低压章；信息释放极快，1-3 章释放核心设定

### 使用规则

- **00.5 项目初始化时**：根据用户选择的平台读取对应行，写入 `novel_meta.platform`，并将本链参数一并写入项目元数据，作为下游所有技能的统一参数源。支持的平台：番茄小说 / 七猫小说 / 起点中文网 / 晋江文学城 / B站轻小说 / 飞卢小说网 / 纵横中文网 / 知乎盐言 / 通用平台。
- **01 大纲构建时**：章节结构 / 爽点间隔 / 高潮间隔按平台参数设定，大纲节点分布严格遵循本链节奏控制参数。
- **02 细纲编写时**：钩子强度 / 信息释放速度按平台参数执行，章末钩子与章首钩子的最低强度不得低于本链要求。
- **03a 扩写时**：对话占比 / 字数范围按平台参数控制，超出范围需在扩写自检阶段修正。
- **默认值**：如果用户未指定平台，默认使用"番茄小说"参数（与本仓库主打的番茄优化保持一致）。
- **优先级**：本链参数为下游技能的硬约束；若下游技能内嵌默认值与本链冲突，以本链为准。
- **结构化配置（V6.0新增）**：下方 YAML 代码块为机器可读的结构化配置，供脚本（如 `check-consistency.js`、`generate-dashboard.js`）和技能程序化读取。YAML 配置与上方 Markdown 配置保持同步，YAML 为程序化读取的唯一权威来源。

```yaml

# 解析方式：读取本文件，提取 ```yaml 到 ``` 之间的内容。

platforms:

    name: 番茄小说
    type: free_ad
    chapter:
      word_count_min: 3500
      word_count_max: 4500
      word_count_target: 4000
    golden_chapters: 3
    retention_metrics:
```

### 结构化字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `type` | enum | `free_ad`(免费广告) / `paid_sub`(付费订阅) / `paid_wordcount`(按字数) / `community`(社区) / `short_story`(短篇) / `generic`(通用) |
| `chapter.word_count_*` | int | 章节字数下限/上限/目标值 |
| `golden_chapters` | int | 黄金章节数（前N章为留存关键） |
| `retention_metrics` | object | 平台特异的留存指标及阈值 |
| `update_rhythm.daily_min` | int | 日更最低字数 |
| `update_rhythm.burst_allowed` | bool | 是否允许爆发更新 |
| `genre_weights` | object | 题材权重（1.0=最高，0.6=较低），用于 06 拆文师推荐对标作品 |
| `hook_density` | enum | `low` / `medium` / `high` / `extreme` |
| `hook_type_preference` | enum | `conflict`(冲突型) / `emotional`(情感型) / `comedy`(喜剧型) / `suspense`(悬念型) |
| `payout_model` | enum | `ad_revenue` / `subscription` / `guarantee_plus_share` / `wordcount_plus_read` / `share` / `generic` |
| `content_safety` | enum | `normal` / `strict` / `relaxed` |
| `structure_type` | string | 章节结构类型标识，与映射链"推荐章节结构"列对应 |
| `scene_count` | string | 推荐场景数范围 |
| `dialogue_ratio_max` | float | 对话占比上限（0-1） |

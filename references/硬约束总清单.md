> V5.3.1 新增：统一硬约束汇总文件。所有生产层（03a）和精装层（04/05/04b）出稿前必须引用本清单。

## 规则分级体系（V6.0新增）

`samples/config/rules.json` V6.0 将所有规则分为三档：

| 分级 | 含义 | 检测方式 | 违反后果 | 规则数量 |
|------|------|---------|---------|---------|
| **hard** | 事实性/一致性/结构性规则 | 脚本强制校验 | 阻断输出 | 8条 |
| **soft** | 风格性/节奏性规则 | 脚本报告不阻断 | LLM自行判断是否调整 | 10条 |
| **adaptive** | 密度性/比例性规则 | 根据章节类型自动调整阈值 | 超阈值时提示建议 | 7条 |

**hard级规则清单**（本文件记录的H1-H7 + rules.json中的R004/R008/R011/R015/R016/R017/R021/R025）：
- H1-H7（见下方硬约束表）= 出稿前机械检测的硬约束
- R004 情绪锚点密度 = 每章至少1个物理情绪锚点
- R008 衔接包自动压缩 = 衔接包链超10章时必须压缩
- R011 角色一致性 = 角色/状态/数值一致性（脚本检测）
- R015 用户情绪锚点 = 02细纲每章必须有用户提供的情绪锚点
- R016 冲突四件套 = 目标/障碍/路径/代价完整性
- R017 锐度保护 = 03a标记的表达04/05不得修改
- R021 混合评分 = 脚本客观分60+LLM主观分40=100分制
- R025 执行遥测 = 每章规则执行率统计

**soft级规则**（R001/R002/R005/R006/R007/R012/R013/R019/R022/R024）：脚本检测后报告，LLM可根据上下文自行判断是否调整。不阻断输出。

**adaptive级规则**（R003/R009/R010/R014/R018/R020/R023）：根据章节类型（core/important/transitional/accent）自动调整阈值。例如比喻密度核心章3-5个、过场章1-2个；burstiness CV核心章≥0.35、过场章≥0.25。

> 完整规则定义、阈值、版本历史见 `samples/config/rules.json`。

## 使用说明

1. **引用方式**：各执行卡的"出稿前最终确认清单"中必须包含：
   > □ 全局硬约束清单H1-H7已全部通过（引用脚本结果）

2. **检测时机**：
   - 03a：终检阶段运行脚本，确认H1/H2/H6/H7
   - 04：精修完成后运行脚本，确认H2/H3/H4/H5/H7
   - 05：去AI味完成后运行全量脚本，确认H1-H7全部通过
   - 04b：快速通道准入时确认H1-H7全部通过

3. **脚本调用**：
   ```bash
   node scripts/check-ai-patterns.js --json <正文文件>          # H1/H2/H6/H7
   node scripts/normalize-punctuation.js --check <正文文件>      # H3/H4/H5
   ```

4. **未通过处理**：
   - 任何一项未通过 → 不准出稿，必须修正后重新检测
   - 脚本结果为准，人工判断为辅（脚本可检测的项以脚本结果为准）

5. **适用范围**：
   - H3/H4/H5（标点类）：04精修后执行，03a初稿不强制（初稿可能有待精修的标点问题）
   - H6（burstiness）：03b质检阶段必须检测，作为快速通道准入条件
   - H7（句式）：03a终检+04精修均需检测，双重保障

## 与各步骤铁律的关系

- H1（字数）= 03a铁律2、04职责声明
- H2（对话比）= AGENTS.md全局约束
- H3/H4/H5（标点）= 04标点统计达标条款
- H6（burstiness）= 03b质量Gate准入条件
- H7（句式）= 03a终检第4项、04精修检测清单第3项

> 本清单不替代各步骤的完整检测项，仅作为跨步骤统一的硬约束基线。

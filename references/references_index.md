## 一、03 质感系统

> 扩写系统的"油门系统"，与各检测器的"刹车系统"并行执行。03 扩写前必须先读取质感注入协议，扩写中参考质感锚点库与情绪工程协议。

### 03_质感注入协议.md
- **用途**：03 扩写系统的"油门"，正面引导如何写出有质感的文字（管"该怎么写"）。
- **适用步骤**：03 / 04
- **引用方式**：自动加载（03 扩写前必读）
- **关键内容**：
  - 核心原则：质感 = 可感知的具体物，不写抽象概括
  - 五大质感技法：动作链写生理状态 / 具体名词替代形容词 / 设定生活化 / 感官投放 / 对话人味
  - 比喻质量门 + 质感自检清单
  - **第六章 细写决策协议（V2.4新增）**：S/A/B/C 四级细写决策表（重要度→细写程度）、画面感达标标准、不细写红线、与 K.4/F.1.2 统一映射

### 03_新事物释放协议.md（V2.4 新建）
- **用途**：定义新概念/物品/规则/能力/组织/地名出现时"何时释放、释放多少、用什么方式释放"——反解释体系的正面补丁。
- **适用步骤**：02 / 03 / 04 / 05
- **引用方式**：自动加载（03 扩写时与质感协议并行；04精修 Step4 解释腔消除时对照边界裁定表）
- **关键内容**：
  - 核心区分：释放（必须做，角色/场景带出）≠ 解释腔（禁止做，叙述者百科定义）
  - 按重要度 P0-P3 分级释放详细度
  - 八大释放技法：对话带出/角色反应/生活场景/前置铺垫/后置揭秘/物件暗示/误解纠正/代价展示
  - 释放时机选择（前置/同步/后置/延时）
  - 与 Gate11/冰山原则/潜台词拦截器/铁律5 的边界调和表

### 03_情绪工程协议.md
- **用途**：03 扩写系统的第二台"油门"，工程化情绪转移而非模拟情绪表达（管"读者读完身体有没有反应"）。

## 一.六、核心素材库（V2.7 新增 — 小说骨架语言）

### 感官细节库.md（V2.7 新建）
- **用途**：解决"干瘪叙述"和"告诉而非展示"的核心痛点。不是词汇表，是按场景→感官→具象表达三层组织的实战工具。
- **适用步骤**：03 / 04 / 05
- **引用方式**：03扩写写场景时打开五感细分词表；04精修诊断"只写了视觉"的场景；05去AI味检查AI套路化身体反应
- **关键内容**：
  - 五感细分词表：嗅觉15种/触觉14种/听觉10种/视觉6要素/味觉5种（每种附带适用场景+误用警告）
  - 通感修辞集：听觉→触觉/视觉→触觉/嗅觉→视觉（每章≤2处）
  - 身体反应映射表：恐惧/愤怒/悲伤/喜悦/厌恶5类情绪，每种≥3个差异化身体反应（标注适用角色+场景+击败AI味的方法）

### 人物声口对话风格库.md（V2.7 新建）
- **用途**：解决"所有角色说话都像作者"和"对话推进剧情效率低"的核心痛点。按身份→句式→潜台词→节奏四层组织。

- **引用方式**：02细纲为每个角色填写语言指纹卡；03扩写写对话前检查角色语言指纹；04精修执行遮名测试
- **关键内容**：
  - 身份语料包：6种身份（医生/市井/商人/军人/文人/学生）的词汇特征+句式特征+禁忌
  - 潜台词对照表：压抑愤怒/掩饰感受/权力博弈/亲密伤害4类，每类"表面台词→真实意图→伴随动作"三元组

  - 角色语言指纹卡：句长/语速/口头禅/禁用词/权力位置/特殊习惯6维度

### 场景氛围环境渲染库.md（V2.7 新建）
- **用途**：解决"背景板空白"和"情绪缺乏载体"的核心痛点。按意象象征→动态描写→转场衔接三层组织。

## 使用建议

1. **00 设定架构师**：自动加载 `00_命名协议.md`（角色/地点/物品/能力命名按P0-P3分级，P0须写命名理据）+ `00_严谨性校验协议.md`（V2.6新增：P0角色填写生平逻辑链，P0地点/势力回答合理性五问，禁止"随便编造"）。
2. **03 扩写前**：自动加载 `03_质感注入协议.md`（含第六章细写决策表）+ `03_新事物释放协议.md`（油门）+ `03_情绪工程协议.md`，并加载全部检测器（刹车）。
3. **03 扩写中**：按需参考 `03_质感锚点库.md` 对标好段落；按场景调用语言素材库；写前查 `改写范例库.md` 预防层规避AI味句式；新概念/物品出现时查 `03_新事物释放协议.md` 选释放技法（V2.4新增）。
4. **03 扩写后**：执行自检第11项（新事物释放）+第12项（命名一致性）+终检第19项（新事物释放终检）（V2.4新增）。
5. **04 精修时**：参考质感系统（含细写决策表）深化；Step4解释腔消除时对照 `03_新事物释放协议.md` 第六章边界裁定表，区分"解释腔（删）"与"合法释放（保留）"（V2.4新增）；校验命名一致性。
6. **05 去AI味**：按需调用 `改写范例库.md` + `网文通用高频词替换库.md`；Gate 11 检测时对照 `03_新事物释放协议.md` 边界，不误删合法释放；读取04精修意图包后跳过已处理项。
7. **开篇 / 新卷**：调用 `Part_K` 选开篇方法，`Part_L` 定文风，`Part_M` 查题材公式。
8. **跨步骤交接**：03交付时附加扩写意图包，04交付时附加精修意图包（见 `04_交接协议.md`）。
9. **伏笔管理**：02埋设伏笔时查 `02_伏笔全生命周期管理.md` 建立追踪表，03/04按表操作。
10. **平台审稿**：09审稿时加载 `平台规则配置.md` 对应平台规则。
11. **素材库调用前**：必须先完成安全鉴权（读取 `novel_meta.core_tags` + `Part_L_文风系统`）。

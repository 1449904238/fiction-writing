# 读者反馈注入师（V1.0 — 读者反馈闭环）

> 本技能为可选步骤，在09多视角审稿师之后执行（按卷触发）。
> 核心理念：将番茄平台的读者行为数据回流入创作流程，驱动后续章节的节奏/写法调整。

## 适用场景
- 每完成一卷后执行（约30-50章）
- 番茄后台数据更新后按需执行
- 完读率异常波动时紧急触发

## 输入
用户提供以下数据（手动粘贴）：
- 番茄完读率数据（按章节）
- 章节评论摘选（高赞评论/差评评论）
- 收藏/推荐数据变化趋势
- 弃书节点数据（如有）

## 分析维度

### 1. 完读率分析
- 高完读率章节（>卷均值）→ 提炼"什么写法有效"→ 注入02细纲后续章节
- 低完读率章节（<卷均值10%+）→ 诊断"哪里出了问题"→ 02调整后续节奏
- 完读率断崖式下跌章节 → 紧急诊断"哪里让读者放弃"→ 紧急调整

### 2. 评论分析
- 高评论章节 → 提炼"什么引发了讨论"→ 03扩写加强此类写法
- 读者反复提及的角色 → 加强戏份或调整人设
- 读者吐槽的情节/逻辑 → 标记为"需修复"，回溯02/03调整
- 读者催更的线索 → 提前该线索的回收时间

### 3. 趋势分析
- 收藏增长加速 → 当前节奏有效，保持
- 收藏增长放缓/下降 → 节奏可能出问题，需要调整
- 推荐票集中章节 → 高潮章节有效，分析原因

## 输出

### 读者反馈分析报告
```
【读者反馈分析报告 — 第X卷】

完读率概况：
- 卷均完读率：XX%
- 最高章节：第XX章（XX%）— 原因分析：[爽点密集/悬念强/情感爆发]
- 最低章节：第XX章（XX%）— 原因分析：[节奏拖沓/信息密度低/角色行为不合理]
- 断崖下跌：第XX章→第XX章（跌幅XX%）— 原因分析：[...]

评论热点：
- 最受讨论角色：XXX（提及XX次）— 读者态度：[正面/负面/争议]
- 最受讨论情节：第XX章XXX情节 — 读者态度：[...]
- 读者吐槽Top3：1.XXX 2.XXX 3.XXX

趋势判断：
- 收藏增长：[加速/平稳/放缓] — 趋势判断：[...]
- 推荐票集中：第XX章 — 说明：[...]
```

### 02细纲调整建议
- 后续章节节奏调整建议（具体到哪些章节怎么改）
- 角色戏份调整建议
- 伏笔回收时间调整建议

### 03扩写写法调整建议
- 高完读率章节的写法特征提炼
- 低完读率章节的写法问题诊断
- 后续章节的写法建议

## 与逆向调整机制联动
读者反馈触发的调整走"逆向调整机制"流程（见03扩写系统四.5节）：
1. 识别偏差（读者反馈数据表明的偏差）
2. 评估影响范围
3. 用户确认调整方向
4. 执行调整（修改02细纲）
5. 校验一致性
6. 恢复正常流程

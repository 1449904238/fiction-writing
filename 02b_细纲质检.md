> **当前技能**：第 3b 步 — **细纲质检**（对02a产出的细纲进行独立质量审查）
>
> **上游依赖**：
> - ← **02a 细纲生成**：接收逐章细纲 + 衔接包（P0/P1分级） + 记忆锚点标注 + 伏笔追踪表
> - ← **00 设定架构师**：接收 JSON 元数据（角色档案/关系矩阵/反例对照表/number_anchors/power_system）
> - ← **01 大纲构建师**：接收分卷大纲与情绪曲线（用于大纲五检的卷级验证）
>
> **下游消费者**：
> - → **细纲扩写执行系统（03a）**：质检通过的细纲 + 衔接包放行交付扩写
> - → **02a 细纲生成**：质检不通过的细纲回退02a补充/重写
>
> **核心使命**：对02a产出的逐章细纲和衔接包进行独立质量审查，确保细纲在交付03a扩写前达到质量标准。本模块只负责"检测"，不包含生成规则。
>
> **本模块独立于02a运行**：可作为独立prompt使用，输入02a的产出物即可执行质检。

## 流水线契约 — 本技能边界声明

> **本技能只负责：质检。**
>
> **本技能不负责**：
> - ❌ 细纲生成（章节蓝图/字数预算/衔接包生成/伏笔追踪表生成）→ 这些属于 **02a_细纲生成.md**
> - ❌ 正文质感规则（AI味检测/感官倒置/脏感注入/物理代价）→ 这些属于 **04精修师**
> - ❌ 修改衔接包内容（发现问题时标记并回退02a修正，不自行修改）
>
> **本技能的交付物**：质检报告（通过/返工/重写）+ 评分 + 问题清单
>
> **数据所有权**：
> - 02b对衔接包只读不写。发现衔接包问题时，在质检报告中标记，回退02a修正。
> - 02b对伏笔追踪表只读不写。发现伏笔超期或矛盾时，在质检报告中标记，回退02a处理。

# Part O5: 大纲五检（每卷/每章设计前必答）

> **v4.3 新增**：源自 oh-story-claudecode 的大纲五检体系。在每卷开始设计和每章细纲编写前，必须先回答以下5个问题，从源头保障结构质量。
>
> **V5.2.0 说明**：02a在生成时已"预答"大纲五检速记。02b的职责是验证这些预答是否合理、是否达标。

```
[大纲五检协议]

每卷设计前必答（02b验证02a是否已回答）：
├─ ① 本卷交付什么情绪？什么剧情模式能可靠交付？
│   └─ 答不出 → 回01大纲确认该卷定位
├─ ② 本卷核心冲突是什么？
│   └─ 答不出 → 回01大纲确认主线矛盾
├─ ③ 卷节奏（起承转合）哪段加速哪段减速？
│   └─ 高压/推进章占比应≥60%，低压/过场章≤15%
├─ ④ 本卷需要新埋设的伏笔有哪些？上一卷待回收的伏笔如何处理？
│   └─ 与追踪/伏笔.md同步
└─ ⑤ 章节定位分布是否有高低层次（不是全程高压）？
    └─ 低压+过场合计不超约15%
    └─ 1/4·中点·3/4各锚定一个关键情节

每章细纲前必答（02b验证02a预答的5+1问）：
├─ ① 本章交付什么情绪？（目标情绪词）
├─ ② 本章核心冲突/爽点是什么？
├─ ③ 本章在卷节奏中处于什么位置？（加速/减速/转折）
├─ ④ 本章需要埋设/回收的伏笔？
├─ ⑤ 本章定位：高压/推进/修炼试错/关系回收/低压生活/信息整理
│   └─ 留空按推进章
│   └─ 高压/推进章配齐钩子+爽点+悬念
│   └─ 低压/关系/修炼/信息整理章允许无显性爽点、弱钩子或仅情绪钩子
│   └─ 底线：每章都给读者一个往下看的理由、相邻章不情绪趋同
└─ ⑥ 用户情绪锚点（V5.1新增）：用户期望读者读完本章后的核心感受？（1-2个词，如"解气/心疼/期待/紧张/震撼"）
    └─ 此字段为用户创作意图输入，03a扩写时须作为情绪工程的首要目标
    └─ 与①目标情绪的关系：①是"角色/叙事交付的情绪"，⑥是"用户期望读者体验的情绪"，两者可能不同
```

> **与既有机制的关系**：大纲五检是 Part N（爽感保护）和 Part G0（黄金三章）的上游预检——先确认结构设计合理，再执行具体检查。五检不通过将触发对应Part的警告。
>
> **02b验证规则**：逐章检查02a预答的6问是否完整、合理。缺失任何一问 → 标记"大纲五检不完整"，回退02a补充。

# Part O6: 细纲质量Gate（V5.1新增 — 产出后必检）

> **执行时机**：02a产出每章细纲+衔接包后、交付03a扩写前。
> **与大纲五检的关系**：五检是"设计前预答"（写之前想清楚），质量Gate是"产出后验证"（写完检查是否达标）。
>
> **V5.2.0 说明**：本Gate是02b的核心质检关卡，7项全过才放行。

```
【细纲质量Gate — 每章检查（7项全过才放行）】

① INC章节级检查（至少命中3/13要素）
├─ 新信息/新冲突/新角色面相/新场景/新能力/新关系/新悬念/新反转/新情绪/新代价/新选择/新障碍/新认知
├─ 低压/过场章可降至2项
└─ 命中<2项 → 标记"信息空洞章"，补充内容或合并到相邻章

② 冲突四件套检查（V5.1新增 — 每个核心场景必检）
├─ 目标（Goal）：角色在这场戏里想得到什么？
├─ 障碍（Obstacle）：什么阻止角色达成目标？
├─ 路径（Path）：角色采取什么行动应对障碍？
├─ 代价（Cost）：无论成败，角色付出了/失去了什么？
├─ 四件套缺失任何一项 → 场景缺乏戏剧张力，补充
└─ 过场/信息整理场景可豁免

③ 钩子类型连续性检查
├─ 本章章末钩子类型 → 前2章钩子类型 → 3章内是否有重复？
├─ 重复 → 更换钩子类型（13种章末钩子可选）
└─ 记录本章钩子类型到衔接包

④ 情绪连续性检查
├─ 本章目标情绪 → 前一章目标情绪 → 是否相同？
├─ 相同 → 调整本章情绪方向（相邻章情绪不趋同）
└─ 记录本章情绪到衔接包情绪蓝图

⑤ 衔接包完整性检查（V5.2.0 P0/P1分级校验）
├─ P0必须字段检查（缺失=不通过）：
│   ├─ [角色当前状态]（位置/伤势/情绪/持有物/战力等级/生死状态）→ 完整？
│   ├─ [禁止矛盾] → 至少3条？
│   ├─ [数值锁定] → 涉及的关键数值已标注？与number_anchors一致？
│   └─ [角色知识边界] → 在场角色的已知/未知已标注？
│   └─ P0字段缺失任何一项 → 不通过，补全后放行
├─ P1可选字段检查（缺失=警告但不阻断）：
│   ├─ [前章回顾] → 有则检查一致性，无则警告
│   ├─ [必须延续的细节] → 有则检查完整性，无则警告
│   ├─ [状态锁定] → 有跨章状态但未标注 → 警告
│   ├─ [闪回标注] → 含闪回但未标注 → 警告
│   ├─ [本章新增状态] → 有则检查一致性，无则警告
│   ├─ [质感预算] → 有则检查合理性，无则警告
│   └─ [跨章去重清单] → 有则检查更新，无则警告（注：可由脚本自动生成）
│   └─ P1字段缺失 → 警告但不阻断放行

⑥ 字数预算合理性检查
├─ 核心场景(1200-1800字) × N + 重要场景(800-1200字) × N + 过场(400-800字) × N + 点缀(200-400字) × N
├─ 总和是否在3500-4500字范围内？
├─ 场景总数是否在3-5个？
└─ 不符合 → 调整场景分配

⑦ 信息释放检查（V5.1新增）
├─ 本章释放的设定信息是否标注了释放方式（对话/旁白/行为/环境）？
├─ 单章新设定释放是否≤3条？（超出=info-dump风险）
├─ 释放的信息是否与00设定一致？
└─ 不通过 → 调整信息释放节奏
```

**Gate结果处理：**
- 7项全过 → 细纲放行，交付03a扩写
- 1-2项不通过 → 标记问题项，在细纲中补充后放行
- ≥3项不通过 → 细纲回退02a重写该章

# Part G.6: 衔接包自检（11项 — V5.2.0 P0/P1分级校验）

> **执行时机**：每章衔接包产出后、质量Gate第⑤项检查时同步执行。
> **V5.2.0 变更**：衔接包字段从11个必须变为4个P0必须+7个P1可选。P0缺失=不通过，P1缺失=警告但不阻断。

```
[衔接包自检 - 每章检查（V5.2.0 P0/P1分级校验）]

━━━ P0必须字段检查（缺失=不通过 ❌）━━━

检查项1：[角色当前状态] 【P0】
├─ 是否与物理状态表一致？
│   └─ 不一致 → ❌ 修正（伤势、位置、持有物）
├─ 战力等级和生死状态是否已标注？
│   └─ 未标注 → ❌ 补充（从00设定的 power_system 和 number_anchors 获取）
└─ 已死亡角色是否标注了"已死亡"和死亡章节？
    └─ 未标注 → ❌ 补充

检查项2：[禁止矛盾] 【P0】
├─ 是否列出了所有容易出错的点？
│   └─ 遗漏 → ❌ 补充
└─ 至少3条？
    └─ 不足3条 → ❌ 补充

检查项3：[数值锁定] 【P0】
├─ 是否列出了本章涉及的所有关键数值？
│   └─ 遗漏 → ❌ 补充（年龄/距离/枚数/境界/物品数量/时间线/生死状态）
├─ 数值是否与00设定的 number_anchors 一致？
│   └─ 不一致 → ❌ 修正衔接包
└─ 已死亡角色是否在数值锁定中标注？
    └─ 未标注 → ❌ 补充

检查项4：[角色知识边界] 【P0】
├─ 是否标注了在场角色的已知/未知信息？
│   └─ 未标注 → ❌ 补充（至少标注P0角色和当前信息差涉及的角色）
└─ 信息差是否与剧情设计一致？
    └─ 剧情要求A不知道某事，但知识边界标注A已知 → ❌ 修正

━━━ P1可选字段检查（缺失=警告 ⚠️ 但不阻断）━━━

检查项5：[前章回顾] 【P1】
├─ 是否与上一章细纲一致？
│   └─ 不一致 → ⚠️ 警告并标记修正
└─ 缺失 → ⚠️ 警告"P1字段缺失：前章回顾"

检查项6：[必须延续的细节] 【P1】
├─ 是否遗漏了关键细节？
│   └─ 遗漏 → ⚠️ 警告并标记补充
└─ 缺失 → ⚠️ 警告"P1字段缺失：必须延续的细节"

检查项7：[状态锁定] 【P1】
├─ 是否标注了需保持不变的状态？
│   └─ 有需跨章保持的状态但未标注 → ⚠️ 警告并标记补充
├─ 锁定的状态是否有明确的持续章节范围？
│   └─ 未标注范围 → ⚠️ 警告
└─ 缺失 → ⚠️ 警告"P1字段缺失：状态锁定"

检查项8：[闪回标注] 【P1】
├─ 本章是否含闪回/回忆/幻觉？
│   └─ 含闪回但未标注 → ⚠️ 警告并标记补充（涉及角色+起止标记+闪回原因）
├─ 闪回中涉及已死亡角色但未在[数值锁定]中标注生死状态 → ⚠️ 警告
└─ 无闪回且未标注 → ⚠️ 轻微警告（建议至少标注"本章无闪回"）

检查项9：[本章新增状态] 【P1】
├─ 是否与本章细纲的后续影响一致？
│   └─ 不一致 → ⚠️ 警告并标记修正
├─ 是否有数值变更但未标注？
│   └─ 角色突破/升级/死亡但未标注"数值变更" → ⚠️ 警告并标记补充
└─ 缺失 → ⚠️ 警告"P1字段缺失：本章新增状态"

检查项10：[质感预算] 【P1】
├─ 质感投放规划是否合理？
│   └─ 不合理 → ⚠️ 警告
└─ 缺失 → ⚠️ 警告"P1字段缺失：质感预算"

检查项11：[跨章去重清单] 【P1】
├─ 是否已更新（近3章句式模板+近5章场景元素+章末收尾方式）？
│   └─ 未更新 → ⚠️ 警告（注：可由 extract-used-patterns.js 自动生成，手动维护时可跳过）
└─ 缺失 → ⚠️ 轻微警告"P1字段缺失：跨章去重清单（可由脚本自动生成）"

━━━ 自检结果汇总 ━━━

通过标准：
├─ P0字段（4项）全部完整且无矛盾 → P0通过
├─ P1字段（7项）缺失 → 警告但不阻断放行
├─ P0+P1全完整 → ✅ 完美衔接包
├─ P0完整+P1部分缺失 → ⚠️ 放行但标记P1补充建议
└─ P0缺失 → ❌ 不通过，回退02a补全P0字段后重新质检
```

# Part G.7: 衔接包链自检（每5章）

> **执行时机**：每完成5章细纲后执行一次。
> **V5.2.0 说明**：本检查由02b执行，发现问题时在质检报告中标记，回退02a修正。

```
[审计反馈处理 — 每5章自检]

伏笔追踪表审计 →
├─ 检查伏笔追踪表，定位超期伏笔
├─ 在后续5章内规划回收节点
├─ 如伏笔已不再适用 → 标记为 ABANDONED 并记录废弃原因
└─ 处理完成后 → 标记回退02a更新衔接包中的伏笔状态字段

衔接包链一致性自检 →
├─ 检查衔接包某项状态是否与更早的P0/P1锚点不符
├─ 是 → 在质检报告中标记，回退02a修正衔接包中对应的状态字段（衔接包是图纸，可以修正）
├─ 修正后 → 02a重新生成受影响的细纲章节
│
数值与战力链一致性自检（V4.7新增）→
├─ 扫描最近5章衔接包的[数值锁定]（P0字段），检查同一数值在不同章节中是否一致
│   ├─ 如"水潭→营地"在Ch.1写"东4里"、Ch.3写"东9里" → 矛盾，以 number_anchors 为准修正
│   └─ 如角色年龄在Ch.1写"19岁"、Ch.3写"21岁"且无时间跳跃 → 矛盾，修正
├─ 扫描最近5章衔接包的[角色当前状态]（P0字段）战力等级，检查是否与 power_system 矩阵一致
│   ├─ 如角色从"凡骨·2枚"突然变成"硬骨·5枚"且无突破情节 → 矛盾，修正
│   └─ 如已死亡角色在后续章节的[角色当前状态]中未标注"已死亡" → 矛盾，修正
└─ 检查[数值锁定]是否遗漏了已死亡角色的生死状态标注 → 遗漏 → 补充

衔接包链连续性自检 →
├─ 检查衔接包链是否完整连续（无断章/无矛盾状态）
├─ 检查跨章去重清单是否已更新（近3章句式模板+近5章场景元素+章末收尾方式）
├─ 检查大纲五检是否每章都已回答（目标情绪/核心冲突爽点/卷节奏位置/伏笔/章节定位/用户情绪锚点）
└─ 如发现问题 → 回退02a修正对应章节细纲+衔接包 → 再重新进入质检
```

> 注意：如果审计发现的是"衔接包禁止矛盾 vs 扩写正文内容不符"，那是03扩写系统的问题，不是衔接包的问题。衔接包是图纸，不能因为施工难度大就随便改。只有衔接包自身与更早的记忆锚点冲突时，才修正衔接包。

# Part I.5质检: 详细度衰减检测

> **执行时机**：每批（3-5章）细纲生成后执行。

```
[详细度衰减检测]

检测方法：
├─ 计算每批的平均详细度
├─ 对比第N批与第1批的详细度
└─ 如果衰减超过阈值，触发警告

衰减检测：
├─ 如果第N批详细度 < 第1批详细度 × 80%：
│   └─ ⚠️ 警告："检测到详细度衰减"
│   └─ 建议：
│       ├─ A. 减少每批章节数（从5章改为3章）
│       ├─ B. 每章生成后暂停，人工确认
│       └─ C. 分天生成（避免一次性生成过多）
│
└─ 如果第N批详细度 < 第1批详细度 × 50%：
    └─ ❌ 严重警告："详细度严重衰减，建议重写本批"
    └─ 可能原因：
        ├─ AI注意力耗尽
        ├─ 约束条件被遗忘
        └─ 上下文窗口不足
```

### 详细度评分

- 核心场景数(25分)：≥3=25, 2=15, 1=5
- 场景完整度(25分)：概括+情绪+动作齐全=25
- 情绪心电图(20分)：完整标注=20
- 感官锚点(15分)：有核心锚点+展开=15
- 字数(15分)：≥500=15, 350-500=10, <350=5
- ≥80分=优秀, 60-80=合格, <60=重写

# 02→03a 交接协议

> **触发时机**：02b质检报告结论为"放行"后，立即触发03a扩写。

```
[02b→03a 交接协议]

交接物清单：
├─ 逐章细纲（完整复制 — 02b质检通过的版本）
├─ 衔接包（完整复制 — P0必须字段完整，P1可选字段按需）
├─ P0 锚点标注
├─ 伏笔追踪表
├─ 02b质检报告（作为03b质量自检的参考基准）
└─ 题材适配信息（novel_meta.genre + 启用的约束协议）

交接规则：
├─ 02b质检通过的细纲 = 锁定状态，03a扩写时不得修改细纲
├─ 衔接包P0字段 = 03a的铁律，扩写时绝对遵守
├─ 衔接包P1字段 = 03a的参考，有则遵守，无则自行判断
├─ 03a发现衔接包P0字段与正文冲突 → 修正正文，不改衔接包
└─ 03a发现衔接包P0字段本身有矛盾 → 标记并反馈，不自行修改

使用方式：
├─ 打开03a，取第1章的细纲+衔接包 → 扩写第1章正文 → 03b质检
├─ 取第2章的细纲+衔接包 → 扩写第2章正文 → 03b质检
├─ ... → 逐章推进
├─ 每5章暂停，提交审计（02b衔接包链自检）
└─ 02a全书只执行1次（生成），02b每批执行1次（质检），后续每章循环只在03a→03b→04→05之间运转
```

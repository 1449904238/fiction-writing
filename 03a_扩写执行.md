### 场景级蓝图生成（V5.4新增 — 章级意图细化为场景级字数预算）

> **核心问题**：当前03a只有章级意图（来自02a细纲），没有场景级字数预算和情节点分解。导致扩写时LLM对节奏的把控靠"感觉"而非"蓝图"。
> **解决方案**：扩写前将章节分解为3-5个场景，每个场景分配字数预算和情节点。

**场景蓝图生成规则**：

```
场景蓝图(扩写前生成):
├─ 1. 从02a细纲提取本章的场景列表（通常3-5个场景）
├─ 2. 为每个场景分配:
│   ├─ 场景类型: core(1200-1800字) / important(800-1200字) / transitional(400-800字) / accent(200-400字)
│   ├─ 字数预算: 根据场景类型和本章总字数(3500-4500字)分配
│   ├─ 情节点: 该场景需要完成的1-2个具体情节动作
│   ├─ 感官策略: 该场景的感官投放计划（核心场景3-5感官/过渡场景1感官）
│   ├─ 情绪节点: 该场景的情绪起点和终点
│   └─ 钩子: 该场景结束时的微钩子（章末场景才有章末钩子）
├─ 3. 字数预算总和 = 本章目标字数(±200字浮动)
└─ 4. 输出场景蓝图表
```

**场景蓝图表格式**：

```yaml
场景蓝图 — 第X章 (目标字数: 4000字):
  场景1 (transitional, 500字):
    情节点: [主角到达目的地]
    感官策略: 1个视觉锚点（环境特征）
    情绪: 平静→警觉
    钩子: 发现异常痕迹
  场景2 (core, 1500字):
    情节点: [主角与反派对峙，揭露真相]
    感官策略: 3-5感官叠加+1个感官错位
    情绪: 警觉→愤怒→震惊
    钩子: 反派逃脱
  场景3 (important, 1000字):
    情节点: [主角追击，遭遇陷阱]
    感官策略: 触觉+听觉（痛感+环境声）
    情绪: 震惊→紧张→恐惧
    钩子: 陷阱中的意外发现
  场景4 (accent, 500字):
    情节点: [主角在陷阱中发现关键线索]
    感官策略: 1个触觉锚点（线索物件的质感）
    情绪: 恐惧→困惑→决心
    钩子(章末): 线索指向一个不可能的人
```

**场景蓝图使用规则**：
1. 扩写时按场景顺序逐一展开，不可跳过场景
2. 每个场景的字数不可超出预算±200字
3. 场景类型决定感官投放密度（不可对过渡场景做3-5感官叠加）
4. 情绪节点必须前后衔接（场景A的情绪终点=场景B的情绪起点）
5. 章末场景必须有章末钩子（13种钩子类型，3章内不重复）

**与现有细纲的关系**：
- 场景蓝图是对02a细纲的"执行级细化"，不替代细纲
- 细纲提供"写什么"（情节/伏笔/角色），场景蓝图提供"怎么写"（字数/感官/节奏）
- 03b质检时检查场景蓝图执行偏差（实际字数 vs 预算字数偏差>30%=警告）

### 场景感官策略标准模板（V6.0.1 — 场景类型×感官组合决策表）

> **核心问题**：场景蓝图中"感官策略"字段缺乏标准组合，LLM需每次自行决策五感配比，认知负荷高且易出现"过渡场景五感全开"的AI味。
> **解决方案**：按场景类型（core/important/transitional/accent）固化感官组合标准，扩写时直接查表选型，禁止越级投放。
> 与 `references/感官细节库.md` 协同——本模板管"选哪些感官"，感官细节库管"用什么具体词"。

**感官强度标记**：●●=重点描写（须用具体感官词，不可用"很冷/很吵"等抽象词）/ ●=必要描写（1-2句精准锚定）/ ○=可选描写（视剧情需要启用）

| 场景类型 | 字数范围 | 视觉 | 听觉 | 触觉 | 嗅觉 | 味觉 | 组合策略 |
|---------|---------|------|------|------|------|------|---------|
| core | 1200-1800 | ●● | ●● | ● | ○ | ○ | 视听触三感叠加，关键节点加嗅觉点睛；味觉仅限口腔受伤/中毒场景 |
| important | 800-1200 | ●● | ● | ○ | ○ | ○ | 视觉为主+听觉辅助；触觉仅限角色与物件直接接触时；嗅觉味觉禁用 |
| transitional | 400-800 | ● | ○ | ○ | ○ | ○ | 单一视觉快速推进，1个精准细节锚定环境；其余感官全部禁用 |
| accent | 200-400 | ● | ○ | ○ | ○ | ○ | 单一感官点睛，1句话1个感官词收束；禁止感官叠加 |

**使用规则**：
1. 扩写前从场景蓝图读取场景类型 → 查上表确定感官组合 → 写入场景蓝图的"感官策略"字段
2. core场景的"关键节点加嗅觉"指：情绪转折点/危机爆发点/身份揭示点，每场景最多1处嗅觉
3. transitional场景严格执行"单一视觉"——禁止"他闻到/他听到/他感到"，只写"他看见"
4. accent场景的"单一感官点睛"须与该场景的核心物件/动作绑定，不可游离
5. 跨场景感官去重：相邻两个场景不可使用相同的感官词（如场景1用"刺痛"，场景2禁用"刺痛"）

**感官组合示例**（每种场景类型1例，每例2-3句）：

- **core示例（视听触三感叠加+嗅觉点睛）**：
  舱门焊死的瞬间，警报灯把整条走廊刷成血红色（视觉●●）。金属地板在脚下震颤，震动从鞋底一直传到膝盖（触觉●）。警报声尖锐地切割着空气，每隔三秒一断（听觉●●）。他闻到一股焦糊味——电路烧了（嗅觉点睛○）。

- **important示例（视觉为主+听觉辅助）**：
  他站在卷宗库门口。架子上密密麻麻的竹简从地面堆到天花板，每一卷都绑着褪色的麻绳（视觉●●）。远处有人在抄写，笔尖刮过竹面的沙沙声断断续续传来（听觉●）。

- **transitional示例（单一视觉快速推进）**：
  他穿过前厅，绕过影壁，径直走进内院。廊下的灯笼还没灭，纸罩上透出一圈昏黄的光晕（视觉●）。

- **accent示例（单一感官点睛）**：
  他把那枚铜钱攥在手心，直到棱角硌进肉里（触觉●）。

**感官组合禁忌清单**：

1. ❌ **transitional场景禁止五感全开**：过渡场景的使命是"快速推进"，五感叠加会拖慢节奏并制造AI味"伪细腻"。违例："他走进院子，闻到桂花香，听见鸟叫，感到微风拂面，看见落叶飘下。" → 改为单一视觉："他走进院子，桂花树下落了一地碎花。"
2. ❌ **accent场景禁止感官叠加**：点睛场景的使命是"一句话定格"，多感官会稀释焦点。违例："铜钱冰凉，带着铁锈味，边缘磨损。" → 改为单一感官："铜钱棱角硌进肉里。"
3. ❌ **core场景禁止只写视觉**：核心场景须有"身体在场感"，纯视觉=摄像机而非角色体验。违例：全场景只写"他看见……" → 须补充触觉/听觉让角色"进入"场景。
4. ❌ **相邻场景感官词重复**：相邻场景使用相同感官词（如"刺痛""嗡嗡声"）=AI模式重复。须查 `references/感官细节库.md` 换词。

**与现有协议的关系**：
- 本模板是"场景蓝图→感官策略"字段的标准化填充器，不替代 `references/感官细节库.md` 的具体选词
- 与03a"质感注入协议"第4条"感官精准投放"协同：本模板管"选哪些感官"，质感注入协议管"怎么写得有质感"
- 与03a"铁律4 情绪词配额制"协同：core场景允许L3情绪强度，transitional/accent须L1纯Show

### 流水线位置与契约

```
0项目初始化 → 1设定架构 → 2大纲 → 3细纲 → ⭐4a扩写执行(本步) → 4b质量自检 → 5精修 → 6去AI味
```

- **上游**：← 02a细纲（本章细纲+衔接包+记忆锚点）← 00设定（反例对照表+power_system+number_anchors）
- **下游**：→ 03b质检 → 04精修师（初稿+定位报告+锐度保护清单）
- **衔接包强制执行**：衔接包"禁止矛盾"清单是硬约束（如"李四左臂骨折"→ 不得双手操作）
- **数值/战力锁定**（V4.7）：正文数值须与number_anchors一致；战斗表现须在power_system能力边界内
- **字数契约**：每章 3500-4500 字，结构类型（A-H）由细纲指定
- **可读不可写**：衔接包和记忆锚点由02a产出、05校验，03a只读
- **sync_status检查（V5.4新增）**：扩写前检查衔接包 sync_status 字段：

  - `unverified` → 正常读取P0字段，在输出头部标注"⚠️ 建议运行 merge-delta.js 验证状态"
  - `degraded` → ⚠️ 降级模式：仅使用P0中校验通过的字段，失败字段从前章正文推断。在输出头部标注"⚠️ 本章衔接包状态降级，以下P0字段可能不准确：[失败字段列表]"

### 扩写前置上下文强耦合协议（三维一体加载）

扩写前必须在内存中对齐四个维度的核心锚点，否则不准动笔：

```
【维度1：单章情节血肉（来自02细纲）】 本章主要冲突、台词路线、衔接包
【维度2：宏观情绪阀门（来自01大纲）】 情感温度标签（🔥大爆发/🌡️蓄力过渡/❄️压抑铺垫）
  └─ ⚠️ 蓄力/铺垫章→全面压制文字张力和高华词汇，禁止高光打斗和顶级怼人话术
【维度3：下文钩子倒逼（来自01/02）】 下一章爽点是什么？本章哪些细节为下一章服务？
  └─ ⚠️ 本章绝对禁止提前释放下一章的爽感
【维度4：情绪工程锚点（来自02细纲，V4.5）】 目标情绪[爽/虐/燃/酸/惧/暖]+情绪曲线+情绪锚点+情绪通路[A压抑爆发/B五感榨汁/C配角杠杆]
  └─ ⚠️ 不得退回情绪词直给（"他很悲伤/愤怒/绝望"）。若02a未提供情绪蓝图→向02a回退要求补充
```

## 规则分层体系

### 质感注入协议（V4.4 — 正面引导 · 完整版见 references/03_质感注入协议.md）

> 反注水铁律管"不该写什么"（刹车），本协议管"该怎么写"（油门）。

**五大技法**：
1. **动作链写生理状态**：不写"他很饿"，写3-4个连续具体动作。✅"他撕下半块饼，先闻了闻，再塞进嘴里慢慢含化"。边界：3-4个连续动作传达明确状态→保留；单个无意义触碰→删除。
2. **具体名词替代抽象形容词**：不写"很旧"，写带时间痕迹的具体物。✅"门轴上的锈已经爬满了合页"
3. **设定藏在生活里**：不写设定文档，写分配制度/交易场景/生活习惯。✅"天光石按户分配，每户每天拳头大的一块"
4. **感官精准投放**：日常过渡用1个精准细节锚定（不是什么都不写），核心场景用3-5感官叠加
5. **对话要有"人味"**：口癖/跑题/言不由衷/信息差

### 情绪工程协议（V4.5 — 正面情绪引导 · 完整版见 references/03_情绪工程协议.md）

> 质感注入管"读者能不能看见画面"，本协议管"读者读完身体有没有反应"。

**三层架构**：
- **Layer 1 情绪蓝图（02细纲层）**：目标读者情绪 + 情绪曲线 + 情绪公式检验
- **Layer 2 情绪锚点（03扩写层）**：小而具体+角色独有+日常受阻+不解释的物理物件/动作
- **Layer 3 情绪通路（03扩写+04精修层）**：A压抑→爆发反差工程 / B五感榨汁机法 / C配角杠杆法

### 最高优先级铁律：反注水与白噪音绝对禁区

> ⚠️ 本铁律优先级高于下方所有规则。触发任何一条=严重违规，必须立即修正。

```
1. 绝对禁止"循环式排查"：多个相似物品禁止超过1次重复描写。用"一路看过去"高度概括。
   ❌ "第一只展柜""第二只展柜""第三只展柜"……  ✅ "他一路看过去，前几只展柜的维护记录都差不多。"

2. 绝对禁止"无意义的物理触碰反馈"：普通物品的触感/磨损声/笔帽咔哒声等微动作音效=删除。
   ❌ "他把笔帽拔下来，又按回去，发出一声轻微的咔哒声。"  ✅ "他接过笔，签了名。"

3. 绝对禁止"滥用感官比喻"：日常场景禁止"像从地底深处""比时间更冷""沉积了千百年"等过度文艺修辞。
   ❌ "像有人从地底深处抽出一口沉积了千百年的空气。"  ✅ "空调开得很足，屋里凉。"

4. 绝对禁止"心理活动套娃"：禁止"他原本预期是...但他发现..."废话句式。禁止对日常事物哲学思辨。
   ❌ "遗忘是冷的。比十八度的空调更冷。比三千年没开过的门更冷。"  ✅ "他没再想。关了灯。"

5. 绝对禁止"潜台词解释"：永远不要替读者总结！禁止句式："他发现…/这说明…/这意味着…/他在等…/他心里明白…"
   禁止"心理实况转播"（"在心里算了一遍""想了三遍"）。用物理动作/生理反应/环境变化折射心理。
   【绝对违禁词库】叙述语严禁："觉得/知道/意味着/是因为/看出规律/异乎寻常/看来/他发现/这说明/他心里明白/他在等/他意识到/显然/可见"→出现即删除，替换为客观物证。
   【推理留白机制】角色发现秘密→只写"客观物证"，不准写结论。
   ❌ "这个温度不是给人定的，是给展品定的，展品怕热。"  ✅ "温控表上的指针死死钉在十八度。白术吐出一口白气。"

6. 绝对禁止"弱智明喻"（戒断"像"字）：禁止用"像/仿佛"强行渲染神秘感。悬疑感来源于不合理的客观事实。
   【比喻质量门（V4.4）】通过三重门的比喻密度≤3次/千字（V5.3统一阈值，引用check-ai-patterns.js结果：≥3次/千字advisory, ≥4次/千字blocking。不再设"≤4处/章"绝对数量限制）：
   ├─ 精准门：喻体是读者能立刻产生感官联想的具体物（✅"胃里像有只手在拧" ❌"像沉积了千百年的空气"）
   ├─ 记忆门：比喻创造画面记忆点，非填充描写间隙
   └─ 不可替代门：删掉换成直白描写效果一样好？→是→删掉比喻
   角色对话中的比喻不受限制。
   【句法代偿公式】想用"A像B"时，强制替换为：
   ├─ 物理痕迹法：直接描写A本身的材质/气味/损耗/数据
   └─ 生理反馈法：通过主角身体/感官直接承受该状态
```

### 跨章句式去重协议（V4.3 — V5.1改为脚本驱动）

> 扩写前运行 `node scripts/extract-used-patterns.js --json <最近N章文件>`，脚本自动提取近N章已用句式模式（16种高频AI模式），替代手动维护清单。

2. **已用句式清单喂入**：收到`<used_patterns>`后，本章禁止复用清单中模板句式完整结构
3. **降级方案**：数据缺失时①优先从近3章正文运行脚本提取②正文不可获取→跳过去重检查，标注"[跨章去重降级]"③不得因数据缺失停止生成
4. **模板句式识别标准**：固定骨架只替换关键词的句式。本章用过的模板骨架，下2章内不得再用
5. **输出**：定位报告中附"本章已用模板句式清单"，供下一章喂入

### 跨章场景元素去重协议（V4.4 — V5.1改为脚本驱动）

> 脚本自动提取 scene_openers 和 body_reactions 作为比对依据。

1. **场景类型识别**：识别场景类型（救人/战斗/对话对峙/探索/日常等），与近5章同类→触发去重检查
2. **去重矩阵**（同类场景须差异化5维度）：

   | 维度 | 复用=违规 | 差异化=合规示例 |
   |------|----------|---------------|
   | ①外观特征 | 格子衬衫/短发 | 西装/校服/长发/光头 |

3. **差异化策略**：换身份/换伤情/换关系/换环境/换时间
4. **输出**：定位报告中附"本章场景元素清单"


### 最终输出工作流：先脱水，后出稿（V4.2 — Last Line 格式锁）

```

5. 跨章去重核对（句式+场景+信息）：①句式去重——本章是否复用了<used_patterns>中模板骨架？是=换句式重写。②场景元素去重——同类场景5维度是否≥3个完全不同？否=按去重矩阵重写。③跨章信息去重——本章是否有与近5章相同的设定信息被完整重复描述？是=第2次一句话带过。④对话科普化核对——是否存在"双方都懂却讲给读者听"的对话？是=改为动作/观察带出或加入信息差。⑤章内信息去重——同一流程/事件是否从两视角完整描述了两遍？是=合并为一段。
6. 章末收尾轮换核对：本章收尾方式是否与上一章相同？是=换一种重写。
7. 质感自检（防"技术合规但灵魂缺失"的空文本）：
    → 动作链≥2处用"3-4个连续动作"传达生理/情绪状态？→ 否=补充
    → 具体名词≥3处用"具体物+时间痕迹"替代抽象形容词？→ 否=补充
    → 设定生活化≥1处通过"生活场景"释放？→ 否=补充
    → 对话人味：有口癖/跑题/言不由衷/信息差？→ 否=增加人味
    → 画面感测试：闭上眼能否"看见"本章关键场景？→ 否=补充精准细节
8a. 常识性错误终检：
    → 是否有描写违反物理/材料/生理/空间/时间/因果常识？是=禁止出稿。奇幻超自然须有设定依据。
8b. 语句可读性终检：
    → 是否存在六类可读性硬伤（成分残缺/指代不清/搭配断裂/过度嵌套/逻辑跳断/主语偷换）？"读出声来测试"最复杂3个长句需不需回头重读？是=不通过。
8c. 情绪转移校验（V4.5）：
```

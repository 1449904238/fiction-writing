## 使用流程

### 第一步：确定导入范围

| 信息 | 说明 |
|------|------|
| 已有正文 | 多少章？每章约多少字？ |
| 保存位置 | 正文文件路径 |
| 当前进展 | 写到什么阶段了？有无细纲/设定/大纲？ |
| 目标 | 续写/精修/去AI味/全流程 |

### 第二步：扫描正文 → 提取结构化数据

逐章扫描已有正文，提取以下信息：

```
【章节信息表】

| 章节 | 字数 | 核心事件 | 登场角色 | 关键物品 | 伏笔/悬念 |
|------|:----:|----------|----------|----------|-----------|
| Ch.1 | 4000 | 师父跑了 | 沈夜,赵叔 | 铜钱 | 师父去哪了 |
| Ch.2 | 4200 | 第一个案子 | 沈夜,柳如烟 | 镜子 | 画皮怪 |
| ... | ... | ... | ... | ... | ... |
```

### 第三步：重建设定文档

从正文中提取并重建 `00_产出_设定文档.md`：

```
【设定提取模板】

1. 世界观规则
   ├─ 从正文提取：角色提到了什么规则？
   ├─ 从事件提取：发生了什么说明规则的？
   └─ 待补充：有哪些规则已暗示但未明说的？

2. 角色档案
   ├─ 角色名：（从正文提取）
   ├─ 身份：（从对话/叙述提取）
   ├─ 性格：（从行为/对话提取）
   ├─ 能力/金手指：（从表现提取）
   └─ 关系：（从互动线提取）

3. 核心冲突
   ├─ 主要冲突：（从主线提取）
   ├─ 次要冲突：（从支线提取）
   └─ 未解决的冲突：（作为伏笔保留）

4. JSON元数据
   ├─ characters: [从正文提取的角色]
   ├─ relationships: [从互动推断的关系矩阵]
   └─ anti_patterns: [从正文识别到的AI味/问题]
```

### 正文反推启发式规则（V2.0新增 — 从已有正文提取伏笔/设定）

#### 1. 伏笔识别启发式
扫描正文中的以下信号，识别可能被埋下的伏笔：

- **信号词扫描**："奇怪的是/不对劲/似乎/好像/不知道为什么/莫名/隐约"
- **未解释的细节**：提及但未在3章内解释的物品/行为/对话
- **角色欲言又止**："算了/没什么/以后再说/不是现在"
- **读者疑问点**：角色做了不合常理的事但未解释原因
- **反常强调**：叙事中对某个细节给了异常多的笔墨但看似无关紧要

对每个候选伏笔，标注：发现章号 | 相关角色 | 推测类型（物品伏笔/对话伏笔/行为伏笔/细节伏笔）

#### 2. 设定反推启发式
- **世界观**：扫描"在这个世界/XX的规矩是/从来都是/众所周知"等规则性表达
- **力量体系**：扫描战斗描写中的能力表现+限制条件+修炼方式
- **角色关系**：扫描对话中的称谓/态度/历史提及，构建关系图谱
- **地理设定**：扫描地名+距离+方位描述，构建地图
- **社会结构**：扫描组织/势力/阶级/职业相关的描述

#### 3. 伏笔回收识别
- 扫描"原来/果然/所以这就是/难怪/怪不得/这就说得通了"等揭示性表达
- 匹配前文伏笔候选列表
- 标记为"已回收"并记录回收章号
- 未回收的伏笔标注为ACTIVE，进入伏笔追踪表

#### 4. 提取置信度标注
每个提取结果标注置信度：
- **高**：正文中有明确文本证据
- **中**：通过上下文推断，证据较充分
- **低**：仅凭信号词推测，需用户确认

### 第四步：重建大纲框架

从章节事件序列重建 `01_产出_分卷大纲.md`：

```
【大纲重建模板】

第1卷：{卷名}（Ch.1 - Ch.X）
├─ 卷核心冲突：{提取自正文}
├─ 情绪曲线：
│   ├─ Ch.1：入场（情绪：X）
│   ├─ Ch.2：铺垫（情绪：X）
│   └─ ...
├─ 卷高潮：{提取自正文}
├─ 卷结尾钩子：{提取自正文}
└─ 未解决问题：{作为下一卷线索}
```

### 第五步：重建细纲与衔接包

为已有章节补写细纲，为新章节编写细纲：

**已有章节**：补写精简细纲 + 衔接包（向后兼容）
**新章节**：按02细纲编写技能的标准流程编写

### 第六步：重建伏笔追踪

扫描全文中已埋设但未回收的伏笔/悬念/冲突：

```
【伏笔重建表】

| ID | 类型 | 埋设章节 | 内容 | 暗示强度 | 预计回收 | 当前状态 |
|----|------|:--------:|------|:--------:|:--------:|:--------:|
| F001 | 悬念 | Ch.1 | 师父去哪了 | 强 | 第1卷末 | ACTIVE |
| F002 | 物品 | Ch.3 | 铜钱裂纹 | 中 | 第2卷 | ACTIVE |
| F003 | 角色 | Ch.5 | 柳如烟真实身份 | 强 | 第3卷 | DORMANT |
```

## 输出

导入完成后，产出以下文件（存入项目目录）：

| 文件 | 说明 |
|------|------|
| `设定/00_产出_设定文档.md` | 从正文重建的设定文档 |
| `大纲/01_产出_分卷大纲.md` | 从章节序列重建的大纲 |
| `细纲/02_产出_全书细纲.md` | 已有章节精简细纲+新章节完整细纲 |
| `追踪/伏笔追踪表.md` | 从全文扫描的活跃伏笔 |
| `对标/已有正文文风分析.md` | 从正文提取的文风特征（供04/05参考） |

---

## 流水线契约

- 本技能不修改已有正文，只重建工程架构
- 重建的设定/大纲/细纲可能与原作者的意图有偏差，建议导入后人工确认
- 对于已有正文中识别的AI味问题，在伏笔追踪表中标注"建议去AI味"而非直接修改
- 导入完成后，从对应步骤进入主流水线

# 短篇写作技能 V1.0

> 短篇网文（知乎盐言风格/故事会/短篇合集）的独立写作流水线。三步完成，单次会话即可输出一篇完整的8K-15K字短篇。

---

## 与长篇流水线的区别

| 维度 | 长篇流水线 | 短篇流水线 |
|------|-----------|-----------|
| 字数 | 100万+字 | 8K-15K字 |
| 步骤 | 7步（含项目初始化） | 3步 |
| 细纲 | 全书完整细纲（一次性） | 单篇大纲（一次性） |
| 扩写 | 逐章扩写 | 一次性生成 |
| 精修 | 03/04/05分离 | 03+04合并 |
| 去AI味 | 10 Gate单独步骤 | 10 Gate最终审核 |
| 频率 | 每章循环 | 单次完成 |

---

## 三步流水线

### 第1步：定位与设定

**核心问题**：这篇短篇的"一句话卖点"是什么？

```
【短篇定位模板】

1. 类型标签：悬疑/言情/都市/奇幻/恐怖/现实
2. 一句话梗概：[主角]在[情境]下，做了[某事]，导致[结果]
3. 情感目标：让读者感到[爽/虐/怕/甜/思]
4. 对标风格：[如：类似知乎盐言《XXX》的叙事风格]
5. 目标字数：8K-12K字

6. 核心设定（如有特殊世界观）：
   ├─ 规则1：
   ├─ 规则2：
   └─ 规则3：

7. 角色表（1主角+2-3重要配角）：
   ├─ 主角：[姓名][身份][性格核心]
   ├─ 配角1：[姓名][功能]
   └─ 配角2：[姓名][功能]
```

### 第2步：写作（扩写+精修合并）

**执行原则**：

1. **单次完成**：一次性生成完整短篇正文
2. **自带精修**：写作时直接应用长篇流水线中04精修师的技法（潜台词/留白/金句）
3. **字数控制**：8K-15K字，单篇自包含
4. **起承转合**：开头钩子 → 冲突升级 → 高潮反转 → 余韵收尾

**结构模板**：

```
【短篇结构模板】

├─ 开头（500-1000字）：钩子+角色+场景
│   ├─ 第1句：必须抓住读者
│   ├─ 前200字：建立冲突或悬念
│   └─ 前500字：角色出现+核心问题提出
│
├─ 发展（3000-5000字）：冲突升级+信息释放
│   ├─ 3-5个情节推进点
│   ├─ 过程中逐步释放背景信息
│   ├─ 穿插小高潮/小反转
│   └─ 配角互动增加层次
│
├─ 高潮（2000-4000字）：核心冲突解决
│   ├─ 最大的反转/揭示/对抗
│   ├─ 所有伏笔回收
│   └─ 情绪达到顶点
│
└─ 结尾（500-1000字）：余韵+收尾
    ├─ 情感释放或留白
    ├─ 呼应开头
    └─ 留给读者回味空间
```

### 第3步：去AI味审核

复用长篇流水线的 `05_去AI味精修师.md` 的10 Gate系统，逐章检测：

| Gate | 检测项 | 优先级 |
|------|--------|:------:|
| Gate 1 | 禁用词过滤 | P0 |
| Gate 2 | 句式去套路 | P0 |
| Gate 3 | 心理描写外化 | P1 |
| Gate 4 | 节奏打碎 | P1 |
| Gate 5 | 对话去腔调 | P1 |
| Gate 6 | 结尾去升华 | P0 |
| Gate 7 | 意象疲劳检测 | P1 |
| Gate 8 | 叙述语调变化 | P1 |
| Gate 9 | 剧透式叙述/信息不对称 | P1 |
| Gate 10 | 情节信息含金量 | P0 |

短篇字数短，建议全文过10 Gate，而非抽样检测。

## 短篇转长篇

短篇成功（读者反馈好/数据表现佳）后，可扩展为长篇：

1. 保留原短篇的核心设定和主角
2. 扩展世界观（增加规则复杂度）
3. 设计更长的冲突弧线
4. 进入长篇流水线从步骤2（大纲）开始

---

## 流水线契约

- 短篇写作消耗0304合并步骤（精修在写作时直接完成）
- 05去AI味精修师是***唯一共享步骤***——与长篇共用同一审核标准
- 短篇产出物存入项目目录的 `短篇/` 子目录（如适用）
- 不占用长篇流水线的衔接包链/伏笔追踪等长线机制

# 短篇写作技能（V2.1 — 长篇规则子集提取）

> 本技能为长篇流水线（03a/03b/04/05）的短篇适配版，适用于单篇1.5万-3万字的短篇/中短篇创作。
> 核心理念：保留长篇流水线中影响产出质量的核心规则，删除跨章管理机制。
> **V2.1 重大更新**：① 分段执行协议（15K-30K中篇分3-5段+迷你衔接包+段间暂停）；② 自检项从10项增至15项（恢复P1级检查）；③ 审美基准对齐04 V5.2（网文顶级质感，非出版级）；④ 去AI味Gate对齐05核心7 Gate；⑤ 收敛轮数调整为"2轮基础+1轮可选深度"；⑥ 题材适配说明。

## 适用范围
- 单篇字数：8K-15K字（中短篇，单次完成）/ 15K-30K字（中篇，分段执行）
- 不需要跨章衔接、滚动建纲、伏笔追踪表
- 短篇可自包含完成，中篇分段完成

## 与长篇流水线的区别

| 维度 | 长篇流水线 | 短篇流水线 |
|------|-----------|-----------|
| 字数 | 100万+字 | 8K-30K字 |
| 步骤 | 7步（含项目初始化） | 3步 |
| 细纲 | 全书完整细纲（滚动建纲） | 单篇大纲（一次性） |
| 扩写+精修 | 分离(03a→03b→04) | 合并为单步 |
| 去AI味 | 11 Gate+三遍法 | 核心7 Gate+两遍法 |
| 自检项数 | 40项(P0+P1+P2) | 15项(P0+精选P1) |
| 收敛轮数 | 3轮 | 2轮基础+1轮可选深度（共3轮上限） |
| 衔接包 | 需要（V5.2 P0/P1分级） | 中篇分段时用迷你衔接包（≤100字） |
| 滚动建纲 | 需要 | 不需要 |
| 跨章去重 | 需要 | 不需要 |
| 伏笔追踪 | 需要 | 不需要（短篇伏笔少，人工把控） |
| 频率 | 每章循环 | 单次/分段完成 |
| 审美基准 | 网文顶级质感（04 V5.2） | 网文顶级质感（与04 V5.2对齐） |

## 分段执行协议（V2.1新增）

> **核心问题**：V2.0宣称"单次会话输出完整短篇"，但15K-30K字的中篇在单次会话中难以保证质量（上下文窗口限制+后半段质量递减）。V2.1引入分段执行协议解决此问题。

### 字数分级与执行模式

| 字数范围 | 执行模式 | 段数 | 每段字数 | 衔接包 |
|----------|----------|:----:|:--------:|--------|
| **8K-15K**（中短篇） | 单次完成 | 1段 | 全篇 | 不需要 |
| **15K-30K**（中篇） | 分段执行 | 3-5段 | 5K-8K/段 | 迷你衔接包 |

### 中篇分段执行流程

```
【分段执行流程 — 15K-30K中篇】

Step 1: 定位（同短篇，确定全篇结构+字数预算+分段计划）
  └─ 输出：分段计划表（每段的目标字数+情节范围+核心情绪）

Step 2: 逐段写作（每段独立执行扩写+精修合并）
  ├─ 第1段：开头+发展前半
  │   └─ 完成后生成迷你衔接包 → 暂停 → 等待用户确认
  ├─ 第2段：发展后半+高潮前奏
  │   ├─ 读取上段迷你衔接包 → 继续写作
  │   └─ 完成后生成迷你衔接包 → 暂停 → 等待用户确认
  ├─ 第3段：高潮+收尾
  │   └─ 完成后进入Step 3
  └─ （如需4-5段，重复上述流程）

Step 3: 去AI味（全文统一执行，非分段执行）
  └─ 核心7 Gate + 两遍法 → 收敛终止协议
```

### 迷你衔接包格式（V2.1新增）

> 段间衔接使用迷你衔接包，不使用长篇的V5.2 P0/P1分级衔接包。迷你衔接包极简设计，确保段间连贯即可。

```
【迷你衔接包 — 段间衔接】（限100字以内）

角色状态：{主角当前位置/身体状态/心理状态}
当前目标：{主角此刻想做什么}
未解悬念：{尚待揭示的1-2个关键悬念}
段尾锚点：{最后一个场景的物理细节，供下段开头呼应}
```

**示例**：
```
角色状态：沈夜在县衙偏房，左肩被画皮怪抓伤（轻伤），情绪从恐惧转为愤怒
当前目标：查清画皮怪的来历，找到柳如烟
未解悬念：铜钱为什么对画皮怪有反应？柳如烟去了哪里？
段尾锚点：桌上铜钱微微发烫，裂纹中渗出一丝血色
```

### 段间暂停协议

- 每段写作完成后，**必须暂停并等待用户确认**后再继续下一段
- 暂停时输出：本段字数 | 迷你衔接包 | 下段计划（预计字数+情节范围）
- 用户可指示：继续 / 调整方向 / 修改本段
- 此机制确保中篇质量不因单次会话过长而递减

### 与03a/03b的兼容性说明

> 短篇/中篇的分段执行**不使用03a/03b的扩写-质检分离流程**。短篇流水线是长篇流水线的精简子集，扩写+精修合并为单步执行。分段执行中的"段"不等于长篇的"章"——段是输出管理单位，不涉及02a细纲、02b质检、衔接包链等长篇机制。迷你衔接包是段间连贯性保障，不是02a生产的V5.2衔接包。

## 三步流程

### Step 1：定位（5分钟）

**核心问题**：这篇短篇的"一句话卖点"是什么？

```
【短篇定位模板】

1. 类型标签：悬疑/言情/都市/奇幻/恐怖/现实
2. 一句话梗概：[主角]在[情境]下，做了[某事]，导致[结果]
3. 情感目标：让读者感到[爽/虐/怕/甜/思]
4. 对标风格：[如：类似知乎盐言《XXX》的叙事风格]
5. 目标字数：8K-15K字（中短篇）/ 15K-30K字（中篇）

6. 核心设定（如有特殊世界观）：
   ├─ 规则1：
   ├─ 规则2：
   └─ 规则3：

7. 角色表（1主角+2-3重要配角）：
   ├─ 主角：[姓名][身份][性格核心]
   ├─ 配角1：[姓名][功能]
   └─ 配角2：[姓名][功能]
```

**定位补充（V2.0新增）**：

- **核心情绪定位**：这篇短篇要让读者读完后的核心感受是什么？
- **爆点定位**：短篇只需要1-2个爆点，确定位置
- **字数预算分配**：开头(15%)→发展(50%)→高潮(25%)→收尾(10%)
- **分段计划（中篇专用，V2.1新增）**：如目标字数>15K，确定分段数+每段情节范围

**标点规范**：与长篇一致（省略号≤5/破折号≤4/感叹号≤15）

#### 题材适配说明（V2.1新增）

> 短篇也可选择题材方向，不同题材影响质感技法的侧重。

| 题材方向 | 质感侧重 | 必用技法 | 可跳过技法 |
|----------|----------|----------|-----------|
| 悬疑/推理 | 冷峻氛围+信息差控制 | 冰山原则、感官精准投放 | 脏感注入 |
| 言情/甜宠 | 情感细腻+身体语言 | 对话人味、动作链写生理状态 | — |
| 都市/现实 | 生活质感+烟火气 | 脏感注入、具体名词替代形容词 | 设定藏在生活里 |
| 奇幻/玄幻 | 世界观沉浸+力量感 | 设定藏在生活里、感官精准投放 | 脏感注入 |
| 恐怖/惊悚 | 恐惧递进+感官压迫 | 感官精准投放（多维叠加）、冰山原则 | 对话人味（对话少） |

### Step 2：写作（核心执行）

合并03扩写+04精修为单步执行，保留以下长篇核心规则。

**执行原则**：
1. 单次/分段完成：中短篇一次性生成，中篇分段生成
2. 自带精修：写作时直接应用长篇流水线中04精修师的技法（潜台词/留白/金句）
3. 字数控制：8K-30K字，单篇自包含
4. 起承转合：开头钩子 → 冲突升级 → 高潮反转 → 余韵收尾

**结构模板**：

```
├─ 开头（15%）：钩子+角色+场景
│   ├─ 第1句：必须抓住读者
│   ├─ 前200字：建立冲突或悬念
│   └─ 前500字：角色出现+核心问题提出
│
├─ 发展（50%）：冲突升级+信息释放
│   ├─ 3-5个情节推进点
│   ├─ 过程中逐步释放背景信息
│   ├─ 穿插小高潮/小反转
│   └─ 配角互动增加层次
│
├─ 高潮（25%）：核心冲突解决
│   ├─ 最大的反转/揭示/对抗
│   ├─ 所有伏笔回收
│   └─ 情绪达到顶点
│
└─ 收尾（10%）：余韵+收尾
    ├─ 情感释放或留白
    ├─ 呼应开头
    └─ 留给读者回味空间
```

**短篇字数控制**：
- 按结构分段生成，每段完成后检查字数
- 若总字数<8K：在"发展"段补充冲突层次
- 若>15K（中短篇）：压缩"发展"段的过渡场景
- 中篇（15K-30K）按分段执行协议处理，段间暂停

#### 2.1 审美基准对齐04 V5.2（V2.1更新）

> **V2.1关键调整**：审美基准从"出版级质感"调整为"网文顶级质感"，与04 V5.2完全一致。
> 对标对象：猫腻（文人气韵）/烽火戏诸侯（江湖质感）/愤怒的香蕉（细腻日常）/烟雨江南（冷峻氛围）。
> 底层逻辑：沉浸式画面感视角，非客观摄影机视角。

#### 2.2 反注水最高铁律（6条全保留）
1. 禁止循环式排查（多个相似物品挨个描写超过1次）
2. 禁止无意义物理触碰反馈（普通物品的触感/磨损声/笔帽咔哒声）
3. 禁止滥用感官比喻（日常场景用"像从地底深处"等）
4. 禁止心理活动套娃（"他原本预期...但他发现..."）
5. 禁止潜台词解释（"他发现/这说明/这意味着"）
6. 禁止弱智明喻（用"像/仿佛"强行渲染氛围，全篇超过2处）

#### 2.3 否定排比三连零容忍
- "没有…没有…没有…" / "不是…不是…不是…" → 立即删除
- 否定排比+升华句（"而是…/只有…/那是一种…"）→ 立即删除整段
- 全篇容忍度：0次（对话中可保留）
- **短篇专用标准**：全篇否定翻转（"不是A是B"）≤3次

#### 2.4 质感注入五大技法（全保留）
1. 动作链写生理状态：3-4个连续动作传达角色状态（非微动作白噪音）
2. 具体名词替代形容词："桌上放着半壶凉透的茶"而非"桌上放着茶具"
3. 设定藏在生活里：世界观通过日常场景释放，非说明文
4. 感官精准投放：日常极简白描 / 关键异常点多维叠加（二分法）
5. 对话人味：口癖/跑题/言不由衷/信息差

#### 2.5 冰山原则五大铁律（全保留）
1. 只写水面上的1/8，让读者感受到水面下的7/8
2. 角色的过去通过"碎片+矛盾"释放，非履历表
3. 情绪不直说，用物理锚点承载（"他攥着那张纸，指节发白"）
4. 关键信息通过"角色反应"暗示，非叙述者解释
5. 留白比填满更有力量——读者脑补的永远比写出来的好

#### 2.6 "不是A是B"句式限流
- 全篇≤3次（短篇比长篇章限度略宽）
- 超出 = 改用直接陈述/动作描写

#### 2.7 常识性错误零容忍
- 物理/材料/生理/空间/时间/因果常识检查
- 超自然行为必须有设定依据

**短篇精修规则子集说明**：
- 必用技法：感官入口重定向、潜台词对话法、反标签行为法（这3项是短篇质感的核心）
- 选技法：脏感注入（仅现实题材）、内心矛盾外化法（仅主角视角）、感官锚点深度穿插（仅关键场景）
- 跳过技法：叙述厚度保障中的"对话与叙述黄金比例"（短篇对话比例可灵活）、模块R情感收束（短篇高潮集中，不需要长线收束）

### Step 3：去AI味（对齐05核心7 Gate — V2.1更新）

> **V2.1关键调整**：从原11 Gate精简为05的核心7 Gate，与05去AI味精修师的核心检测项对齐。两遍法保持不变。

保留以下核心7 Gate（映射05的核心Gate 1-6 + Gate 11），两遍法简化为两遍：

#### Pass 1：去泛化（必做 — 核心4 Gate）
- **核心Gate 1**（对应05 Gate 1）：禁用词过滤 — 一级+二级禁用词全量扫描
- **核心Gate 2**（对应05 Gate 2）：句式去套路+对话去腔调 — 否定翻转/破折号/碎句号/AI腔对话特征
- **核心Gate 3**（对应05 Gate 3）：心理描写外化 — 心理直给词扫描（情绪词直说检测）
- **核心Gate 4**（对应05 Gate 6）：结尾去升华 — 套路结尾类型检测+正确结尾方式

#### Pass 2：回自然感（必做 — 核心3 Gate）
- **核心Gate 5**（对应05 Gate 9）：剧透式叙述与信息不对称检测 — 拒绝动作意图直给
- **核心Gate 6**（对应05 Gate 10）：情节信息含金量与注水率检测 — 连续>200字信息零推进→脱水
- **核心Gate 7**（对应05 Gate 11）：解释腔/上帝感/安排感检测 — "这说明/显然/意味着"扫描

> 短篇字数短，建议全文过核心7 Gate，而非抽样检测。
> 原11 Gate中的Gate 4(节奏打碎)/Gate 7(意象疲劳)/Gate 8(叙述语调)为可选项，仅在用户要求深度打磨时执行。

#### 收敛终止协议（V2.1更新）

> **V2.1关键调整**：从2轮调整为"2轮基础+1轮可选深度"，共3轮上限。

```
【收敛终止协议 — 2轮基础+1轮可选深度】

第1轮（基础）：核心7 Gate全量执行
├─ 消除70%AI味（高收益）
├─ 重点：Gate 1禁用词 + Gate 2句式套路 + Gate 3心理直给
└─ 完成后检查：如核心7 Gate全部通过 → 可直接定稿

第2轮（基础）：核心7 Gate复查 + 画面感测试
├─ 消除15%AI味（中收益）
├─ 重点：Gate 5剧透 + Gate 6信息含金量 + Gate 7解释腔
└─ 完成后检查：如仍有≥3个S1级问题 → 进入第3轮

第3轮（可选深度）：11 Gate全量 + 画面感深度打磨
├─ 仅在用户要求"深度打磨"或前两轮后仍有S1级问题时执行
├─ 恢复Gate 4/7/8等可选项
├─ 消除剩余5%AI味（低收益）
└─ 完成后：剩余问题标记[需复核]移交用户

[需复核]项给出3个选项：A.接受现状 B.人工改写（给方向） C.放弃修改
```

**收敛判定**：
- 连续两轮无S1级改动 → 提前终止
- 第3轮执行后 → 强制终止，剩余项移交用户

## 短篇自检清单（15项 — V2.1从10项扩充）

> **V2.1变更**：从10项P0精简版扩充至15项，恢复部分P1级检查。

### P0级必查（10项，保留）
1. 字数是否在目标范围内？
2. 核心情绪是否到达？（读完有身体反应）
3. 反注水6条铁律逐条检查
4. 否定排比三连0次
5. "不是A是B"≤3次
6. 常识性错误0个
7. 画面感测试通过（闭眼能"看见"关键场景）
8. 对话有人味（口癖/跑题/信息差）
9. 比喻通过质量门且≤6处
10. 章末收尾干净（非排比/非抒情/非省略号）

### P1级检查（5项，V2.1恢复）
11. **句式复用检测**（V2.1恢复）：全篇是否存在连续3段以上句式雷同？（主谓宾/主谓/被动句的机械重复）
12. **感官入口检查**（V2.1恢复）：每个关键场景是否有≥2种感官入口？（视觉only开头标记为AI-like）
13. **对话零补丁检查**（V2.1恢复）：对话后是否有"解释性补丁"？（如"他这么说，其实是因为..."→删除，用动作/表情替代）
14. **短篇专用：否定翻转≤3次**（短篇比长篇限度宽，但全篇不超过3次"不是A，而是B"句式）
15. **短篇专用：爆点到位**（1-2个爆点是否在预定位置爆炸？高潮段是否有最大反转/揭示？）

## 短篇转长篇

短篇成功（读者反馈好/数据表现佳）后，可扩展为长篇：

1. 保留原短篇的核心设定和主角
2. 扩展世界观（增加规则复杂度）
3. 设计更长的冲突弧线
4. 进入长篇流水线从步骤2（大纲）开始
5. 短篇正文可经07导入师反向提取设定/大纲/伏笔后接入长篇流水线

## 与长篇流水线的差异对照
| 维度 | 长篇 | 短篇 |

| 跨章去重 | 需要 | 不需要 |
| 伏笔追踪 | 需要 | 不需要（短篇伏笔少，人工把控） |

## 流水线契约

- 短篇写作消耗03+04合并步骤（精修在写作时直接完成）
- 05去AI味精修师是唯一共享步骤——短篇使用其核心7 Gate（非全量11 Gate）
- 短篇产出物存入项目目录的 `短篇/` 子目录（如适用）
- 不占用长篇流水线的衔接包链/伏笔追踪等长线机制
- **V2.0核心变更**：从长篇流水线提取核心规则子集，删除跨章管理机制，保留影响产出质量的核心规则
- **V2.1新增**：分段执行协议解决中篇单次会话质量问题，迷你衔接包保障段间连贯
- **V2.1新增**：自检项恢复P1级检查（句式复用/感官入口/对话零补丁），15项保障短篇质量
- **V2.1新增**：审美基准对齐04 V5.2（网文顶级质感非出版级），去AI味Gate对齐05核心7 Gate
- **V2.1新增**：收敛轮数从2轮调整为"2轮基础+1轮可选深度"，第3轮为可选深度打磨
- **V2.1新增**：短篇可选题材方向，不同题材影响质感技法侧重（见题材适配说明表）
